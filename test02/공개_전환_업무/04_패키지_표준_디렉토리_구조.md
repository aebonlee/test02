# 패키지 표준 디렉토리 구조

> **버전:** 1.1
> **목적:** 다운로드 패키지에 포함될 폴더/파일 구조 정의

---

## 핵심 원칙

| 구분 | 설명 |
|------|------|
| **패키지 포함** | SSAL Works가 직접 제공하는 것 |
| **별도 설치** | SSAL Works가 제공하지 않음 (사용자가 직접 설치) |

### 코드 저장 원칙

```
1. Stage 폴더에 먼저 저장 (원본, 프로세스 관리용)
      ↓
2. git commit 시 Pre-commit Hook 자동 실행
      ↓
3. 루트 폴더로 자동 복사 (배포용)
```

**핵심:** Stage가 원본, 루트는 자동 복사본

## Pre-commit Hook 설치

패키지 설치 후 최초 1회 실행:

```bash
node scripts/setup-hooks.js
```

**설치 내용:**
- `.git/hooks/pre-commit` 파일 생성
- git commit 시 자동으로 `sync-to-root.js` 실행
- Stage 폴더 → 루트 폴더 자동 동기화

**동작 흐름:**
```
git commit 실행
      ↓
Pre-commit Hook 자동 실행
      ↓
sync-to-root.js 실행 (Stage → Root 복사)
      ↓
build-web-assets.js 실행 (선택)
      ↓
커밋 진행
```

## 스크립트 저장 원칙

```
1. 단일 대상 스크립트 → 해당 폴더에 저장
   예: generate-ordersheets-js.js → OrderSheet_Templates/

2. 복수 대상 스크립트 → 루트 scripts/에 저장
   예: build-web-assets.js → scripts/
```

## 참고: React/Next.js로 전환할 경우

> 현재 Vanilla를 사용하지만, React로 전환 시 아래 구조 적용

```
루트/
├── app/                    ← 페이지 + API + 레이아웃 통합
├── components/             ← 재사용 UI 조각
├── public/                 ← 정적 자원 (Vanilla의 assets)
└── lib/                    ← 유틸리티, 클라이언트 설정
```

**Vanilla → React 매핑:**
| Vanilla | React |
|---------|-------|
| pages/ | app/ |
| api/ | app/api/ |
| assets/ | public/ |
| index.html | app/page.tsx |
| 404.html | app/not-found.tsx |
