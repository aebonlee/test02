# 한국 특허청(KIPO) 특허출원서 양식 가이드

> **조사일**: 2025-12-20
> **출처**: 특허청, 법제처, 특허로

---

## 1. 제출 서류 구성 (4개)

| 서류 | 양식 | 설명 |
|------|------|------|
| **특허출원서** | 별지 제14호서식 | 출원인 정보, 발명의 명칭 등 |
| **명세서** | 별지 제15호서식 | 발명의 상세한 설명 + 청구범위 |
| **도면** | 별지 제17호서식 | 발명의 구성도, 플로차트 등 |
| **요약서** | 별지 제16호서식 | 발명 내용 요약 (400자 이내) |

---

## 2. 특허출원서 (별지 제14호서식)

### 기재사항
```
【서류명】특허출원서
【출원구분】
【참조번호】 (선택)
【출원인】
   【성명(명칭)】
   【출원인코드】
   【주소(영업소)】
   【국적】
【대리인】 (있는 경우)
   【성명(명칭)】
   【대리인코드】
   【주소(영업소)】
【발명의 국문명칭】
【발명의 영문명칭】
【발명자】
   【성명】
   【주소】
【심사청구】 여부
【출원서류 목록】
   【명세서】1 통
   【도면】1 통
   【요약서】1 통
【기타】
```

---

## 3. 명세서 (별지 제15호서식)

### 구성 항목 (필수)

```
【발명의 명칭】
   - 한글명칭
   - {영문명칭}

【기술분야】
   - 발명이 속하는 기술분야 기술

【발명의 배경이 되는 기술】
   - 종래 기술 설명
   - 종래 기술의 문제점

【발명의 내용】
   【해결하려는 과제】
   【과제의 해결 수단】
   【발명의 효과】

【도면의 간단한 설명】
   【도 1】 설명
   【도 2】 설명
   ...

【발명을 실시하기 위한 구체적인 내용】
   - 상세한 실시예 설명
   - 도면 참조 설명

【부호의 설명】
   - 도면에 사용된 부호 목록

【산업상 이용가능성】 (선택)
   - 산업적 활용 방안

【특허청구범위】
   【청구항 1】 (독립항)
   【청구항 2】 (독립항 또는 종속항)
   【청구항 3】 ...
```

### 작성 요령

1. **발명의 명칭**
   - 간명하게 발명 내용 표시
   - 영문 명칭은 { }안에 입력

2. **발명의 상세한 설명**
   - 20~30장 분량이 일반적
   - 통상의 기술자가 용이하게 실시할 수 있을 정도로 기재

3. **청구범위**
   - 독립항: 발명의 핵심 구성 기재
   - 종속항: 독립항을 한정/부가하여 구체화

## 4. 도면 (별지 제17호서식)

### 규격

| 항목 | 규칙 |
|------|------|
| **색상** | 흑백만 (착색 금지) |
| **해상도** | 300~400 dpi (300 권장) |
| **형식** | 흑백 TIFF |
| **도면 번호** | 【도 1】, 【도 2】 형식 |
| **페이지 표시** | 1/5, 2/5... (상단 중앙) |
| **문자 크기** | 세로 3.2mm 이상 |

### 도면부호 규칙

```
제1 실시예: 10, 20, 30... 또는 100, 110, 120...
제2 실시예: 200, 210, 220...
...

- 동일 구성요소는 동일 부호 사용
- 명세서와 도면의 부호 완전 일치 필수
- 발명의 설명에 사용하지 않은 부호 기재 금지
```

### 도면 유형
- 블록도/구성도
- 플로차트
- 시스템 구성도
- 상태 전이도
- 데이터 흐름도

---

## 5. 요약서 (별지 제16호서식)

### 구성

```
【요약】
   - 발명의 내용 요약 (400자 이내)
   - 기술분야, 해결과제, 해결수단, 효과 포함

【대표도】
   - 대표 도면 번호 (예: 도 1)
```

### 작성 요령
- 기술정보 용도로만 사용
- 보호범위 결정에는 사용 불가
- 간결하고 명확하게 작성

---

## 6. 청구범위 구조

### 독립항 (방법/시스템/저장매체)

```
【청구항 1】 (방법)
○○○ 방법에 있어서,
(a) ○○○ 하는 단계;
(b) ○○○ 하는 단계;
(c) ○○○ 하는 단계;
를 포함하는 방법.

【청구항 2】 (시스템)
청구항 1의 방법을 수행하기 위한 시스템에 있어서,
○○○ 모듈;
○○○ 모듈;
○○○ 모듈;
을 포함하는 시스템.

【청구항 3】 (저장매체)
청구항 1의 방법을 실행하도록 지시하는 명령이 기록된
비일시적 컴퓨터 판독가능 매체.
```

### 종속항

```
【청구항 4】
청구항 1에 있어서,
상기 (a) 단계는 ○○○를 포함하는 방법.

【청구항 5】
청구항 1에 있어서,
○○○가 ○○○ 형식인 것을 특징으로 하는 방법.
```

## 7. 명세서 작성 시 유의사항

### 필수 원칙

1. **실시가능요건**
   - 통상의 기술자가 과도한 실험 없이 실시 가능하도록 기재

2. **명확성**
   - 주어와 술어 관계 명확
   - 수식어와 피수식어 관계 명확

3. **용어 일관성**
   - 학술용어 또는 관용어 사용
   - 명세서 전체에 걸쳐 용어 통일

4. **종래기술 구분**
   - 공지기술과 신규기술 명확히 구분

5. **실시예 다수 기재**
   - 여러 실시예가 있으면 가능한 모두 기재

## 11. SAL 3D Grid 특허 출원서 작성 시 적용

### 문서 구성
1. **특허출원서** - 출원인/발명자 정보
2. **명세서** - 기존 초안 내용을 양식에 맞춰 재구성
3. **도면** - SVG로 10개 도면 제작 (도1~도10)
4. **요약서** - 400자 이내 요약

### 도면 목록 (기존 초안 기준)
- 도 1: SAL 3D Grid 개념도
- 도 2: SAL ID 스키마와 예시
- 도 3: SAL ID 파싱/정규화 플로우
- 도 4: ID→DAG 자동 구성 블록도
- 도 5: 스케줄링/레벨링 엔진
- 도 6: 3D 렌더링/UI 상호작용
- 도 7: 블록체인 스타일 ID 체인 구조
- 도 8: 리포트/알림/감사 로그 파이프라인
- 도 9: 도메인별 적용 예
- 도 10: 증분 재계산 프로세스 흐름도
