Wix, Squarespace, 노션...

- 조금만 커스텀하려면 결국 코드가 필요하다

그래서 질문을 바꿔봤습니다.

**"코딩을 배우지 않고도, 개발자처럼 웹사이트를 만들 수 있다면?"**

---

### SSAL Works를 소개합니다

SSAL Works는 **비개발자가 AI(Claude Code)와 협업하여 웹사이트를 직접 개발하는 플랫폼**입니다.

템플릿을 끼워 맞추는 게 아닙니다.
AI에게 "이렇게 만들어줘"라고 지시하면, AI가 실제 코드를 작성합니다.
당신은 기획자이자 디렉터가 됩니다.

---

### 7개월간 Sunny가 한 일

2025년 5월부터 12월까지 7개월간 AI와 함께 수많은 일을 했습니다.

- 사업기획서 **수십 개** 작성
- 시스템 설계서 **6개** 작성
- 특허출원 **5건**
- 책 **3권** 동시 작성

그 중 **웹사이트 개발은 10월부터 시작**해서, **3개월도 안 되는 기간에 풀스택 웹사이트 3개를 완성**했습니다.

10년간 개발자 없이는 불가능하다고 생각했던 것을, 3개월 만에 해냈습니다.
혼자서. AI와 함께.

이 노하우를 공유하고 싶었습니다.

### 기존 도구들과 뭐가 다른가요?

웹사이트 개발에는 여러 영역이 필요합니다:
- **진행**: 프로젝트를 단계별로 실행하는 것
- **통제**: AI에게 작업을 지시하고 결과를 받는 것
- **관리**: 전체 Task를 파악하고 추적하는 것
- **학습**: 만들면서 필요한 지식을 배우는 것
- **질문**: 막힐 때 물어볼 곳이 있는 것

기존 도구들은 이 중 **일부만** 해결합니다.

| 플랫폼 | 진행 | 통제 | 관리 | 학습 | 질문 |
|--------|:----:|:----:|:----:|:----:|:----:|
| Replit | △ | △ | ✗ | ✗ | ✗ |
| Cursor | ✗ | △ | ✗ | ✗ | ✗ |
| Notion | ✗ | ✗ | ○ | ✗ | ✗ |
| v0.dev / bolt.new | ✗ | △ | ✗ | ✗ | ✗ |
| Udemy / Codecademy | ✗ | ✗ | ✗ | ○ | ✗ |
| **SSAL Works** | **○** | **○** | **○** | **○** | **○** |

- **Replit, Cursor**: 개발자용 IDE. 비개발자가 쓰기엔 진입장벽이 높음
- **Notion**: 문서 관리는 좋지만, 코드 실행 불가
- **v0.dev, bolt.new**: AI가 코드 생성하지만, 프로젝트 관리/학습 기능 없음
- **Udemy**: 학습은 되지만, 실제 프로젝트 실행과 분리됨

SSAL Works는 이 5가지를 **하나의 화면에서** 해결합니다.

---

### 하나의 화면, 5가지 영역

SSAL Works의 화면은 다음과 같이 구성됩니다:

```
┌────────────┬───────────────────┬────────────┐
│            │                   │            │
│  📋 진행   │  🎮 통제          │  📚 학습   │
│            │  (Control Space)  │            │
│ Left       ├───────────────────┤ Right      │
│ Sidebar    │  📊 관리          │ Sidebar    │
│            │  (Grid)           │  ❓ 질문   │
│            │                   │            │
└────────────┴───────────────────┴────────────┘
```

**1. 진행 (Process)** — 좌측 사이드바
프로젝트 실행 프로세스를 단계별로 안내합니다.
P1(사업계획) → P2(기획) → P3(프로토타입) → S1~S5(개발) 순서로 진행.

**2. 통제 (Control)** — 중앙 상단 Control Space
**가장 핵심 기능입니다.**
Order Sheet를 작성해서 Claude Code에게 작업을 지시합니다.
"회원가입 API 만들어줘" → AI가 실제 코드 작성 → 결과 확인

**3. 관리 (Management)** — 중앙 하단 Grid
PROJECT SAL Grid로 전체 Task를 3차원으로 관리합니다.
- Stage(X축): 개발 단계
- Area(Y축): 작업 영역 (Frontend, Backend, Database 등)
- Level(Z축): 작업 깊이

**4. 학습 (Learning)** — 우측 상단
Books, Tips, FAQ, 가이드 문서가 있습니다.
프로젝트를 진행하면서 필요한 지식을 그때그때 학습합니다.

**5. 질문 (Questions)** — 우측 하단
- AI Q&A: ChatGPT, Gemini, Perplexity에게 기술 질문
- 써니에게 묻기: Human 멘토의 비즈니스 자문

### 핵심 철학: "머슬 메모리의 외주화"

개발을 배우려면 수많은 것들을 손가락이 기억할 때까지 익혀야 합니다.
- 에디터 단축키
- 프로그래밍 문법
- 프레임워크 사용법
- 디버깅 방법

이런 **"기술적 근육 기억(Muscle Memory)"**을 쌓는 데 보통 6개월~1년이 걸립니다.

SSAL Works는 이 부분을 **AI에게 전적으로 맡깁니다.**
당신은 **"무엇을 만들 것인가"**라는 본질적인 기획과 지시에만 집중하세요.

> *"기술은 몰라도, 내 사이트는 내가 만든다"*

### 이름의 의미

**SSAL** = **S**(Sunny) + **SAL**(3차원 좌표)

- **S (Sunny)**: 창립자이자 모든 사용자를 대변하는 페르소나
- **S (Stage)**: X축 — 개발 단계
- **A (Area)**: Y축 — 작업 영역
- **L (Level)**: Z축 — 작업 깊이

그리고 **쌀(Rice)**.
한국인에게 밥심이 필요하듯, 디지털 창작의 주식(主食)이 되겠다는 의지입니다.

### 누구를 위한 서비스인가요?

- 아이디어는 있지만 개발자를 구하기 어려운 **예비 창업자**
- 외주 비용이 부담되는 **1인 사업자**
- 사이드 프로젝트를 직접 만들어보고 싶은 **직장인**
- 포트폴리오용 웹사이트가 필요한 **취준생**
- AI 시대에 새로운 스킬을 익히고 싶은 **누구나**
