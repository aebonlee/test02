- **Verification Agent**: code-reviewer

## Part 1: 인앱 알림 UI 검증

### 1. UI 검증
- [ ] 헤더에 알림 벨 아이콘이 표시되는가
- [ ] 읽지 않은 알림 배지가 표시되는가
- [ ] 알림 벨 클릭 시 드롭다운이 토글되는가

### 2. 기능 테스트
- [ ] loadUserNotifications(): 알림 목록 로드 성공
- [ ] markNotificationRead(): 읽음 처리 성공
- [ ] markAllNotificationsRead(): 전체 읽음 처리 성공

### 3. 관리자 연동 확인
- [ ] 입금 확인 시 사용자에게 알림 생성되는가
- [ ] 빌더 계정 개설 시 알림 생성되는가

---

## Part 2: My Page 문의 관리 검증

### 1. 문의 목록 UI
- [ ] 로그인한 사용자의 문의만 표시되는가
- [ ] 문의 카드에 제목, 상태 배지, 날짜가 표시되는가
- [ ] 문의 카드 클릭 시 상세 모달이 열리는가

### 2. 상태 표시
- [ ] pending: 노란색 "대기" 배지
- [ ] in_progress: 파란색 "처리중" 배지
- [ ] answered: 초록색 "완료" 배지

### 3. 새 문의 작성
- [ ] "새 문의 작성" 버튼 클릭 시 모달 열림
- [ ] 문의 유형 선택 가능 (일반/기술/결제/구독/기타)
- [ ] 제목, 내용 입력 가능
- [ ] 등록 시 DB에 저장되고 목록에 반영

### 4. 문의 상세 보기
- [ ] 문의 제목, 유형, 상태, 작성일 표시
- [ ] 문의 내용 전문 표시
- [ ] 답변이 있으면 답변 내용 표시
- [ ] 답변이 없으면 "아직 답변이 없습니다" 표시

### 5. 에러 처리
- [ ] 비로그인 시 로그인 페이지로 리다이렉트
- [ ] 문의가 없을 때 "문의 내역이 없습니다" 표시
- [ ] API 오류 시 적절한 에러 메시지
