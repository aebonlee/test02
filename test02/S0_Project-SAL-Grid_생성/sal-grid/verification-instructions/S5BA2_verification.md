# S5BA2: 예시 프로젝트 서약서 시스템 - 검증 지침

## 검증 정보
- **Task ID**: S5BA2
- **Verification Agent**: code-reviewer

## 검증 항목

### 1. UI 검증
- [ ] 서약서 모달이 정상 표시되는가
- [ ] 서약 내용 3개항이 모두 표시되는가
- [ ] 서약자 정보 (일자, 이름, 빌더ID, 이메일)가 자동 표시되는가
- [ ] [위와 같이 서약합니다] 버튼이 작동하는가
- [ ] [다음에 하기] 버튼이 모달을 닫는가

### 2. 로그인 검증
- [ ] 비로그인 시 로그인 요청 메시지가 표시되는가
- [ ] 로그인 상태에서 서약서 모달이 표시되는가

### 3. 1회 연결 제한 검증
- [ ] 처음 클릭 시 서약서 모달이 표시되는가
- [ ] 이미 서약한 경우 "이미 연결하셨습니다" 메시지가 표시되는가
- [ ] DB에서 연결 이력을 정확히 조회하는가

### 4. DB 저장 검증
- [ ] 서약 완료 시 download_agreements 테이블에 기록이 저장되는가
- [ ] user_id, user_email, user_name, builder_account_id 필드가 정확한가

### 5. 이메일 발송 검증
- [ ] 서약 완료 시 이메일이 발송되는가
- [ ] 이메일 내용에 서약 내용과 서약자 정보가 포함되어 있는가

### 6. 구글 드라이브 연결 검증
- [ ] 서약 완료 후 구글 드라이브 링크가 새 탭에서 열리는가
- [ ] 성공 메시지가 표시되는가

## 통과 기준

- 모든 체크리스트 항목 통과
- 에러 없이 전체 흐름 완료
- DB 저장 및 이메일 발송 정상 작동
