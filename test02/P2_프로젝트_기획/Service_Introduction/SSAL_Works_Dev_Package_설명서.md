# Dev Package 설명서

> AI와 협업하여 웹 프로젝트를 완성하는 올인원 개발 패키지

---

## 1. Dev Package란?

**Dev Package**는 Claude Code(AI)와 함께 웹 프로젝트를 처음부터 끝까지 수행할 수 있도록 설계된 **통합 개발 환경 패키지**입니다.

### 핵심 정체성

| 구분 | 설명 |
|------|------|
| **무엇인가** | AI 협업 웹 개발을 위한 표준화된 프로젝트 템플릿 |
| **목적** | 비개발자도 AI와 함께 웹 서비스를 만들 수 있게 함 |
| **특징** | 프로세스 + 자동화 + AI 설정이 하나로 통합 |

### 한 문장 정의

> "다운로드 → 압축 해제 → Claude Code 실행" 3단계로 프로젝트 개발 환경이 완성되는 패키지

---

## 2. 패키지 구성 요소

### 2.1 프로젝트 디렉토리 구조 (P0~S5)

| 단계 | 폴더명 | 역할 |
|------|--------|------|
| P0 | 작업_디렉토리_구조_생성 | 프로젝트 초기 설정 |
| P1 | 사업계획 | 비즈니스 모델 정의 |
| P2 | 프로젝트_기획 | 기능 명세, UI/UX 설계 |
| P3 | 프로토타입_제작 | 화면 프로토타입 개발 |
| S0 | Project-SAL-Grid_생성 | Task 관리 체계 구축 |
| S1 | 개발_준비 | 개발 환경, DB 스키마 |
| S2 | 개발-1차 | 인증, 회원가입 |
| S3 | 개발-2차 | AI 연동, 핵심 기능 |
| S4 | 개발-3차 | 결제, 관리자 기능 |
| S5 | 개발_마무리 | 배포, QA, 안정화 |

**총 10단계**의 체계적인 프로젝트 수행 구조 제공

---

### 2.2 AI 협업 설정 (.claude/)

#### 메인 설정 파일

| 파일 | 내용 |
|------|------|
| **CLAUDE.md** | AI 7대 작업 규칙 + 5대 절대 규칙 |
| **CAUTION.md** | 주의사항 (RLS, TODO 등) |
| **settings.local.json** | 로컬 설정 |
| **mcp_servers.json** | MCP 서버 연결 설정 |

#### 그룹 1: AI 도구 확장

| 폴더 | 내용 | 용도 |
|------|------|------|
| **commands/** | 커스텀 슬래시 명령어 | `/명령어`로 반복 작업 자동화 |
| **subagents/** | 서브 에이전트 설정 | 전문 Agent 호출 (code-reviewer 등) |
| **skills/** | 스킬 설정 | 특수 기능 확장 (PDF, Playwright 등) |

#### 그룹 2: AI 작업 규칙

| 폴더 | 내용 | 용도 |
|------|------|------|
| **rules/** | 7대 작업 규칙 | 파일명, 저장위치, Grid 작성 등 |
| **methods/** | 작업 방법론 | Supabase CRUD 등 실행 방법 |
| **compliance/** | AI 준수사항 | 12대 AI 행동 규칙 |

#### 그룹 3: 작업 기록

| 폴더 | 내용 | 용도 |
|------|------|------|
| **work_logs/** | 작업 로그 | 세션별 작업 내역 기록 |

**Claude Code 실행 즉시** 모든 규칙과 설정이 자동 적용됩니다.

### 2.3 Order Sheet 템플릿 (32개)

| 단계 | 개수 | 용도 |
|------|------|------|
| P0~P3 | 19개 | 기획 단계 작업 지시 |
| S0~S5 | 12개 | 개발 단계 작업 지시 |
| 상황별 | 1개 | 완료 프로젝트 수정용 |

**Order Sheet란?**
- Claude Code에게 작업을 지시하는 표준화된 양식
- 단계별로 무엇을 해야 하는지 명확히 정의
- 사용자는 템플릿을 선택하고 내용만 채우면 됨

---

### 2.4 자동화 스크립트 (scripts/)

| 스크립트 | 기능 |
|---------|------|
| **setup-hooks.js** | Pre-commit Hook 자동 설치 (최초 1회) |
| **sync-to-root.js** | Stage→Root 자동 동기화 (커밋 시 자동) |
| **build-web-assets.js** | MD→JS 빌드 (Order Sheet 수정 시) |

**자동화 워크플로우:**
```
Stage 폴더에 코드 저장
       ↓
git commit 실행
       ↓
Pre-commit Hook 자동 실행
       ↓
배포용 Root 폴더로 자동 복사
```

| 구성 요소 | 역할 |
|----------|------|

### 2.6 배포 구조 (Vercel 즉시 배포)

```
Root/
├── api/              ← 백엔드 API (Serverless Functions)
├── pages/            ← 프론트엔드 페이지
├── assets/           ← 정적 자원 (CSS, JS, 이미지)
├── scripts/          ← 자동화 도구
├── index.html        ← 메인 페이지
└── 404.html          ← 에러 페이지
```

**Vercel 배포 준비 완료** 상태로 제공

## 4. 패키지 설치 및 사용법

### 4.1 설치 (3단계)

```bash
# 1. 패키지 다운로드 및 압축 해제
unzip SSAL_Works_Dev_Package.zip

# 2. 폴더 이름을 프로젝트명으로 변경
mv SSAL_Works_Dev_Package MyProject

# 3. Pre-commit Hook 설치 (최초 1회)
cd MyProject
node scripts/setup-hooks.js
```

### 4.2 사용 시작

```bash
# Claude Code 실행
claude

# Order Sheet 선택하여 작업 지시
"P1-1 Order Sheet를 실행해줘"
```
