# 페이스북 홍보글: 회계감사 경험이 만든 AI 개발 방법론

---

## 글 본문

**"비개발자가 어떻게 이런 개발 방법론을 만들었냐고요?"**

7개월간 AI와 함께 3개의 풀스택 웹사이트를 만들면서
드디어 완성한 SAL Grid 방법론.

체계적이고, 치밀하고, 물샐 틈 없는 개발 관리 방법론입니다.

---

**비밀은 '회계감사' 경험입니다.**

저는 공인회계사입니다.
개발 기술은 모릅니다.
하지만 회계감사는 압니다.

회계감사의 본질이 뭘까요?
**"남이 한 일을 체계적으로 검증하는 것"**

AI가 한 코드를 체계적으로 검증하는 것?
**똑같은 원리입니다.**

---

**회계감사 개념이 SAL Grid에 그대로 녹아있습니다:**

| 회계감사 | SAL Grid |
|---------|----------|
| 감사 계획서 | Order Sheet |
| 감사 절차서 | Task Instruction |
| 내부통제 테스트 | Verification Instruction |
| Working Paper | Task Results |
| Review Process | Stage Gate 검증 |
| 파트너 승인 | PO 승인 |

---

**왜 개발자들은 이런 방법론을 못 만들까요?**

개발자들은 코드 짜는 데 집중합니다.
관리, 통제, 검증 체계를 설계하지 않습니다.

저는 코드를 못 짭니다.
대신 AI가 짠 코드를 **관리하고 통제하고 검증**합니다.

**Preparer와 Reviewer를 분리하듯이,**
**Task Agent와 Verification Agent를 분리합니다.**

**Working Paper 없으면 증빙이 없듯이,**
**Task Results 없으면 완료가 아닙니다.**

**파트너 승인 없이 감사보고서 안 나가듯이,**
**PO 승인 없이 다음 단계로 안 넘어갑니다.**

---

**비개발자가 만든 개발 방법론.**
**오히려 개발자들이 놓치기 쉬운 관리/통제/검증이 탄탄합니다.**

2,500시간의 시행착오 끝에 완성했습니다.

이 방법론으로 누구나 풀스택 웹사이트를 만들 수 있습니다.
기술이 아니라 방법론이 필요했던 겁니다.

**SSAL Works**
AI와 함께 웹사이트를 만드는 곳.
