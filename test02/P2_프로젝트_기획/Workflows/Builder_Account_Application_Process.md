# 빌더 계정 개설 프로세스

> **버전:** v1.0 (초안)
> **작성일:** 2025-12-23
> **상태:** 향후 도입 예정

---

## ⚠️ 도입 시점 관련 메모

**이 프로세스는 초기에는 적용하지 않습니다.**

| 시점 | 적용 방식 | 이유 |
|------|----------|------|
| **초기** (현재) | 기존 방식 유지 (결제 → 개설) | 고객 확보, 인지도 확보 우선 |
| **성장기** (어느 정도 알려진 후) | 이 프로세스 적용 (신청 → 심사 → 결제) | 품질 관리, 브랜드 가치 |

처음부터 심사 프로세스를 도입하면 고객을 한 명도 확보하지 못할 수 있음.
서비스가 어느 정도 알려지고 나면, 그때 이 프로세스를 도입할 것.

## 1. 프로세스 개요

### 1.1 배경

**SSAL Works는 컴퍼니빌더형 액셀러레이터 프로그램의 일환으로 제공되는 서비스입니다.**

이 플랫폼은 아무나 사용할 수 있는 일반 SaaS가 아닙니다.
돈을 내면 자동으로 이용권이 부여되는 방식이 아닙니다.

웹사이트로 구현하고자 하는 **아이디어**가 있고,
그것을 실현하고자 하는 **염원**이 있으며,
끝까지 해내겠다는 **의지**가 있는 사람.

이런 **빌더**에게만 플랫폼 이용 자격이 주어집니다.

따라서, 빌더 계정 개설은 **신청 → 심사 → 승인 → 결제** 프로세스를 따릅니다.

## 2. 신청서 양식

### 2.1 심사 기준 (3가지 핵심 항목)

| # | 항목 | 질문 | 의도 |
|---|------|------|------|
| 1 | **아이디어** | 웹사이트로 구현하고자 하는 아이디어가 있습니까? | 구체적인 비전이 있는가 |
| 2 | **염원** | 이 아이디어를 실현하고 싶은 열망이 있습니까? | 진심으로 원하는가 |
| 3 | **의지** | 이것을 끝까지 해내겠다는 의지가 있습니까? | 실행력이 있는가 |

### 2.2 신청서 필드

| 필드 | 필수 | 유형 | 설명 |
|------|:----:|------|------|
| 이름 | ✅ | 텍스트 | 실명 |
| 이메일 | ✅ | 이메일 | 연락용 이메일 |
| 연락처 | ✅ | 전화번호 | 연락 가능한 번호 |
| **아이디어** | ✅ | 장문 텍스트 | "어떤 웹사이트를 만들고 싶으신가요? 아이디어를 자유롭게 설명해주세요." |
| **염원** | ✅ | 장문 텍스트 | "왜 이 아이디어를 실현하고 싶으신가요? 어떤 의미가 있나요?" |
| **의지** | ✅ | 장문 텍스트 | "이것을 해내기 위해 어떤 노력을 하실 준비가 되어 있나요?" |
| 첨부파일 | ❌ | 파일 업로드 | 사업계획서, 기획서, 참고자료 등 (선택) |
| 추가 메시지 | ❌ | 장문 텍스트 | 추가로 전달하고 싶은 내용 |

## 3. 심사 프로세스

### 3.1 심사 기준

| 기준 | 평가 내용 | 통과 조건 |
|------|----------|----------|
| **아이디어** | 구체적인가? 실현 가능한가? | 최소한의 구체성 있음 |
| **염원** | 진심이 느껴지는가? 왜 하고 싶은지 이유가 있는가? | 진정성 있음 |
| **의지** | 실행할 준비가 되어 있는가? | 노력할 자세 있음 |

### 3.2 심사 결과

| 결과 | 조건 | 후속 조치 |
|------|------|----------|
| **승인** | 3가지 기준 충족 | 결제 안내 이메일 발송 |
| **보류** | 추가 확인 필요 | 추가 질문 또는 면담 요청 |
| **거절** | 기준 미충족 | 거절 사유 안내, 재신청 가능 |

### 3.3 심사 소요 시간

- **목표:** 신청 후 **3영업일 이내** 결과 안내

## 4. 결제 프로세스

### 4.1 승인 후 결제 안내

승인 시 발송하는 이메일 내용:

```
제목: [SSAL Works] 빌더 계정 개설 승인 - 결제 안내

안녕하세요, {이름}님.

빌더 계정 개설 신청이 승인되었습니다!
아래 계좌로 빌더 계정 개설비를 입금해주시면 빌더 계정이 개설됩니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
💰 빌더 계정 개설비: ₩3,000,000
🏦 입금 계좌: [은행명] [계좌번호] (예금주: SSAL Works)
📝 입금자명: {이름}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

입금 확인 후 빌더 계정이 개설되며,
웰컴 크레딧 ₩50,000이 지급됩니다.

궁금한 점이 있으시면 언제든 문의해주세요.

감사합니다.
SSAL Works 드림
```

### 4.2 입금 확인 후 개설 완료

입금 확인 시 처리 사항:
1. 빌더 계정 ID 부여 (예: BLDR-2412-001)
2. 웰컴 크레딧 ₩50,000 지급
3. 서비스 활성화
4. 환영 이메일 발송

## 5. 상태 관리

### 5.1 신청 상태

| 상태 | 영문 | 설명 |
|------|------|------|
| 신청 완료 | `submitted` | 신청서 제출됨, 심사 대기 |
| 심사중 | `reviewing` | 관리자가 심사 중 |
| 승인 | `approved` | 심사 통과, 결제 대기 |
| 보류 | `pending` | 추가 확인 필요 |
| 거절 | `rejected` | 심사 탈락 |
| 결제 완료 | `paid` | 입금 확인됨 |
| 개설 완료 | `completed` | 빌더 계정 개설됨 |

### 5.2 상태 흐름

```
submitted → reviewing → approved → paid → completed
                     ↘ pending → (재심사)
                     ↘ rejected
```

## 6. 데이터베이스 설계

### 6.1 builder_applications 테이블

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → auth.users |
| name | TEXT | 이름 |
| email | TEXT | 이메일 |
| phone | TEXT | 연락처 |
| idea | TEXT | 아이디어 (장문) |
| aspiration | TEXT | 염원 (장문) |
| determination | TEXT | 의지 (장문) |
| attachments | JSONB | 첨부파일 정보 |
| additional_message | TEXT | 추가 메시지 |
| status | TEXT | 상태 |
| reviewer_id | UUID | 심사자 |
| review_note | TEXT | 심사 코멘트 |
| reviewed_at | TIMESTAMPTZ | 심사 완료일 |
| rejection_reason | TEXT | 거절 사유 |
| created_at | TIMESTAMPTZ | 신청일 |
| updated_at | TIMESTAMPTZ | 수정일 |

### 6.2 첨부파일 저장

- Supabase Storage 사용
- 버킷: `builder-applications`
- 경로: `{user_id}/{application_id}/{filename}`

## 7. 관리자 기능

### 7.1 신청 목록 조회

관리자 대시보드에 "빌더 계정 신청 관리" 메뉴 추가:
- 전체/심사대기/승인/거절 필터
- 신청서 상세 보기
- 첨부파일 다운로드
- 승인/보류/거절 처리

### 7.2 심사 처리

| 액션 | 필수 입력 | 후속 처리 |
|------|----------|----------|
| 승인 | (없음) | 결제 안내 이메일 자동 발송 |
| 보류 | 보류 사유 | 추가 확인 요청 이메일 발송 |
| 거절 | 거절 사유 | 거절 안내 이메일 발송 |

## 9. 기존 프로세스와의 차이점

| 항목 | 기존 (자동 개설) | 변경 (심사 후 개설) |
