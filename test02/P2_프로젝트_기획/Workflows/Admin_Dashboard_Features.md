## 개요

### 목적
SSAL Works 관리자(써니)가 플랫폼의 모든 운영 작업을 수행할 수 있는 통합 관리 시스템

### 접근 권한
- **관리자 계정 전용** (써니)
- 일반 사용자 접근 불가
- Supabase Auth 인증 기반

### 기술 스택
- **프론트엔드:** Vanilla JavaScript (ES6+)
- **백엔드:** Supabase (PostgreSQL + Realtime)
- **차트:** Chart.js 4.4.0
- **보안:** DOMPurify 3.0.6 (XSS 방지)

## 메뉴별 기능 상세

### 1. 📊 대시보드 개요 (홈)

#### 1-1. 주요 통계 카드 (4개)

**카드 1: 총 회원 수**
- 현재 값: 1,234명
- 전일 대비 증감: ↑ +15명
- 색상: Primary (보라색)

**카드 2: 이번 달 매출**
- 현재 값: ₩5,200,000
- 전월 대비 증감: ↑ +8%
- 색상: Success (초록색)

**카드 3: 크레딧 판매**
- 현재 값: 830,000 C
- 전월 대비 증감: ↓ -5%
- 색상: Warning (노란색)

**카드 4: 미처리 문의**
- 현재 값: 12건
- 긴급 알림: ⚠️ 긴급 3건
- 색상: Danger (빨간색)

#### 1-2. 최근 활동 타임라인

**실시간 업데이트 항목:**
- 🔴 긴급 문의 (예: 결제 오류) - 3분 전
- 🟢 신규 가입 (예: 홍길동) - 12분 전
- 🟢 구독 갱신 완료 - 1시간 전

**표시 정보:**
- 활동 아이콘 (상태별 색상)
- 활동 메시지
- 상대 시간 표시

#### 1-3. 긴급 처리 필요 (우선순위 순)

**클릭 가능한 작업 항목:**

| 우선순위 | 작업 내용 | 건수 | 이동 |
|---------|---------|------|------|
| 🔴 High | 미납 회원 (구독 정지 예정) | 3명 | → 구독 관리 |
| 🟡 Medium | 잔액 부족 회원 | 12명 | → 크레딧 관리 |
| 🟡 Medium | 신규 문의 (미답변) | 8건 | → 문의 관리 |

**동작:**
- 클릭 시 해당 섹션으로 자동 이동
- Hover 시 카드 강조 효과

#### 1-4. 차트 (2개)

**차트 1: 회원 가입 추이**
- 기간: 최근 7일
- 유형: Line Chart
- 라이브러리: Chart.js

**차트 2: 매출 추이**

- 유형: Bar Chart
- 라이브러리: Chart.js

#### 1-5. 최근 문의 (실시간)

**제한:** 최신 5건만 표시
**액션:** "전체 보기" 버튼 → 문의 관리 섹션 이동

### 2. 📢 공지사항 관리

#### 2-1. 통계 카드 (3개)

- **전체 공지:** 총 공지사항 개수
- **게시 중:** 현재 활성화된 공지사항
- **중요 공지:** 중요 표시된 공지사항 개수

#### 2-2. 공지사항 CRUD

**생성 (Create):**
- 버튼: "+ 새 공지 추가"
- 모달 폼 표시:
  - 중요도 선택: 🔴 중요 / 일반
  - 제목 입력 (필수, 최대 200자)
  - 내용 입력 (필수, 줄바꿈 지원)
  - 저장/취소 버튼

**조회 (Read):**
- 테이블 형식 표시
- 컬럼: 중요도 | 제목 | 게시일 | 상태 | 조회수 | 작업
- Supabase에서 실시간 로드

**수정 (Update):**
- 각 행의 "수정" 버튼
- 기존 데이터 불러오기
- 동일한 모달 폼 사용

**삭제 (Delete):**
- 각 행의 "삭제" 버튼
- 확인 모달 표시
- Supabase에서 DELETE 실행

#### 2-3. 데이터 구조

```sql
notices 테이블:
- id (UUID, PK)
- title (VARCHAR, 200)
- content (TEXT)
- important (BOOLEAN)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)
- view_count (INTEGER)
```

### 3. 📚 학습용 컨텐츠 관리

#### 3-1. 통계 카드 (3개)

- **대분류 (Depth 1):** 5개
- **중분류 (Depth 2):** 23개
- **소분류 (Depth 3):** 115개

#### 3-2. 트리 구조 관리

**계층 구조 예시:**

```
📁 계정 관리 (Depth 1)
  └── 📂 회원가입 (Depth 2)
      ├── 📄 이메일 가입 (Depth 3) 🔗 연결됨
      ├── 📄 소셜 로그인 (Depth 3) 🔗 연결됨
      └── 📄 본인인증 방법 (Depth 3) ⚠️ 미연결
  └── 📂 로그인 (Depth 2)
      ├── 📄 로그인 방법 (Depth 3) 🔗 연결됨
      └── 📄 비밀번호 찾기 (Depth 3) 🔗 연결됨

📁 Order 작성 (Depth 1)
  └── ...
```

**각 항목별 작업 버튼:**

**Depth 1 (대분류):**
- [수정] - 이름 수정
- [+중분류] - 하위 중분류 추가
- [삭제] - 전체 트리 삭제 (경고 표시)
- [↑] / [↓] - 순서 변경

**Depth 2 (중분류):**
- [수정] - 이름 수정
- [+소분류] - 하위 소분류 추가
- [삭제] - 해당 중분류 및 하위 소분류 삭제

**Depth 3 (소분류):**
- [수정] - 이름 수정
- [링크] - Google Drive URL 연결
- [삭제] - 해당 소분류만 삭제

**링크 상태 표시:**
- 🔗 연결됨 (초록색): URL 설정됨
- ⚠️ 미연결 (노란색): URL 미설정

#### 3-3. 접기/펼치기

- 각 항목 클릭 시 하위 트리 토글
- 화살표 아이콘 (▶ / ▼) 회전 애니메이션

#### 3-4. 데이터 구조

```sql
learning_contents 테이블:
- id (UUID, PK)
- depth1 (VARCHAR, 100) - 필수
- depth2 (VARCHAR, 100) - nullable
- depth3 (VARCHAR, 100) - nullable
- url (VARCHAR, 500) - nullable
- description (TEXT) - nullable
- display_order (INT)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)
```

## 주요 관리 작업 흐름

### 흐름 1: 신규 문의 답변

```
1. 대시보드 홈에서 "미처리 문의" 알림 확인
   ↓
2. "긴급 처리 필요" 섹션에서 클릭 또는
   좌측 메뉴 "💬 문의 관리" 클릭
   ↓
3. 문의 목록에서 미답변 항목 확인
   ↓
4. [답변하기] 버튼 클릭
   ↓
5. 모달에서 질문 내용 확인
   ↓
6. 답변 작성 (에디터 사용)
   ↓
7. [저장] 버튼 클릭
   ↓
8. 자동 동작:
   - 상태 → "답변완료" 변경
   - 사용자에게 이메일 알림 발송
   - 플랫폼 내 알림 생성
   ↓
9. Toast 알림: "답변이 저장되었습니다"
```

### 흐름 2: 공지사항 작성

```
1. 좌측 메뉴 "📢 공지사항 관리" 클릭
   ↓
2. [+ 새 공지 추가] 버튼 클릭
   ↓
3. 모달 폼 표시
   ↓
4. 중요도 선택: 🔴 중요 / 일반
   ↓
5. 제목 입력 (최대 200자)
   ↓
6. 내용 입력 (줄바꿈 지원)
   ↓
7. [저장] 버튼 클릭
   ↓
8. Supabase에 INSERT
   ↓
9. 통계 카드 자동 업데이트
   ↓
10. 공지사항 목록에 새 항목 추가
   ↓
11. Toast 알림: "공지사항이 등록되었습니다"
   ↓
12. 사용자 화면에 즉시 표시 (Realtime)
```

### 흐름 3: 학습 콘텐츠 추가 (3단계)

```
1. 좌측 메뉴 "📚 학습용 컨텐츠 관리" 클릭
   ↓
2. 트리 구조에서 원하는 대분류 선택 (예: "계정 관리")
   ↓
3. 대분류 항목 Hover → [+중분류] 버튼 표시
   ↓
4. [+중분류] 클릭 → 모달 폼 표시
   ↓
5. 중분류 이름 입력 (예: "프로필 관리")
   ↓
6. [저장] 클릭
   ↓
7. Supabase INSERT:
   - depth1: "계정 관리"
   - depth2: "프로필 관리"
   - depth3: NULL
   ↓
8. 트리에 새 중분류 추가
   ↓
9. 새 중분류 항목 Hover → [+소분류] 버튼 표시
   ↓
10. [+소분류] 클릭 → 모달 폼 표시
   ↓
11. 소분류 정보 입력:
    - 이름: "프로필 사진 변경"
    - Google Drive URL: https://drive.google.com/...
    - 설명: "프로필 사진 변경 방법 안내"
   ↓
12. [저장] 클릭
   ↓
13. Supabase INSERT:
    - depth1: "계정 관리"
    - depth2: "프로필 관리"
    - depth3: "프로필 사진 변경"
    - url: "https://drive.google.com/..."
    - description: "..."
   ↓
14. 트리에 새 소분류 추가 (🔗 연결됨 표시)
   ↓
15. 통계 카드 자동 업데이트 (소분류 +1)
   ↓
16. Toast 알림: "학습 콘텐츠가 추가되었습니다"
   ↓
17. 사용자 화면에 즉시 표시 (Realtime)
```

### 흐름 5: 구독 정지 회원 처리

```
1. 대시보드 홈에서 "미납 회원 3명 (구독 정지 예정)" 알림 확인 (🔴 High)
   ↓
2. 클릭하여 "구독 관리" 섹션으로 이동
   ↓
3. "월 사용료 현황" 탭에서 상태 = "미납" 필터 적용
   ↓
4. 미납 회원 목록 확인:
   - 사용자 ID
   - 이메일
   - 다음 결제일 (경과일 표시)
   - 미납 기간
   ↓
5. 각 회원별 검토:
   [상세] 버튼 클릭
   ↓
6. 상세 정보 확인:
   - 결제 내역
   - 결제 수단 (유효 여부)
   - 이전 미납 이력
   ↓
7-A. 결제 수단 문제인 경우:
   → 사용자에게 알림 발송 (결제 수단 재등록 요청)
   → 7일 유예 기간 제공

7-B. 의도적 미납인 경우:
   → [구독 정지] 버튼 클릭
   → 확인 모달 표시
   → 확인 후 처리:
     - monthly_subscriptions 테이블: status → "suspended"
     - 사용자 계정 접근 제한 (로그인 가능, 기능 사용 불가)
     - 사용자에게 구독 정지 이메일 발송
   → Toast 알림: "구독이 정지되었습니다"

7-C. 재결제 시도:
   → [결제 재시도] 버튼 클릭
   → PG사 API 호출하여 재결제 시도
   → 성공 시: 상태 → "정상" 변경
   → 실패 시: 7일 유예 후 자동 정지
```

## 통계 및 알림

### 실시간 통계 업데이트

**Supabase Realtime 구독:**
- `notices` 테이블 변경 시 → 공지사항 통계 자동 업데이트
- `learning_contents` 테이블 변경 시 → 학습 콘텐츠 통계 자동 업데이트
- `support_requests` 테이블 변경 시 → 문의 통계 자동 업데이트
- `payment_transactions` 테이블 변경 시 → 결제 통계 자동 업데이트

### Toast 알림 시스템

**알림 유형:**
- 🟢 Success: 작업 성공 (예: "저장되었습니다")
- 🔴 Error: 오류 발생 (예: "저장 실패")
- 🟡 Warning: 경고 (예: "삭제할 수 없습니다")
- 🔵 Info: 정보 (예: "처리 중...")

**표시 위치:** 우측 상단 (Fixed)
**표시 시간:** 3초 후 자동 사라짐
**애니메이션:** Slide-in from right

### 배지 알림

**좌측 메뉴 배지:**
- 💬 문의 관리: 미답변 건수 (빨간색)
- 숫자 표시, 0이면 숨김

### 긴급 알림 (대시보드 홈)

**우선순위 High (🔴):**
- 미납 회원 (구독 정지 예정)
- 시스템 오류

**우선순위 Medium (🟡):**
- 잔액 부족 회원
- 신규 문의 (미답변)
- 갱신 예정 회원
