## 1. 개요

### 1.1 목적

SSAL Works 플랫폼을 원활하게 운영하기 위해 관리자가 수행해야 하는 모든 작업을 정리합니다.

### 1.2 비즈니스 모델

| 항목 | 금액 | 결제 방식 | 비고 |
|------|------|----------|------|
| 빌더 계정 개설비 | ₩3,000,000 | 무통장 입금 | 1회 |
| 월 이용료 | ₩50,000/월 | 토스페이먼츠 | 4개월차부터 |
| 초기 무료 크레딧 | ₩50,000 | 자동 지급 | 개설비 입금 확인 시 |
| 무료 이용 기간 | 3개월 | - | 개설비 납부 후 1~3개월차 |
| 크레딧 이익률 | 30% | - | 크레딧 판매 시 마진 |

### 1.3 관리자 대시보드 접속

| 환경 | URL |
|------|-----|
| 프로덕션 | https://ssalworks.ai.kr/admin-dashboard.html |
| 로컬 | http://localhost:3000/admin-dashboard.html |

### 1.4 작업 우선순위

| 등급 | 설명 | 처리 시한 |
|------|------|----------|
| 🔴 긴급 | 즉시 처리 필요 | 30분 이내 |
| 🟡 중요 | 당일 처리 | 업무 시간 내 |
| 🟢 일반 | 여유 있게 처리 | 2-3일 이내 |

## 2. 일일 업무

### 2.1 아침 루틴 (09:00 - 09:30)

#### 대시보드 확인

**접속:** Admin Dashboard → 대시보드 개요

**확인 항목:**

| 지표 | 확인 내용 | 액션 |
|------|----------|------|
| 총 가입자 수 | 전일 대비 증감 | 급증 시 원인 파악 |
| 활성 사용자 | 개설비 납부 완료 사용자 | - |
| 오늘 매출 | 당일 크레딧 충전액 | - |
| 미처리 문의 | 긴급 문의 유무 | 긴급 건 즉시 처리 |

#### 긴급 처리 필요 항목

```
우선순위별 처리:

1. 🔴 미확인 개설비 입금 (24시간 이내 확인 필요)
2. 🔴 결제 오류 문의 (즉시 확인)
3. 🟡 일반 문의 답변 (당일 처리)
4. 🟢 크레딧 소진 임박 알림 (안내 이메일 발송)
```

### 2.2 오전 업무 (09:30 - 12:00)

#### 빌더 계정 개설비 입금 확인

**중요도:** 🔴 긴급

**절차:**

1. Admin Dashboard → 회원/결제 → 빌더 계정 개설비 관리
2. "입금 대기" 탭에서 대기 건 확인
3. 은행 계좌에서 실제 입금 확인
4. 입금 확인 또는 거부 처리

**상세 절차:**

```
[입금 확인 시]
1. "확인" 버튼 클릭
2. 사용자 정보, 금액 검토
3. "확인" 클릭
   → 자동 처리: 빌더 계정 ID 발급 (아래 표준 참조)
   → 자동 처리: service_status = active
   → 자동 처리: 웰컴 크레딧 ₩50,000 지급

[입금 거부 시]
1. "거부" 버튼 클릭
2. 거부 사유 입력 (필수)
3. "거부" 클릭
   → 사용자에게 거부 사유 안내 이메일 발송
```

**빌더 계정 ID 표준 (12자리):**

```
YY MM NNNNNN XX
│  │  │      └─ 개설비 금액 코드 (2자리)
│  │  └──────── 일련번호 (6자리, 월별 초기화)
│  └─────────── 월 (2자리)
└────────────── 연도 (2자리)
```

| 금액 | 코드 | 금액 | 코드 |
|------|------|------|------|
| 300만원 | TH | 600만원 | SI |
| 400만원 | FO | 700만원 | SE |
| 500만원 | FI | 800만원 | EI |
|  |  | 900만원 | NI |

**예시:** 2025년 12월, 첫 번째, 300만원 → `2512000001TH`

#### 문의 답변

**중요도:** 🟡 중요

**절차:**

1. Admin Dashboard → 회원/결제 → 문의 관리
2. "대기 중" 필터로 미답변 문의 확인
3. 문의 클릭 → 내용 확인
4. 답변 작성 → "답변 저장"

**문의 유형별 대응:**

| 유형 | 우선순위 | 대응 |
|------|---------|------|
| 결제 문의 | 🔴 긴급 | 즉시 확인, 결제 오류 해결 |
| 기술 지원 | 🟡 중요 | 당일 답변 |
| 계정 관련 | 🟡 중요 | 당일 답변 |
| 일반 문의 | 🟢 일반 | 2-3일 내 답변 |
| 제휴/협력 | 🟢 일반 | 검토 후 답변 |

## 3. 주간 업무

### 3.1 월요일 - 주간 시작

#### 주간 통계 확인

**접속:** Admin Dashboard → 통계

**확인 항목:**

| 지표 | 확인 내용 |
|------|----------|
| 신규 가입자 | 지난주 가입자 수 |
| 개설비 납부 | 지난주 개설비 납부 건수 |
| 크레딧 충전 | 지난주 크레딧 충전 금액 |
| 문의 처리율 | 지난주 문의 처리 현황 |

- 지난주 미처리 문의 확인
- 지난주 미확인 입금 건 확인

---

### 3.2 수요일 - 중간 점검

#### 크레딧 현황 점검

**확인 항목:**

**액션:**

- 크레딧 소진 임박 사용자에게 충전 안내 이메일 발송

## 6. 긴급 대응

### 6.1 결제 오류

#### 증상

- 사용자가 결제했는데 크레딧이 충전되지 않음
- 결제 실패 반복

#### 확인 절차

1. Supabase → credit_transactions 테이블 확인
2. 토스페이먼츠 대시보드에서 결제 내역 확인
3. Vercel 로그 확인

#### 해결 방법

```
[크레딧 미충전 시]
1. 결제 성공 여부 확인 (토스페이먼츠)
2. 확인되면 수동으로 크레딧 충전
3. 사용자에게 완료 안내

[결제 실패 시]
1. 오류 원인 파악
2. 사용자에게 다른 결제 수단 안내
```

---

### 6.2 서비스 장애

#### 증상

- 대시보드 접속 불가
- API 오류 발생
- 데이터 로드 실패

#### 확인 절차

1. Vercel 대시보드 → 배포 상태 확인
2. Supabase 대시보드 → 서비스 상태 확인
3. 브라우저 콘솔 → 오류 메시지 확인

#### 해결 방법

```
[Vercel 문제]
- 재배포 시도
- 환경 변수 확인

[Supabase 문제]
- 서비스 상태 페이지 확인
- RLS 정책 확인
- API 키 유효성 확인

[일시적 오류]
- 페이지 새로고침
- 캐시 삭제 후 재시도
```

---

## 부록: 자주 사용하는 SQL 쿼리

### 사용자 조회

```sql
-- 특정 이메일로 사용자 조회
SELECT * FROM users WHERE email = 'example@email.com';

-- 활성 사용자 목록
SELECT * FROM users WHERE service_status = 'active';

-- 관리자 목록
SELECT * FROM users WHERE role = 'admin';
```

### 크레딧 조회

```sql
-- 특정 사용자 크레딧 거래 내역
SELECT * FROM credit_transactions
WHERE user_id = 'user-uuid'
ORDER BY created_at DESC;

-- 크레딧 잔액 낮은 사용자
SELECT id, email, credit_balance
FROM users
WHERE credit_balance < 5000
ORDER BY credit_balance ASC;
```

### 개설비 조회

```sql
-- 입금 대기 건
SELECT * FROM installation_fees
WHERE status = 'pending'
ORDER BY created_at DESC;

-- 특정 기간 입금 완료 건
SELECT * FROM installation_fees
WHERE status = 'confirmed'
AND confirmed_at >= '2025-01-01'
ORDER BY confirmed_at DESC;
```

### 문의 조회

```sql
-- 미답변 문의
SELECT * FROM inquiries
WHERE status = 'pending'
ORDER BY created_at DESC;

-- 특정 유형 문의
SELECT * FROM inquiries
WHERE inquiry_type = 'payment'
ORDER BY created_at DESC;
```
