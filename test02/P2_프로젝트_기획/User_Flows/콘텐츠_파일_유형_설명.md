# 콘텐츠 파일 유형 설명

> **작성일:** 2025-12-12
> **목적:** User Flows 폴더 내 파일들의 용도와 차이점 정리

---

## 핵심 구분: 콘텐츠 3종의 차이

| 구분 | 안내문 | 팝업 | 가이드 |
|------|--------|------|--------|
| **관련 영역** | 개발 프로세스 | 각종 기능/이벤트 | 우측 사이드바 |
| **트리거** | 프로세스 단계 클릭 | 자동 (이벤트 발생) | 사용자 직접 클릭 |
| **표시 위치** | guidePopup (드래그 가능) | 워크스페이스/모달 | 우측 사이드바 트리 |
| **용도** | 작업 단계 안내 + Order Sheet 유도 | 환영/알림 메시지 | 학습/참조 콘텐츠 |

---

## 파일 유형 개요

User Flows 폴더에는 5가지 유형의 파일이 있습니다:

| 파일 유형 | 파일명 패턴 | 대상 | 트리거 |
|-----------|-------------|------|--------|
| **flow.md** | `flow.md` | 개발자 | - (참조용) |
| **ui_specs.md** | `ui_specs.md` | 개발자 | - (참조용) |
| **안내문** | `*_안내.md` | 사용자 | 개발 프로세스 클릭 |
| **팝업** | `*_메시지.md`, `환영_*.md` | 사용자 | 자동 (이벤트 발생) |
| **가이드** | `*_guide.md` | 사용자 | 우측 사이드바 클릭 |

---

## 1. flow.md - 사용자 흐름 정의서

### 용도
- 해당 User Flow의 전체 흐름을 정의
- 단계별 액션, 조건, 분기를 기술

### 대상
- **개발자** (참조용)

### 특징
- 사용자에게 직접 표시되지 않음
- 개발 시 전체 흐름 파악용
- 각 User Flow 폴더의 기준 문서

---

## 2. ui_specs.md - UI 상세 명세서

### 용도
- 프론트엔드 개발자가 UI를 구현할 때 참고하는 기술 문서
- 픽셀 사이즈, 색상 코드, 폰트, 버튼 크기 등 상세 명세

### 대상
- **개발자** (참조용)

### 포함 내용
- 페이지 레이아웃 ASCII 아트
- 색상 팔레트 (HEX 코드)
- 폰트 크기 및 굵기
- 버튼/입력 필드 크기
- 상태별 스타일 (hover, active, disabled)
- 반응형 브레이크포인트

### 예시 파일
- `1_Signup/ui_specs.md`
- `2_Project_Registration/ui_specs.md`
- `3_Subscription/ui_specs.md`
- `4_Credit_Purchase/ui_specs.md`

---

## 3. 안내문 (*_안내.md) - 개발 프로세스용

### 용도
- **개발 프로세스 단계를 클릭했을 때** 표시되는 설명 텍스트
- 해당 작업 단계에서 무엇을 해야 하는지 안내

### 대상
- **사용자** (플랫폼 사용 중)

### 표시 위치
- Dashboard 좌측 사이드바 → 진행 프로세스 클릭
- **guidePopup** (드래그 가능한 팝업)에 표시됨

### 표시 시점
- 사용자가 개발 프로세스 단계를 클릭했을 때

### 코드 연동
```javascript
// index.html의 STAGE_DATA에서 guideContent로 사용
STAGE_DATA['vision_mission'] = {
    guideContent: `<h2>사업계획 - 비전/미션</h2>...`
}
```

### 예시 파일
- `2_Project_Registration/사용법_안내.md` → 플랫폼 전체 사용법
- `2_Project_Registration/작성법_안내.md` → Order Sheet 작성법

### 특징
- 개발 프로세스 각 단계별로 1개씩 존재
- [예] / [아니오] 또는 [확인] 버튼과 함께 표시
- Order Sheet 작성을 유도

---

## 4. 팝업 (*_메시지.md, 환영_*.md) - 자동 표시

### 용도
- **특정 이벤트 발생 시 자동으로 표시**되는 메시지
- 회원가입 완료, 프로젝트 등록 완료 등 중요 순간에 표시

### 대상
- **사용자** (플랫폼 사용 중)

### 표시 위치
- **모달 팝업** (화면 중앙, 오버레이)

### 표시 시점
- 자동 (특정 이벤트 트리거)
  - 회원가입 완료 시
  - 프로젝트 등록 완료 시
  - 첫 로그인 시 등

### 예시 파일
- `1_Signup/환영_메시지.md` → 회원가입 완료 후 자동 표시

### 특징
- 사용자 액션 없이 **자동 팝업**
- 한 번만 표시 (dismissed 처리)
- 무료/유료 기능 안내, 다음 단계 안내 등

---

## 5. 가이드 (*_guide.md) - 학습/참조용

### 용도
- **플랫폼 전체 사용법**을 설명하는 학습 자료
- 우측 사이드바 "📚 학습용 콘텐츠" 영역에서 접근

### 대상
- **사용자** (플랫폼 사용 중)

### 표시 위치
- **우측 사이드바 → 학습용 콘텐츠 트리**
- 트리 구조로 카테고리별 정리 (Claude 사용법, 웹개발 지식 등)

### 표시 시점
- 사용자가 우측 사이드바에서 직접 클릭하여 열람

### 예시 파일
- `1_Signup/home_screen_guide.md` → 홈 화면 11개 영역 사용법

### 특징
- **상시 접근 가능** - 언제든 다시 볼 수 있음
- 학습용 콘텐츠와 동일한 성격
- 전체 플랫폼 기능 설명
- 검색 기능 지원 (학습용 콘텐츠 검색)

## 비교 요약 (사용자 대상 콘텐츠 3종)

| 구분 | 안내문 | 팝업 | 가이드 |
|------|--------|------|--------|
| **관련 영역** | 좌측 - 개발 프로세스 | 각종 기능/이벤트 | 우측 - 학습용 콘텐츠 |
| **트리거** | 프로세스 단계 클릭 | 자동 (이벤트 발생) | 사이드바에서 직접 클릭 |
| **UI 표시** | guidePopup (드래그 가능) | 워크스페이스 또는 모달 | 우측 사이드바 트리 |
| **버튼** | [확인]/[취소] | - | - |
| **용도** | 작업 단계 설명 + Order Sheet 유도 | 환영/알림 메시지 | 학습/참조 콘텐츠 |
| **재열람** | 단계 클릭 시 재표시 | 페이지 로드 시 | 상시 접근 가능 |
| **내용 특성** | 해당 단계 작업 안내 | 환영/시작 안내 | 지식/사용법 설명 |
