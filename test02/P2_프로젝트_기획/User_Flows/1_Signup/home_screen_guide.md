## 1. 홈 화면 전체 구조

SSAL Works 홈 화면은 **11개 영역**으로 구성되어 있으며, 각 영역은 특정 목적을 가지고 있습니다.

```
┌─────────────────────────────────────────────────────────────────┐
│  SSAL Works Dashboard                                           │
├──────────┬───────────────────────────────────┬──────────────────┤
│          │                                   │                  │
│  좌측    │        중앙 워크스페이스           │    우측          │
│ 사이드바 │                                   │   사이드바       │
│          │                                   │                  │
│  ① 📦 PROJECT    │  ④ Workspace (상단 75%)      │  ⑥ 📚 학습 콘텐츠  │
│  ② 📊 진행       │  ⑤ Project SAL Grid          │  ⑦ 🙋 FAQ         │
│     프로세스     │     (하단 25%)               │  ⑧ 🔔 Claude Code │
│  ③ 🔗 연계       │                              │     업데이트      │
│     서비스       │                              │  ⑨ 🤖 AI묻기      │
│     바로가기     │                              │  ⑩ ☀️ Sunny묻기   │
│                 │                              │  ⑪ 📢 공지사항    │
└──────────┴───────────────────────────────────┴──────────────────┘
```

**3개 부분으로 구성:**
- **좌측 사이드바**: 프로젝트 관리 및 탐색 (3개 영역)
- **중앙 워크스페이스**: 실제 작업 공간 (2개 영역)
- **우측 사이드바**: 학습 및 지원 (6개 영역)

## 2. 좌측 사이드바 (3개 영역)

### 영역 ① 📦 PROJECT

**위치:** 좌측 사이드바 최상단

**목적:** 내가 만든 모든 프로젝트를 관리합니다.

**화면 구성:**
```
┌───────────────────────┐
│ 📁 내 프로젝트        │
├───────────────────────┤
│ [+ 새 프로젝트]       │
│                       │
│ 🌐 포트폴리오 사이트  │
│    진행 중 (3/12)     │
│                       │
│ 🏢 회사 홈페이지      │
│    완료 (25/25)       │
└───────────────────────┘
```

**사용 방법:**

1. **새 프로젝트 만들기**
   - `[+ 새 프로젝트]` 버튼 클릭
   - 프로젝트 이름 입력: "나의 첫 웹사이트"
   - 프로젝트 유형 선택: `웹사이트 개발`
   - 간단한 설명 입력 (선택)
   - `생성하기` 클릭

2. **기존 프로젝트 선택**
   - 프로젝트 카드 클릭
   - 중앙 워크스페이스에 프로젝트 내용 표시됨
   - PROJECT SAL Grid 자동 로드

3. **프로젝트 관리**
   - 프로젝트 카드에서 `...` (더보기) 클릭
   - `편집`: 프로젝트 정보 수정
   - `삭제`: 프로젝트 및 모든 Task 삭제 (주의!)

**사용 시나리오:**
```
📌 시나리오: 첫 프로젝트 만들기

1. 회원가입 후 홈 화면 진입
2. 좌측 사이드바에 "프로젝트가 없습니다" 표시
3. [+ 새 프로젝트] 클릭
4. "나의 포트폴리오 사이트" 입력
5. 생성 완료 → 프로젝트 카드 생성됨
6. 이제 중앙에서 작업 시작 가능!
```

**주요 기능:**
- ✅ 프로젝트 생성/수정/삭제 (CRUD)
- ✅ 프로젝트별 진행 상황 표시 (완료/전체 Task)
- ✅ 최근 작업한 프로젝트 최상단 표시
- ✅ 프로젝트 개수 무제한 (구독자 전용)

### 영역 ② 📊 진행 프로세스

**위치:** 좌측 사이드바 중간

**목적:** 선택한 프로젝트의 개발 프로세스를 단계별로 확인합니다.

**화면 구성:**
```
┌───────────────────────┐
│ 📊 프로세스 단계      │
├───────────────────────┤
│ Phase 1: 기획         │
│   ✅ 와이어프레임     │
│   ✅ 디자인 시스템    │
│   🔵 컬러 팔레트      │
│                       │
│ Phase 2: 설계         │
│   ⚪ API 설계        │
│   ⚪ DB 스키마       │
│                       │
│ Phase 3: 개발         │
│   ⚪ ...             │
└───────────────────────┘
```

**아이콘 의미:**
- ✅ **완료**: 녹색 체크 (작업 완료)
- 🔵 **진행 중**: 파란색 점 (현재 작업 중)
- ⚪ **대기**: 회색 점 (아직 시작 안 함)
- 🔴 **보류**: 빨간색 점 (문제 발생)

**사용 방법:**

1. **프로세스 단계 확인**
   - 프로젝트 선택 시 자동 표시
   - Phase별로 그룹화되어 표시
   - 현재 어느 단계까지 왔는지 한눈에 파악

2. **Task 클릭하여 이동**
   - Task 클릭 → 중앙 워크스페이스에 상세 정보 표시
   - 해당 Task의 Order Sheet 생성 가능
   - 진행 상황 업데이트 가능

3. **진행 상황 업데이트**
   - Task 우클릭 → 상태 변경
   - `대기` → `진행 중` → `완료` → `보류`

**사용 시나리오:**
```
📌 시나리오: 다음 할 일 찾기

1. 프로젝트 선택: "포트폴리오 사이트"
2. 프로세스 영역 확인
3. Phase 1에서 ✅(완료) 2개, 🔵(진행 중) 1개 확인
4. 🔵 "컬러 팔레트" 클릭
5. 중앙 워크스페이스에 Task 상세 정보 표시
6. Order Sheet 생성하여 AI에게 작업 지시
```

**주요 기능:**
- ✅ Phase별 Task 그룹화 (1-9단계)
- ✅ Task 상태 실시간 업데이트
- ✅ 진행률 시각화 (완료/전체)
- ✅ Task 클릭 시 중앙 워크스페이스 연동

### 영역 ③ 🔗 연계 서비스 바로가기

**위치:** 좌측 사이드바 하단

**목적:** 자주 사용하는 기능에 빠르게 접근합니다.

**화면 구성:**
```
┌───────────────────────┐
│ ⚡ 바로가기            │
├───────────────────────┤
│ 🏠 대시보드           │
│ 📊 프로젝트 그리드    │
│ 💬 써니에게 묻기      │
│ 🪙 크레딧 충전        │
│ ⚙️ 설정               │
└───────────────────────┘
```

**사용 방법:**

1. **대시보드**
   - 클릭 → 홈 화면으로 이동
   - 모든 프로젝트 개요 확인

2. **프로젝트 그리드**
   - 클릭 → PROJECT SAL Grid 전체 화면 모드
   - 3D 시각화 (예정)

3. **써니에게 묻기**
   - 클릭 → 우측 사이드바의 "써니에게 묻기" 섹션으로 스크롤
   - 빠른 질문 작성 가능

4. **크레딧 충전**
   - 클릭 → 크레딧 충전 페이지로 이동
   - 현재 잔액 확인 및 충전 신청

5. **설정**
   - 클릭 → 사용자 설정 페이지
   - 프로필, 구독, 알림 설정 등

**사용 시나리오:**
```
📌 시나리오: 크레딧 부족 시 빠른 충전

1. AI 사용 중 "크레딧 부족" 알림
2. 좌측 사이드바 하단 바로가기 확인
3. 🪙 "크레딧 충전" 클릭
4. 충전 금액 선택 (₩10,000)
5. 토스페이먼츠 결제 진행
6. 완료 후 다시 작업 계속
```

**주요 기능:**
- ✅ 자주 사용하는 기능 1클릭 접근
- ✅ 현재 페이지와 관계없이 항상 표시
- ✅ 크레딧 잔액 실시간 표시
- ✅ 커스터마이징 가능 (예정)

## 3. 중앙 워크스페이스 (2개 영역)

### 영역 ④ Workspace (상단 75%)

**위치:** 중앙 워크스페이스 상단

**목적:** Order Sheet를 작성하고 AI에게 작업을 지시하며, 결과물을 확인하는 메인 작업 공간입니다.

**화면 구성:**
```
┌─────────────────────────────────────────────────────────┐
│ 🛠️ Workspace - Order Sheet 작성 및 결과 확인            │
├─────────────────────────────────────────────────────────┤
│ Task: P1F3 - 컬러 팔레트 작성                           │
│                                                         │
│ Order Sheet 내용:                                       │
│ ┌─────────────────────────────────────────────────────┐│
│ │ 브랜드 컬러(Emerald Green, Amber Gold)를 사용한    ││
│ │ 웹사이트 컬러 팔레트를 작성해주세요.               ││
│ │                                                     ││
│ │ 필요 항목:                                          ││
│ │ - Primary, Secondary, Accent 색상                  ││
│ │ - 각 색상의 명도 단계 (100-900)                    ││
│ └─────────────────────────────────────────────────────┘│
│                                                         │
│ [Inbox 불러오기] [Outbox 불러오기] [Order Sheet 발행]   │
│                                                         │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                         │
│ 작업 결과:                                              │
│ ✅ 컬러 팔레트 완성 (color_palette.css)                │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**Workspace의 역할:**
- Order Sheet 작성 및 발행
- Inbox/Outbox 관리
- AI 작업 결과 확인

**사용 방법:**

1. **Task 추가**
   - `[+ Task 추가]` 버튼 클릭
   - Task ID 입력: `P1F3` (Phase 1, Frontend, Level 3)
   - 업무명 입력: "컬러 팔레트 작성"
   - 상세 설명 입력 (선택)
   - 프롬프트 입력: AI에게 줄 지시사항
   - 담당자 선택: Sunny
   - 생성 클릭

2. **Task 카드 클릭**
   - Task 카드 클릭 → 팝업 오픈
   - 상세 정보 확인 (설명, 프롬프트, 산출물, 코멘트)
   - `[Order Sheet 생성]` 버튼 → 영역 5로 이동

3. **Task 수정/삭제**
   - Task 카드 우클릭 또는 `...` 클릭
   - `수정`: 정보 변경
   - `삭제`: Task 삭제
   - `Export JSON`: Claude Code용 JSON 파일 생성

4. **필터링**
   - Phase 필터: Phase 1만 보기
   - Area 필터: Frontend만 보기
   - 상태 필터: 진행 중인 Task만 보기

5. **드래그 앤 드롭**
   - Task 카드를 드래그하여 순서 변경 (예정)

**사용 시나리오:**
```
📌 시나리오: 새 Task 추가 및 Order Sheet 생성

1. [+ Task 추가] 클릭
2. Task ID: P1F3
3. 업무명: "컬러 팔레트 작성"
4. 프롬프트: "브랜드 컬러(Emerald Green, Amber Gold)를 사용한
   웹사이트 컬러 팔레트를 Figma로 작성해주세요."
5. 생성 클릭 → Task 카드 생성됨
6. Task 카드 클릭 → 팝업 오픈
7. [Order Sheet 생성] 클릭
8. 영역 5(작업 영역)에 Order Sheet 자동 생성
9. AI에게 전달하여 작업 시작!
```

**주요 기능:**
- ✅ Task CRUD (Create, Read, Update, Delete)
- ✅ SAL 3D 구조로 시각화
- ✅ Phase/Area/상태별 필터링
- ✅ Task 의존성 확인 (예정)
- ✅ 진행률 시각화 (X/Y개 완료)
- ✅ Export to JSON (Claude Code 연동)

**팁:**
- Task ID 규칙: `P[Phase][Area][Level]` (예: P1F3 = Phase 1, Frontend, Level 3)
- 프롬프트를 상세하게 작성할수록 AI가 더 정확하게 작업합니다
- Task를 너무 크게 만들지 말고, 작은 단위로 쪼개세요

### 영역 ⑤ Project SAL Grid (하단 25%)

**위치:** 중앙 워크스페이스 하단

**목적:** 프로젝트의 모든 Task를 SAL(Stage-Area-Level) 3D 그리드로 시각화하고 관리합니다.

**화면 구성:**
```
┌─────────────────────────────────────────────────────────┐
│ 🛠️ 작업 영역 - Order Sheet 생성                         │
├─────────────────────────────────────────────────────────┤
│ Task: P1F3 - 컬러 팔레트 작성                           │
│                                                         │
│ Order Sheet 내용:                                       │
│ ┌─────────────────────────────────────────────────────┐│
│ │ 브랜드 컬러(Emerald Green, Amber Gold)를 사용한    ││
│ │ 웹사이트 컬러 팔레트를 Figma로 작성해주세요.       ││
│ │                                                     ││
│ │ 필요 항목:                                          ││
│ │ - Primary, Secondary, Accent 색상                  ││
│ │ - 각 색상의 명도 단계 (100-900)                    ││
│ │ - 텍스트, 배경, 경계선 색상                        ││
│ │ - Success, Warning, Error 상태 색상                ││
│ └─────────────────────────────────────────────────────┘│
│                                                         │
│ 우선순위: [Medium ▼]                                    │
│                                                         │
│ [취소]  [Order Sheet 발행]                             │
└─────────────────────────────────────────────────────────┘
```

**사용 방법:**

1. **자동 생성 (Task에서)**
   - PROJECT SAL Grid에서 Task 클릭
   - `[Order Sheet 생성]` 버튼 클릭
   - Task의 프롬프트가 자동으로 채워짐

2. **수동 생성**
   - 작업 영역에서 직접 내용 입력
   - Task 없이도 Order Sheet 발행 가능

3. **내용 작성**
   - 명확하고 구체적으로 작성
   - AI가 이해하기 쉽게 단계별로 작성
   - 필요한 파일, 라이브러리, 스타일 명시

4. **우선순위 설정**
   - `Low`: 나중에 해도 됨
   - `Medium`: 보통 (기본값)
   - `High`: 빨리 해야 함
   - `Urgent`: 긴급!

5. **Order Sheet 발행**
   - `[Order Sheet 발행]` 버튼 클릭
   - JSON 파일이 `Web_ClaudeCode_Bridge/Inbox/` 폴더에 저장됨
   - Socket.io를 통해 실시간 알림 발생 (💬 "새 Order가 도착했습니다!")
   - Claude Code가 작업 시작

**Order Sheet JSON 구조:**
```json
{
  "task_id": "P1F3",
  "task_name": "컬러 팔레트 작성",
  "phase": 1,
  "area": "F",
  "description": "브랜드 컬러를 사용한 컬러 팔레트 작성",
  "prompt": "브랜드 컬러(Emerald Green, Amber Gold)를 사용한...",
  "priority": "medium",
  "expected_outputs": ["color_palette.fig"],
  "created_at": "2025-12-02T16:30:00Z"
}
```

**사용 시나리오:**
```
📌 시나리오: AI에게 작업 지시하기

1. PROJECT SAL Grid에서 P1F3 Task 클릭
2. [Order Sheet 생성] 클릭
3. 작업 영역에 Order Sheet 자동 생성됨
4. 내용 확인 및 수정:
   "브랜드 컬러(Emerald Green #10B981, Amber Gold #F59E0B)를

## 7. 문제 해결 (Troubleshooting)

### 7.1 Order Sheet가 전송 안 돼요

**증상:** [Order Sheet 발행] 클릭해도 반응 없음

**해결 방법:**
1. inbox_server.js 실행 확인
   ```bash
   node Web_ClaudeCode_Bridge/inbox_server.js
   ```
2. Socket.io 연결 상태 확인 (콘솔 로그)
3. 브라우저 새로고침 (Ctrl+F5)
4. Inbox 폴더 권한 확인

### 7.2 크레딧이 차감되지 않아요

**증상:** AI 질문 후 크레딧 잔액 그대로

**해결 방법:**
1. 페이지 새로고침 (F5)
2. Supabase Realtime 연결 확인
3. 잔액 부족 여부 확인 (₩0 이하)
4. 써니에게 문의 (우측 사이드바)

### 7.3 프로젝트 그리드가 안 보여요

**증상:** 프로젝트 선택해도 그리드 빈 화면

**해결 방법:**
1. 프로젝트에 Task가 있는지 확인
   - Task 없으면: [+ Task 추가] 클릭
2. 필터 설정 확인
   - "전체" 필터로 변경
3. 브라우저 캐시 삭제
4. 다른 브라우저로 시도

### 7.4 써니 답변이 안 와요

**증상:** 48시간 지나도 답변 없음

**해결 방법:**
1. 스팸 메일함 확인
2. 문의 상태 확인 (🕐 답변 대기?)
3. 추가 문의 작성
   - 제목: "이전 문의 답변 확인 요청"
   - 내용: 문의 날짜 및 제목 명시

### Q1: 11개 영역을 다 사용해야 하나요?

**A:** **중앙 워크스페이스 (영역 ④, ⑤)**가 핵심입니다.

- 영역 ④ (Workspace): Order Sheet 작성 및 AI에게 실제 작업 지시

| 질문 유형 | 사용 |
