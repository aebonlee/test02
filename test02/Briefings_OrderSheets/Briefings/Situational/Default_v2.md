# SSAL Works에 오신 것을 환영합니다

**SSAL Works**는 AI를 활용해서 SAL Grid 등의 체계적 개발 방법론을 적용하여 풀스택 웹사이트를 개발하는 플랫폼입니다.

- **AI란?**
  - Claude Code: Anthropic이 만든 AI 코딩 도구로, 사람 대신 코딩을 수행하며, 주력 AI입니다.
  - Gemini CLI: Google에서 개발한 AI 모델로, 비용 효율적인 분석, 코드 리뷰, CLI 환경에서의 지시 이행 등 보조적인 역할을 수행합니다.
- **SAL Grid란?**
  - SAL(Stage, Area, Level의 3차원 좌표계) 기반의 Grid를 활용하여 개발 프로젝트의 실행/관리/통제/검증을 진행하는 시스템입니다.
- **풀스택 웹사이트란?**
  - 프론트엔드(화면) + 백엔드(서버/API) + 데이터베이스를 모두 갖춘 완전한 웹사이트입니다.

<span style="font-size: 12px;">개발기술 관련 지식이 없어도, 아이디어, 염원, 의지만 있으면 실제 비즈니스에 사용할 수 있는 완전한 풀스택 웹사이트를 만들 수 있습니다.</span>

## 바이브 코딩의 한계

**바이브 코딩(Vibe Coding)**이란 AI에게 "이거 만들어줘"라고 말하고 결과물을 받는 방식입니다. 간단한 것은 되지만, 복잡한 프로젝트에서는 한계가 있습니다.

- "로그인 기능 추가해줘" → 파일 20개 생성 → 어디에 뭐가 있는지 모름
- "버튼 색깔 바꿔줘" → 다른 기능이 망가짐 → 되던 게 안 됨
- 대화 100턴 넘어감 → AI가 앞에서 한 걸 까먹음 → 같은 실수 반복
- 3일 작업한 결과물 → 꼬여서 수습 불가 → 결국 처음부터 다시...

**SSAL Works는 체계적인 개발 방법론으로 이 문제를 해결합니다.**

## 10가지 차별점

**[방법론 4가지]**

1. **10단계 진행 프로세스**: P0~P3(기획) + S0~S5(개발)의 10단계로 아이디어부터 배포까지 전 과정을 체계화
2. **SAL Grid**: 5 Stage × 11 Area × 3 Level의 3차원 좌표계로 71개(예시) Task를 22개 속성으로 관리. SAL ID는 절차적 순서, 의존성, 인접성, 병렬성 정보를 구조적으로 내재화하여 ID만으로 작업의 위치와 맥락을 즉시 파악 가능
3. **3단계 검증**: 작업 AI → 검증 AI → 빌더 최종승인 순서로 진행. 작업 담당 AI와 검증 담당 AI 분리로 품질 확보
4. **AI 규칙 체계**: 7대 Rules + 3대 Methods + 12대 Compliance + 5대 절대 규칙으로 AI 작업의 일관성과 품질 확보

**[도구 3가지]**

5. **Dev Package**: 프로젝트 등록 시 자동 다운로드되며, 폴더 구조, 규칙 파일, 설정 일체를 포함
6. **Order Sheet**: 단계별 AI 작업 지시서 템플릿으로, 필요시 내용을 추가 작성한 후 Claude Code에 전달하면 작업 시작
7. **SAL Grid Viewer**: 71개(예시) Task의 작업 상태와 검증 상태를 Stage별로 시각화하여 개발 진행 현황을 한눈에 파악

**[콘텐츠 1가지]**

8. **학습 콘텐츠**: Books 80편, 실전 Tips 66개, 외부 연동 설정 Guide 5개를 제공하며, 초보자도 따라할 수 있는 수준으로 구성

**[서비스 2가지]**

9. **AI Tutor**: SSAL Works 지식베이스(학습용 Books, 실전 Tips, 외부 연동 설정 Guide, 안내문, Order Sheet, 매뉴얼, 서비스 소개, 기술 스택, AI 준수사항, 작업 규칙, 작업 방법 등)를 참고하여 맞춤 답변
10. **1:1 코칭 3개월**: 개발뿐 아니라 창업 전반에 대해 Sunny가 3개월간 1:1 맞춤형 코칭

## 빌더 계정

**빌더(Builder)**란 단순히 웹사이트를 만드는 사람이 아니라, 웹사이트를 통해 비즈니스를 실현하는 사람입니다.

<span style="font-size: 12px;">※ 빌더 = PO(Product Owner) = 인간. 모두 같은 의미로, AI와 대비되는 사용자(당신)를 지칭합니다.</span>

빌더 계정을 개설하면 실제 개발 작업을 시작할 수 있습니다.

**빌더 계정으로 할 수 있는 것**

1) 프로젝트 등록 및 Dev Package 다운로드
2) Order Sheet로 Claude Code에게 작업 지시
3) 작업 Reports 불러오기
4) 10단계 진행 프로세스 진척도 확인
5) SAL Grid Viewer로 개발 진행 현황 확인
6) SSAL Works 예시 프로젝트 전체 소스코드 및 관련 문서 제공
7) AI Tutor 사용
8) 다른 AI에게 질문하기
9) Sunny에게 질문하기
