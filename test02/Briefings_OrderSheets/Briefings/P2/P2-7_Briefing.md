# Tech_Stack

## 단계 개요

**데이터를 저장하는 구조**를 설계합니다.

서비스에서 사용하는 데이터(회원 정보, 상품, 주문 등)를 어떻게 저장하고 관리할지 정하는 단계입니다.

## 필요성

- **데이터 정리**: 어떤 데이터가 필요한지 명확해집니다
- **관계 정의**: 데이터 간의 연결 관계를 정합니다
- **성능 확보**: 효율적인 구조로 빠른 서비스가 됩니다

## 설계 내용

### ERD (관계도)
- 엔티티: 저장할 데이터 종류 (회원, 상품 등)
- 관계: 데이터 간 연결 (1:1, 1:다수 등)
- 속성: 각 데이터의 세부 항목

### 테이블 설계
- 구조: 컬럼과 타입
- 제약: 필수값, 고유값 등
- 인덱스: 검색 속도 최적화

### 보안 설계
- 접근 권한: 누가 어떤 데이터에 접근하는가
- 데이터 보호: 민감 정보 암호화

## PO가 결정할 것

- 주요 데이터 (서비스에서 다루는 핵심 데이터)
- 데이터 관계
- 보안 수준
