# User_Flows

## 단계 개요

사용자가 서비스를 **어떻게 사용하는지** 흐름을 설계합니다.

사용자가 목표를 달성하기 위해 거쳐야 하는 단계들을 시각화하는 단계입니다.

## 필요성

- **사용자 관점**: 개발자가 아닌 사용자 입장에서 설계합니다
- **누락 방지**: 필요한 화면이나 기능을 놓치지 않습니다
- **UX 개선**: 불편한 흐름을 미리 발견합니다

## 설계 내용

### 사용자 여정
- 주요 사용자 시나리오
- 단계별 화면 흐름
- 분기점과 조건

### 화면 목록
- 필요한 화면 전체 목록
- 화면 간 연결 관계
- 화면별 주요 기능

### 예외 처리
- 오류 발생 시 흐름
- 권한 없음 처리
- 데이터 없음 처리

## PO가 결정할 것

- 주요 사용자 시나리오
- 핵심 화면 목록
- 예외 처리 방식
