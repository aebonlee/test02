# UI_UX_Mockup

## 단계 개요

**화면의 모양**을 그려봅니다.

실제 서비스처럼 보이지만 동작하지 않는 화면을 만들어서, 개발 전에 디자인을 검토하는 단계입니다.

## 필요성

- **시각화**: 말로 설명하기 어려운 것을 눈으로 볼 수 있습니다
- **피드백 수집**: 개발 전에 디자인에 대한 의견을 받을 수 있습니다
- **비용 절감**: 개발 후 수정보다 목업 수정이 훨씬 쉽습니다

## 제작 내용

### 목업 제작 범위
- 주요 화면: 핵심 기능 화면
- 반응형: PC, 모바일 버전
- 상태 변화: 로딩, 오류 등

### 검토 항목
- 레이아웃: 요소 배치가 적절한가
- 정보 구조: 내용이 잘 정리되어 있는가
- 사용자 동선: 흐름이 자연스러운가

## PO가 결정할 것

- 제작할 화면 목록
- 참고 디자인 URL
- 우선순위
