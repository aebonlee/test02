# 중요한 파일은 주기적으로 백업하기

> 이 문서는 중요한 프로젝트 파일을 주기적으로 백업하는 방법을 설명합니다.

---

## 왜 백업이 필요한가

| 상황 | 위험 |
|------|------|
| 실수로 삭제 | 작업물 손실 |
| 잘못된 수정 | 복구 불가 |
| 하드웨어 고장 | 모든 데이터 손실 |
| 랜섬웨어 | 파일 암호화 |

---

## 백업 방법

| 방법 | 장점 | 단점 |
|------|------|------|
| Git/GitHub | 버전 관리 포함 | 대용량 파일 제한 |
| 클라우드 (Drive, Dropbox) | 자동 동기화 | 동기화 충돌 |
| 외장하드/USB | 오프라인 보관 | 수동 복사 필요 |

---

## Git을 이용한 백업

```
"중요한 변경사항 커밋하고 푸시해줘"
```

---

## 백업 주기

| 작업 유형 | 권장 백업 주기 |
|----------|--------------|
| 중요 기능 완료 | 즉시 커밋 |
| 매일 작업 종료 | 일일 커밋 |
| 마일스톤 완료 | 태그 생성 |

---

## 백업 대상

| 백업 O | 백업 X |
|--------|--------|
| 소스 코드 | node_modules |
| 설정 파일 | 빌드 결과물 |
| 문서 | 임시 파일 |
| .env.example | .env (민감정보) |

---

## Claude Code로 백업 요청

```
"오늘 작업 내용 커밋해줘"
"중요한 변경사항 GitHub에 푸시해줘"
"v1.0 태그 만들어줘"
```

---

## 백업 확인

```
"마지막 커밋 언제야?"
"GitHub에 제대로 올라갔어?"
"원격 저장소 상태 확인해줘"
```

---

## 복구 방법

```
"이전 버전으로 되돌려줘"
"어제 커밋으로 복구해줘"
"특정 파일만 이전 버전으로 복구해줘"
```

---

## 백업 습관

- 퇴근 전 푸시
- 중요 기능 완료 시 태그
- 주기적으로 원격 상태 확인
