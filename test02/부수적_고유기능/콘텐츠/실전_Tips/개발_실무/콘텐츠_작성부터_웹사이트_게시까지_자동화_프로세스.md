# 콘텐츠 작성부터 웹사이트 게시까지 자동화 프로세스

> MD 파일 하나 작성하면 웹사이트에 자동 반영되는 시스템 구축하기

---

## 문제: 콘텐츠 하나 추가하는데 여러 파일을 수정해야 하는 번거로움

웹사이트에 새로운 콘텐츠를 추가할 때 흔히 겪는 문제가 있습니다. MD 파일 하나를 작성했는데, 그걸 웹사이트에 반영하려면 여러 파일을 수동으로 수정해야 하는 상황입니다.

예를 들어 새로운 학습 콘텐츠를 추가한다고 생각해봅시다. 먼저 MD 파일을 작성합니다. 그 다음 뷰어 HTML 파일을 열어서 콘텐츠 목록에 새 항목을 추가합니다. 메인 페이지의 검색 데이터에도 추가해야 합니다. Production 폴더에도 복사해야 합니다. 파일 하나 추가하는데 서너 군데를 수정해야 하니 실수하기도 쉽고 시간도 많이 걸립니다.

이런 상황에서 "MD 파일만 작성하면 나머지는 알아서 되면 좋겠다"는 생각이 자연스럽게 듭니다.

---

## 해결책: JSON 단일 소스 + 동기화 스크립트 패턴

이 문제를 해결하는 검증된 패턴이 있습니다. 콘텐츠 정보를 하나의 JSON 파일에 모아두고, Node.js 스크립트가 그 JSON을 읽어서 필요한 모든 파일을 자동으로 업데이트하는 방식입니다.

핵심 아이디어는 단순합니다. 수정해야 할 곳이 여러 군데라면, 그 정보의 원본을 한 곳에 모으고 나머지는 자동 생성하면 됩니다. 이것을 단일 소스 패턴이라고 부릅니다. 원본 데이터는 하나뿐이고, 다른 곳에 있는 데이터는 모두 그 원본에서 파생됩니다.

---

## 자동화 프로세스의 구조

이 프로세스는 세 가지 요소로 구성됩니다.

첫째, JSON 파일입니다. 콘텐츠의 제목, 파일명, 카테고리, 아이콘 등 모든 정보를 담고 있습니다. 새 콘텐츠를 추가할 때 이 파일에 한 줄만 추가하면 됩니다. 이것이 단일 소스입니다.

둘째, 동기화 스크립트입니다. Node.js로 작성된 스크립트로, JSON 파일을 읽어서 필요한 형식으로 변환하고 여러 파일에 자동으로 적용합니다. 한 번 만들어두면 계속 사용할 수 있습니다.

셋째, 타겟 파일들입니다. 뷰어 HTML, 메인 페이지, Production 파일 등 실제로 업데이트되어야 하는 파일들입니다. 스크립트가 이 파일들의 특정 영역을 찾아서 자동으로 교체합니다.

---

## 스크립트의 작동 원리

스크립트는 정규식을 사용해서 타겟 파일의 특정 패턴을 찾아 교체합니다. 예를 들어 HTML 파일 안에 콘텐츠 데이터가 JavaScript 객체 형태로 들어있다면, 스크립트가 그 부분만 찾아서 JSON 데이터 기반으로 새로 생성한 내용으로 바꿉니다.

이 방식의 장점은 타겟 파일의 나머지 부분은 건드리지 않는다는 것입니다. HTML의 구조, 스타일, 다른 스크립트는 그대로 유지되고 콘텐츠 데이터 부분만 갱신됩니다. 파일 전체를 덮어쓰는 것이 아니라 필요한 부분만 정밀하게 교체합니다.

---

## 실제 작업 흐름

자동화가 적용되면 콘텐츠 추가 작업이 매우 간단해집니다.

첫째, MD 파일을 작성합니다. 원하는 내용을 마크다운으로 작성해서 지정된 폴더에 저장합니다.

둘째, JSON 파일에 한 줄을 추가합니다. 새 콘텐츠의 제목과 파일명 정보를 넣습니다.

셋째, 동기화 스크립트를 실행합니다. 터미널에서 명령어 하나만 입력하면 됩니다.

넷째, 끝입니다. 스크립트가 필요한 모든 파일을 자동으로 업데이트합니다. Git push만 하면 웹사이트에 반영됩니다.

---

## SSALWorks 프로젝트의 적용 사례

SSALWorks에서는 이 패턴을 세 가지 콘텐츠 시스템에 적용하고 있습니다.

Tips 시스템은 tips-list.json 파일과 sync-tips.js 스크립트를 사용합니다. 새로운 팁을 추가하면 JSON에 한 줄 추가하고 스크립트를 실행합니다. 그러면 viewer.html, tips-viewer.html, index.html 세 파일이 자동으로 업데이트됩니다.

학습용 Books 시스템도 같은 구조입니다. books-list.json에 새 콘텐츠 정보를 추가하고 sync-books.js를 실행하면 세 파일이 동기화됩니다.

외부 연동 설정 Guide도 마찬가지입니다. guides-list.json과 sync-guides.js가 index.html과 service-guides.js 두 파일을 자동 관리합니다.

---

## 이 자동화의 실질적인 효과

가장 큰 효과는 실수가 줄어든다는 것입니다. 여러 파일을 수동으로 수정할 때는 한 군데 빠뜨리거나 오타를 내기 쉽습니다. 하지만 JSON 한 곳만 수정하면 스크립트가 모든 곳에 동일하게 적용하니 일관성이 보장됩니다.

두 번째는 속도입니다. 새 콘텐츠 추가가 JSON 한 줄 추가와 스크립트 실행으로 끝납니다. 이전에는 파일 여러 개 열어서 각각 수정하고 확인하는 데 몇 분이 걸렸다면, 이제는 몇 초면 됩니다.

세 번째는 위임 가능성입니다. 프로세스가 명확해지면 Claude Code에게 맡길 수 있습니다. 프로세스 문서를 보고 알아서 처리할 수 있으니 사람이 직접 할 필요가 없어집니다.

---

## 결론

콘텐츠 작성부터 웹사이트 게시까지의 과정은 자동화할 수 있습니다. JSON 단일 소스와 동기화 스크립트라는 간단한 패턴으로 구현합니다. 처음 스크립트를 만드는 데 시간이 들지만, 이후 콘텐츠를 추가할 때마다 그 시간을 절약하게 됩니다.

여러 파일을 수동으로 동기화하며 시간을 쓰고 있다면, 이 자동화 패턴을 적용해보세요. 한 번 구축해두면 콘텐츠 관리가 훨씬 수월해집니다.
