# Vanilla에서 React로 전환하는 경우

> Vanilla로 충분한 경우와 React가 필요한 경우를 비교하고, 전환 시점을 판단하는 기준을 알아봅니다.

---

## Vanilla로 충분한 경우

모든 웹사이트가 React를 필요로 하는 것은 아닙니다. 오히려 많은 웹사이트는 Vanilla만으로도 충분합니다.

정적인 콘텐츠가 대부분인 경우입니다. 회사 소개, 블로그, 문서 사이트처럼 내용이 자주 바뀌지 않고 사용자 입력이 거의 없는 사이트는 Vanilla가 더 적합합니다. 로딩도 빠르고 검색 엔진에도 잘 노출됩니다.

간단한 폼만 있는 경우도 마찬가지입니다. 문의하기, 뉴스레터 구독, 이벤트 신청처럼 단순한 입력만 받는다면 굳이 React를 쓸 필요가 없습니다. HTML form 태그와 약간의 JavaScript로 충분합니다.

페이지 수가 적고 구조가 단순한 경우도 해당됩니다. 10페이지 이하의 작은 사이트라면 React의 장점을 느끼기 어렵습니다. 설정하고 빌드하는 시간이 오히려 더 클 수 있습니다.

빠르게 수정하고 배포해야 하는 경우에도 Vanilla가 유리합니다. 빌드 과정이 없으니 파일을 수정하면 바로 반영됩니다.

---

## React가 필요한 경우

반대로 다음과 같은 상황에서는 React 전환을 고려해야 합니다.

첫째, 화면이 자주 바뀌어야 할 때입니다. 사용자가 버튼을 누르면 목록이 추가되고, 삭제 버튼을 누르면 항목이 사라지는 식으로 화면이 실시간으로 변해야 한다면 React가 유리합니다. 장바구니, 좋아요 버튼, 실시간 알림 같은 기능입니다.

둘째, 같은 UI가 여러 곳에서 반복될 때입니다. 카드, 버튼, 모달, 드롭다운 같은 것들이 여러 페이지에서 똑같이 쓰인다면 React의 컴포넌트 시스템이 효율적입니다. 한 번 만들어두면 어디서든 재사용할 수 있습니다. 수정도 한 곳만 고치면 전체에 반영됩니다.

셋째, 서버와 데이터를 주고받는 기능이 많을 때입니다. 로그인 상태 관리, 실시간 알림, 무한 스크롤, 검색 자동완성 같은 기능은 React의 상태 관리가 도움이 됩니다.

넷째, 앱처럼 동작해야 할 때입니다. 페이지 전환 시 깜빡임 없이 부드럽게 넘어가고, 뒤로 가기를 눌러도 상태가 유지되는 것처럼 앱 같은 경험이 필요하다면 React가 적합합니다.

## 전환 판단 기준

다음 질문에 예가 3개 이상이면 React 전환을 고려하세요.

사용자 입력에 따라 화면이 실시간으로 변해야 하나요?

같은 UI 요소가 5개 이상의 페이지에서 반복되나요?

로그인 상태를 여러 페이지에서 공유해야 하나요?

서버에서 데이터를 가져와서 목록을 보여줘야 하나요?

페이지 새로고침 없이 화면이 바뀌어야 하나요?

이 질문들에 대부분 아니오라면 Vanilla로 충분할 가능성이 높습니다. 괜히 복잡한 도구를 쓸 필요가 없습니다.

---

## 전환하지 않아도 되는 경우

프레임워크를 배울 시간이 없거나 팀에 React 경험자가 없는 경우에도 Vanilla가 현실적인 선택입니다. 익숙하지 않은 도구를 억지로 쓰면 오히려 생산성이 떨어집니다.

내부용 관리 도구처럼 사용자가 제한적인 경우도 마찬가지입니다. 검색엔진 최적화가 필요 없고, 로딩 속도에 민감한 사용자가 아니라면 굳이 복잡한 도구를 쓸 필요가 없습니다.

또한 보안이 중요한 경우 Vanilla 유지를 고려할 수 있습니다. 2025년 12월 React Server Components에서 치명적 취약점이 발견되었습니다. 프레임워크를 사용하면 이런 위험도 함께 가져갑니다.

---

## 전환 시 주의사항

React로 전환할 때 몇 가지 주의할 점이 있습니다.

한 번에 모든 페이지를 전환하지 마세요. 가장 복잡한 페이지부터 시작해서 점진적으로 전환하는 것이 좋습니다. 문제가 생겼을 때 원인을 파악하기 쉽습니다.

기존 CSS를 그대로 가져오세요. 스타일을 처음부터 다시 작성하면 시간이 두 배로 듭니다. 기존 CSS 파일을 import하거나 스타일 값을 복사하세요.

기존 동작을 먼저 기록하세요. 전환 전에 어떤 동작이 있었는지 메모해두고, 전환 후에도 같은 동작이 되는지 확인하세요.

## SSALWorks 방법론의 접근

SSALWorks에서는 프로토타입 완료 후 전환 여부를 판단합니다. 처음부터 React로 시작하면 화면 설계가 바뀔 때마다 코드를 많이 수정해야 합니다.

Vanilla로 화면을 확정한 뒤 필요한 페이지만 선별해서 전환합니다. 로그인, 대시보드처럼 동적인 페이지가 우선입니다. 이용약관, FAQ 같은 단순한 페이지는 Vanilla로 유지할 수 있습니다.

상황에 맞는 도구를 선택하는 것이 좋습니다. 일단 Vanilla로 시작하고 필요할 때 전환하는 전략이 안전합니다.
