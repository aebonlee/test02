# 다국어 버전 웹사이트를 효율적으로 개발하는 방법

> 언어별 별도 파일 + Claude Code 자동 번역으로 다국어 웹사이트를 쉽게 개발하는 방법을 설명합니다.

---

## 왜 다국어 지원이 필요한가

웹사이트를 운영하다 보면 해외 사용자를 위해 영어, 일본어, 중국어 등 다국어 버전이 필요해집니다. 처음에는 번역만 하면 될 것 같지만, 막상 시작하면 여러 가지 문제가 생깁니다. 한국어 페이지를 수정하면 다른 언어 페이지도 일일이 수정해야 하고, 어느 페이지가 최신인지 헷갈리기 시작합니다.

이런 혼란을 막으려면 처음부터 체계적인 구조를 잡아야 합니다.

---

## 가장 간단한 방법: 언어별 별도 파일 + 자동 번역

복잡한 번역 도구나 별도의 설정 파일을 만들 필요 없습니다. 가장 간단하고 효율적인 방법은 이렇습니다.

먼저 한국어로 index.html을 완성합니다. 평소처럼 HTML 파일에 한국어 텍스트를 그대로 넣으면 됩니다. 그 다음 Claude Code에게 이 파일을 영어로 번역해서 /en/index.html로 만들어달라고 요청합니다. 일본어가 필요하면 /ja/index.html도 같은 방식으로 만듭니다.

이렇게 하면 한국어 사용자는 ssalworks.ai.kr로, 영어 사용자는 ssalworks.ai.kr/en으로 접속합니다. 검색엔진도 각 언어 페이지를 별도로 인식하여 SEO에도 유리합니다.

---

## Claude Code에게 번역 요청하는 방법

Claude Code는 단순 번역이 아니라 문맥을 이해하고 자연스러운 번역을 합니다. 요청 방법은 간단합니다.

한국어 index.html 파일을 완성한 후 Claude Code에게 이렇게 요청하면 됩니다. "index.html을 영어로 번역해서 /en/index.html로 저장해줘." Claude Code가 파일을 읽고, 모든 한국어 텍스트를 영어로 번역한 다음, 새 파일로 저장합니다.

특정 용어를 그대로 유지하고 싶다면 함께 알려주면 됩니다. 예를 들어 "SSAL Works, 빌더 같은 고유 명사는 번역하지 말고 그대로 써줘"라고 요청하면 됩니다.

---

## 한국어 원본 수정 시 동기화 방법

한국어 버전을 수정했을 때가 핵심입니다. 전체를 다시 번역할 필요 없이 변경된 부분만 동기화하면 됩니다.

Claude Code에게 이렇게 요청합니다. "index.html이 수정되었어. 변경된 부분을 찾아서 /en/index.html과 /ja/index.html에도 반영해줘." Claude Code가 한국어 원본의 변경 사항을 감지하고, 해당 부분만 번역해서 다른 언어 파일에 적용합니다.

이 방식의 장점은 기존 번역을 유지하면서 변경된 부분만 업데이트한다는 것입니다. 전체를 다시 번역하면 이전에 다듬어놓은 표현이 달라질 수 있는데, 부분 동기화는 그런 문제를 방지합니다.

---

## 폴더 구조 예시

다국어 웹사이트의 폴더 구조는 이렇게 간단합니다.

루트 폴더에 한국어 index.html을 두고, 각 언어별 폴더를 만들어 번역된 파일을 배치합니다. 영어는 /en 폴더에, 일본어는 /ja 폴더에 넣습니다. CSS나 이미지 같은 공통 자원은 공유하고, HTML 파일만 언어별로 분리합니다.

페이지가 여러 개라면 같은 구조를 적용합니다. about.html이 있다면 /en/about.html, /ja/about.html도 함께 만듭니다.

---

## 실제 적용 시 주의사항

다국어 웹사이트를 만들 때 몇 가지 주의할 점이 있습니다.

첫째, 한 페이지에 여러 언어를 섞지 마세요. 사용자 경험이 나빠지고 검색엔진도 혼란스러워합니다.

둘째, 언어 전환 버튼은 눈에 잘 띄는 곳에 배치하세요. 보통 헤더 오른쪽 상단에 국기 아이콘이나 언어 코드로 표시합니다.

셋째, 날짜, 통화, 숫자 형식도 언어에 맞게 변환하세요. 한국은 원 표시를, 미국은 달러 표시를 사용해야 자연스럽습니다.

넷째, 이미지 안의 텍스트도 번역 대상입니다. 가능하면 이미지에 텍스트를 넣지 않는 것이 관리하기 편합니다.

---

## SSALWorks 방법론의 가치

Claude Code만으로도 번역은 가능합니다. 하지만 체계적인 구조 없이 진행하면 파일이 흩어지고, 어느 것이 최신 버전인지 헷갈리며, 일부 언어만 업데이트되는 불일치가 발생합니다.

SSALWorks 방법론은 이 문제를 해결합니다. 폴더 구조가 표준화되어 있고, 변경 이력이 체계적으로 관리됩니다. 한국어 원본을 수정하면 Claude Code에게 동기화를 요청하고, 모든 언어 버전이 일관되게 유지됩니다. 비개발자도 이 규칙만 따르면 다국어 웹사이트를 효율적으로 운영할 수 있습니다.

---

## 결론

다국어 웹사이트 개발은 생각보다 간단합니다. 한국어 HTML 파일을 만들고, Claude Code에게 번역해서 /en, /ja 폴더에 저장해달라고 요청하면 됩니다. 수정이 생기면 변경된 부분만 동기화하면 됩니다. 복잡한 번역 도구 없이, 언어별 HTML 파일만으로 다국어 웹사이트를 효율적으로 운영할 수 있습니다.
