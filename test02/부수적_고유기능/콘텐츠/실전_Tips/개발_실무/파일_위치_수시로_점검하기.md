# 파일이 Stage/Area 규칙대로 올바른 위치에 있는지 수시로 점검하기

> 이 문서는 프로젝트 내 파일들이 올바른 위치에 있는지 점검하는 방법을 설명합니다.

---

## 왜 점검이 필요한가

파일이 잘못된 위치에 생기거나, 임시 파일이 방치되거나, 중복 파일이 생기면 시간이 지날수록 "그 파일 어디 있지?"를 자주 외치게 됩니다. 수시로 점검하는 습관이 중요합니다.

---

## 자주 발생하는 문제

| 문제 | 올바른 조치 |
|------|------------|
| 루트에 임시 파일 방치 | 삭제 또는 Testing 폴더로 이동 |
| SQL 파일이 Frontend에 | Database 폴더로 이동 |
| 테스트 파일이 코드와 섞임 | Testing 폴더로 이동 |
| 백업 파일 방치 | 삭제 (Git이 버전 관리) |

---

## 점검 습관

### 작업 시작 전

```
"현재 프로젝트 폴더 상태 간단히 확인해줘"
```

### 작업 완료 후

```
"방금 생성한 파일이 올바른 위치에 있는지 확인해줘"
```

### 커밋 전

```
"git status 확인해줘"
```

### 주 1회 전체 리뷰

```
"전체 프로젝트 폴더 구조 리뷰해줘:
- 빈 폴더 확인
- 잘못된 위치의 파일 확인
- 중복 파일 확인"
```

---

## 잘못된 위치 발견 시

1. 파일의 용도 확인
2. `Project_Directory_Structure.md` 확인하여 올바른 위치 찾기
3. 파일 이동
4. 해당 파일을 참조하는 코드 수정

---

## 위치별 파일 가이드

### 루트 디렉토리

```
있어야 하는 것: .gitignore, CLAUDE.md, 설정 파일
있으면 안 되는 것: temp_*, test_*, *_backup.*, 코드 파일
```

### Production 폴더

```
있어야 하는 것: F, BA, S, BI, E Area의 코드 파일
있으면 안 되는 것: 테스트 파일, 문서 파일, 임시 파일
```

---

*더 자세한 폴더 구조는 `P0_작업_디렉토리_구조_생성/Project_Directory_Structure.md`에서 확인할 수 있습니다.*
