# 코드 파일이 올바른 위치에 저장되었는지 확인하기

> 이 문서는 Claude Code가 생성한 파일이 올바른 위치에 저장되었는지 확인하는 방법을 설명합니다.

---

## 왜 위치 확인이 중요한가

| 문제 | 결과 |
|------|------|
| 잘못된 위치 | import 실패, 빌드 오류 |
| 중복 파일 | 어떤 게 최신인지 혼란 |
| 규칙 위반 | 프로젝트 구조 붕괴 |

---

## 저장 워크플로우 (Pre-commit Hook 자동화)

```
1. Stage 폴더에 저장 (원본)
      ↓
2. git commit 실행
      ↓
3. Pre-commit Hook 자동 실행 (scripts/sync-to-root.js)
      ↓
4. 루트 폴더로 자동 복사 (배포용)
```

**핵심:** Stage가 원본, 루트는 Pre-commit Hook으로 자동 복사

---

## Stage → Root 자동 복사 대상 (5개 Area)

| Area | Stage 폴더 | Root 폴더 (자동 복사) |
|------|-----------|----------------------|
| Frontend | S?_*/Frontend/ | pages/ |
| Backend APIs | S?_*/Backend_APIs/ | api/Backend_APIs/ |
| Security | S?_*/Security/ | api/Security/ |
| Backend Infra | S?_*/Backend_Infra/ | api/Backend_Infra/ |
| External | S?_*/External/ | api/External/ |

---

## 저장 후 확인 방법

```
"방금 생성한 파일 위치 알려줘"
"Stage 폴더에 저장했어?"
"파일 경로 전체로 보여줘"
```

---

## 확인 명령어

```
"git status 보여줘"  → 변경된 파일 경로 확인
"ls pages/auth/"  → 자동 복사된 파일 확인
```

---

## 흔한 실수

| 실수 | 올바른 방법 |
|------|------------|
| 루트에 직접 저장 | Stage 폴더에 저장 (자동 복사됨) |
| 수동으로 복사 | Pre-commit Hook이 자동 처리 |
| 오타 폴더 생성 | 기존 폴더 확인 후 저장 |

---

## 저장 위치 요청

```
"login.html을 S2_개발-1차/Frontend/pages/auth/에 저장해줘"
"S3_개발-2차/Backend_APIs/에 저장해줘"
```

---

## 위치 수정 요청

```
"방금 만든 파일이 잘못된 위치야. 올바른 위치로 이동해줘"
"Stage 폴더로 이동해줘"
```

---

## 확인 습관

매 작업 후:
1. Stage 폴더에 저장되었는지 확인
2. git commit 시 Pre-commit Hook 실행 확인
3. 루트 폴더에 자동 복사되었는지 확인
