# 폴더는 임의로 만들지 말고 기존 구조를 따르기

> 이 문서는 프로젝트 폴더 구조를 유지하고 임의로 폴더를 생성하지 않는 규칙을 설명합니다.

---

## 왜 임의 생성을 금지하는가

| 문제 | 결과 |
|------|------|
| 파일 분산 | 찾기 어려움 |
| 중복 폴더 | 혼란 발생 |
| 규칙 위반 | 일관성 깨짐 |
| 추적 불가 | 관리 어려움 |

---

## 폴더 생성 전 확인

1. 기존에 적합한 폴더가 있는지 확인
2. 없다면 사용자에게 승인 요청
3. 승인 후에만 생성

---

## Claude Code에게 지시

```
"새 폴더 만들기 전에 반드시 물어봐"
"기존 폴더 구조 확인하고 적합한 위치에 저장해"
```

---

## 프로젝트 폴더 구조 예시

```
프로젝트/
├── Production/          ← 배포 파일
│   ├── Frontend/
│   └── api/
├── S1_개발_준비/        ← Stage별 작업
├── S2_개발-1차/
├── .claude/             ← Claude Code 설정
└── Human_ClaudeCode_Bridge/  ← 소통 폴더
```

---

## 적합한 폴더 찾기

| 파일 유형 | 저장 위치 |
|----------|----------|
| HTML/CSS/JS | Frontend/ |
| API 코드 | api/ 또는 Backend_APIs/ |
| SQL | Database/ |
| 문서 | Documentation/ |
| 테스트 | Testing/ |

---

## 승인 요청 양식

```
"폴더 생성 승인 요청:
- 생성할 폴더: [경로]
- 생성 이유: [이유]
- 대안 검토: [기존 폴더 중 사용 가능한 것 없음]

승인하시겠습니까?"
```

---

## 흔한 실수

| 실수 | 올바른 방법 |
|------|------------|
| 루트에 파일 생성 | 적절한 하위 폴더에 |
| 임시 폴더 남겨둠 | 작업 후 정리 |
| 비슷한 이름 폴더 | 기존 폴더 재사용 |

---

## 정리 방법

```
"루트에 이상한 파일이나 폴더 있으면 알려줘"
"빈 폴더 정리해줘"
```
