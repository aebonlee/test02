# 규칙은 코드에 직접 구현하기

> AI가 규칙을 확실히 지키게 하려면 어디에 적어야 할까요?

---

## 문제 상황

Claude Code에게 "회원 ID는 8글자로 만들어줘"라고 했는데,
다음 세션에서 6글자로 만들었습니다. 왜 그럴까요?

---

## 비유로 이해하기

### 메모지에 적어둔 규칙 (문서/주석)

```
🗒️ 냉장고에 붙인 메모: "우유는 항상 2개씩 사오기"
```

- 메모를 **보면** → 규칙을 지킴
- 메모를 **안 보면** → 규칙을 모름

### 자동화된 시스템 (코드 구현)

```
🤖 장보기 앱 설정: "우유 수량: 최소 2개 (자동 적용)"
```

- 앱이 **알아서** 2개로 설정해줌
- 사람이 신경 안 써도 **확실히** 작동

---

## AI에게 적용하면

| 방법 | 비유 | 확실성 |
|------|------|--------|
| 문서에 규칙 작성 | 메모지 붙이기 | ❌ 안 볼 수 있음 |
| 주석에 규칙 작성 | 코드 옆에 메모 | ❌ 안 읽을 수 있음 |
| **코드로 구현** | 자동화 시스템 | ✅ 무조건 작동 |

---

## 왜 이런 차이가 생길까?

Claude Code는 매 세션마다 새로 시작합니다.

**문서에만 적은 경우:**
```
세션 1: "규칙 문서 읽음" → 8글자 ✅
세션 2: "규칙 문서 안 읽음" → 6글자 ❌
```

**코드로 구현된 경우:**
```
세션 1: 함수 호출 → 8글자 ✅
세션 2: 함수 호출 → 8글자 ✅
```

코드는 읽든 안 읽든 실행되면 똑같이 작동합니다.

---

## 그러면 주석은 쓸모없나요?

아니요! 주석은 **설명용**으로 유용합니다.

- **주석** = "이 함수가 뭘 하는지" 알려줌
- **코드** = "실제로 뭘 하는지" 결정함

주석은 설명, 코드는 강제입니다.

---

## Claude Code에게 요청하는 방법

### 좋은 요청
```
"회원 ID 생성 함수 만들어줘.
 함수가 무조건 8글자를 반환하도록 코드를 짜줘."
```
→ 규칙이 코드에 녹아들어감

### 아쉬운 요청
```
"회원 ID는 8글자로 해줘."
```
→ 지금은 지키지만, 나중에 까먹을 수 있음

---

## 핵심 정리

| 질문 | 답변 |
|------|------|
| 규칙을 확실히 지키게 하려면? | 코드 자체에 넣어달라고 하세요 |
| 주석이나 문서는? | 설명용. 강제력 없음 |
| 왜 코드가 확실한가요? | 실행되면 무조건 작동하니까요 |
