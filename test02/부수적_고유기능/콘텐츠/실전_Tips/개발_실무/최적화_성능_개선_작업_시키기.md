# 코드 최적화나 성능 개선 작업 요청하기

> 이 문서는 Claude Code에게 코드 최적화와 성능 개선을 요청하는 방법을 설명합니다.

---

## 최적화 요청 유형

| 유형 | 예시 |
|------|------|
| 속도 개선 | 로딩 시간 단축 |
| 메모리 최적화 | 불필요한 객체 제거 |
| 번들 크기 | 코드 분할, 트리 쉐이킹 |
| 쿼리 최적화 | N+1 문제 해결, 인덱스 추가 |

---

## 요청 방법

```
"이 코드 성능 개선해줘"
"로딩 속도가 느린데 최적화해줘"
"번들 크기 줄여줘"
```

---

## 구체적인 요청

```
"이 컴포넌트가 매번 리렌더링돼. useMemo나 useCallback으로 최적화해줘"
"이 쿼리가 느린데 인덱스 추가하거나 쿼리 개선해줘"
"이미지 로딩이 느려. lazy loading 적용해줘"
```

---

## 최적화 전 측정

```
"현재 성능 측정해줘"
"어디서 병목이 생기는지 분석해줘"
```

---

## 프론트엔드 최적화

| 기법 | 설명 |
|------|------|
| 코드 분할 | 필요한 코드만 로드 |
| 이미지 최적화 | 압축, lazy loading |
| 캐싱 | 브라우저/서버 캐시 활용 |
| 번들 분석 | 불필요한 패키지 제거 |

---

## 백엔드 최적화

| 기법 | 설명 |
|------|------|
| 쿼리 최적화 | 인덱스, N+1 해결 |
| 캐싱 | Redis, 메모리 캐시 |
| 병렬 처리 | 비동기 요청 |
| 커넥션 풀 | DB 연결 재사용 |

---

## 최적화 우선순위

1. 가장 느린 부분부터
2. 자주 사용되는 기능 우선
3. 사용자 체감이 큰 부분

---

## 최적화 후 검증

```
"최적화 후 성능 다시 측정해줘"
"기존 기능이 정상 작동하는지 테스트해줘"
```

---

## 주의사항

- 최적화 전에 측정
- 너무 이른 최적화 피하기
- 가독성 희생 최소화
