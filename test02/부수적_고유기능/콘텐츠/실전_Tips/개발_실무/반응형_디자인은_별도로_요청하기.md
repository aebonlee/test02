# 데스크톱 버전 완성 후 모바일 반응형 디자인은 별도 작업으로 요청하기

> 이 문서는 반응형 디자인 작업을 효과적으로 요청하는 방법을 설명합니다.

---

## 반응형 디자인이란

하나의 웹페이지가 PC, 태블릿, 모바일 등 다양한 화면 크기에 맞게 자동으로 레이아웃을 조정하는 것입니다. PC에서는 넓게, 모바일에서는 세로로 길게 보이도록 자동 변환됩니다.

예를 들어, PC에서 3개의 상품이 가로로 나란히 보이던 것이 모바일에서는 세로로 하나씩 쌓여서 보이게 됩니다.

---

## Claude Code가 기본적으로 해주는 것

별도로 요청하지 않아도 Claude Code는 기본적인 반응형을 적용합니다:
- 화면이 작아지면 레이아웃이 자동으로 조정됨
- 가로 스크롤이 생기지 않도록 처리
- 텍스트와 이미지가 화면에 맞게 축소됨

하지만 이것은 "최소한의 대응"입니다. 레이아웃이 깨지지 않도록 하는 수준이지, 모바일에서 사용하기 편하게 최적화된 것은 아닙니다.

---

## 기본 반응형 vs 모바일 최적화

| 구분 | 기본 반응형 (자동 적용) | 모바일 최적화 (별도 요청 필요) |
|------|------------------------|------------------------------|
| 메뉴 | PC 메뉴가 그대로 작아짐 | 햄버거 메뉴로 변환 |
| 버튼 | 작은 버튼 그대로 | 손가락으로 누르기 쉬운 큰 버튼 |
| 레이아웃 | 단순히 축소됨 | 모바일에 맞게 재배치 |
| 입력창 | PC용 그대로 | 모바일 키보드에 최적화 |
| 이미지 | 그대로 축소 | 모바일용 크기로 최적화 |
| 여백 | PC 여백 유지 | 모바일에 맞는 여백으로 조정 |

기본 반응형은 "보이기는 하는" 수준이고, 모바일 최적화는 "사용하기 편한" 수준입니다.

---

## 모바일 최적화를 받으려면

**핵심 요청 문구:**

```
"PC 버전을 해치지 않는 범위 내에서 모바일 최적화해줘"
```

이렇게 명시하면 Claude Code가 다음을 해줍니다:
- PC에서는 기존 디자인 그대로 유지
- 모바일에서만 햄버거 메뉴 적용
- 모바일에서만 버튼 크기 확대
- 모바일에서만 레이아웃 변경 (예: 3열 → 1열)
- 터치하기 편한 간격 확보
- 모바일에서 불필요한 요소 숨기기

**"PC 버전을 해치지 않는 범위 내에서"** 라는 말이 중요합니다. 이 말이 없으면 PC 디자인까지 함께 변경될 수 있습니다.

---

## 요청 예시

```
"상품 목록 페이지를 PC 버전을 해치지 않는 범위 내에서 모바일 최적화해줘:
- 데스크탑: 현재 4열 그리드 유지
- 모바일: 1열 리스트로 변경
- 네비게이션은 모바일에서 햄버거 메뉴로
- 검색창은 모바일에서 아이콘만 표시하고 클릭하면 확장되게"
```

---

## 왜 별도로 요청해야 하는가

"로그인 페이지 만들어줘"라고만 요청하면 Claude Code는 데스크톱 화면에 맞춰 제작합니다. 기본 반응형은 적용되지만, 모바일에서 사용하기 편하도록 최적화된 것은 아닙니다.

요즘은 웹사이트 방문자의 절반 이상이 모바일로 접속합니다. 모바일에서 사용하기 불편하면 사용자가 이탈할 수 있습니다. 그래서 모바일 최적화는 중요합니다.

---

## 작업 순서 권장

1. 먼저 PC 버전을 완성한다
2. PC 버전이 만족스러우면 모바일 최적화를 별도로 요청한다
3. "PC 버전을 해치지 않는 범위 내에서"라는 문구를 꼭 포함한다
4. 완료 후 실제 모바일 기기나 브라우저 개발자 도구로 확인한다

이렇게 하면 PC 디자인은 그대로 유지하면서 모바일에서만 최적화가 적용됩니다.

---

## 모바일 확인 방법

브라우저에서 F12를 누르면 개발자 도구가 열립니다. 상단의 기기 아이콘(또는 Ctrl+Shift+M)을 클릭하면 모바일 화면으로 볼 수 있습니다. 다양한 기기 크기를 선택해서 테스트해보세요.

---

*반응형 디자인은 "기능"이 아니라 "디자인" 작업입니다. 별도로 요청하세요.*
