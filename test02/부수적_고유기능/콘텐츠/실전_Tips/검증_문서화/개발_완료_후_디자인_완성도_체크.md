# 개발 완료 후 서비스 개시 전 디자인과 UI/UX 완성도 최종 점검하기

> 이 문서는 개발 완료 후 서비스 개시 전까지 디자인과 완성도를 체크하는 방법을 설명합니다.

---

## 왜 체크가 필요한가

개발 완료 후 버그 수정, 기능 보완 작업을 하다 보면 예상치 못한 문제가 발생합니다. CSS가 깨지고, 레이아웃이 틀어지고, 반응형이 망가지는 일이 흔합니다. 수정 하나가 연쇄적으로 다른 곳에 영향을 미치기 때문에 수시로 전체를 점검해야 합니다.

---

## 두 가지 체크

### 디자인 체크

| 확인 항목 | 세부 내용 |
|----------|----------|
| 레이아웃 | 요소 배치, 정렬, 간격이 올바른가? |
| 반응형 | 모바일/태블릿/데스크톱에서 깨지지 않는가? |
| 타이포그래피 | 폰트 크기, 굵기, 줄간격이 일관적인가? |
| 색상 | 브랜드 컬러가 제대로 적용됐는가? |
| 이미지 | 이미지가 찌그러지거나 깨지지 않는가? |

### 완성도 체크

| 확인 항목 | 세부 내용 |
|----------|----------|
| 기능 누락 | 기획된 기능이 모두 구현됐는가? |
| 화면 누락 | 빠진 페이지가 없는가? |
| 에러 처리 | 에러 메시지가 사용자 친화적인가? |
| 로딩 상태 | 로딩 중 표시가 있는가? |
| 빈 상태 | 데이터가 없을 때 안내가 있는가? |

---

## 체크 프로세스

```
[1단계] 서브에이전트 자동 체크
    ↓
[2단계] Main Agent 검토 및 보고
    ↓
[3단계] PO 직접 확인 및 최종 승인
```

AI 체크만으로는 한계가 있습니다. PO가 실제 브라우저와 모바일 기기에서 직접 확인해야 합니다.

---

## Claude Code에게 요청하기

### 디자인 체크 요청

```
"전체 디자인 체크해줘:
- 모든 페이지의 CSS 깨짐 확인
- 반응형 테스트 (모바일/태블릿/데스크톱)
- 레이아웃 틀어진 곳 찾기"
```

### 완성도 체크 요청

```
"원래 계획(User Flow)과 비교해서:
- 빠진 기능 있는지
- 빠진 화면 있는지
- 미구현된 부분 있는지
체크해줘"
```

### 종합 체크 요청

```
"서비스 개시 전 최종 점검해줘:
1. 디자인 체크 (CSS, 반응형, 레이아웃)
2. 완성도 체크 (기능, 화면, 에러 처리)
3. 발견된 문제 우선순위 정리"
```

---

## 주의사항

**체크는 수시로**: 수정할 때마다 관련 영역을 체크합니다.

**실제 기기 테스트**: 브라우저 개발자 도구의 모바일 뷰는 실제 모바일과 다릅니다.

**다양한 브라우저**: Chrome에서 잘 보인다고 Safari, Firefox에서도 잘 보이는 것은 아닙니다.

---

*디자인/완성도 체크는 서비스 품질의 최종 관문입니다.*
