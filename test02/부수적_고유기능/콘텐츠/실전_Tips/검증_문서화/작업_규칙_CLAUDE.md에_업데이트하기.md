# 새로운 규칙이 생기면 바로 CLAUDE.md에 추가하여 AI가 따르게 하기

> 이 문서는 새로운 규칙이 생겼을 때 CLAUDE.md에 추가하는 방법을 설명합니다.

---

## 왜 CLAUDE.md에 추가해야 하는가

Claude Code는 세션이 시작될 때 CLAUDE.md를 읽습니다. 여기에 적힌 규칙만 일관되게 따릅니다. 규칙을 말로만 전달하면 다음 세션에서 잊어버립니다.

```
규칙 안 적음 (잘못된 방식):
"Production에 직접 저장하지 마" → 다음 세션 → 규칙 모름 → 또 실수

CLAUDE.md에 추가 (올바른 방식):
규칙 추가 → 다음 세션 → CLAUDE.md 읽음 → 규칙 따름
```

---

## 업데이트할 규칙 종류

| 종류 | 예시 |
|------|------|
| 금지 규칙 | 새 폴더 생성 전 승인 필수 |
| 저장 위치 | Frontend → S?_*/Frontend/ + Production/Frontend/ |
| 작업 절차 | 완료 시 work_logs 업데이트 필수 |
| 명명 규칙 | 파일명은 kebab-case 사용 |
| 검증 규칙 | 테스트 없이 완료 보고 금지 |

---

## CLAUDE.md 구조

```
CLAUDE.md
├── 절대 불변 규칙 (ABSOLUTE RULES) ← 가장 중요
├── 상세 작업 규칙 참조 (.claude/rules/)
├── AI 12대 준수사항
├── 프로젝트 정보
└── 기타 참고사항
```

중요도에 따라 적절한 위치에 추가합니다.

---

## Claude Code에게 요청하기

### 새 규칙 추가

```
"CLAUDE.md에 '테스트 없이 완료 보고 금지' 규칙 추가해줘"
```

### 절대 규칙 추가

```
"CLAUDE.md 절대 규칙 섹션에 이 내용 추가해줘:
- 외부 API 키가 필요하면 작업 전에 PO에게 요청"
```

### 상세 규칙 추가

```
".claude/rules/에 새 규칙 파일 만들어줘:
07_testing.md - 테스트 관련 규칙"
```

---

## 규칙 작성 팁

### 명확하고 구체적으로

```
모호한 표현: "파일은 적절한 곳에 저장한다"
구체적 표현: "Frontend 파일은 S?_*/Frontend/ + Production/Frontend/에 이중 저장한다"
```

### 예시 포함

```
좋은 예:
규칙: 파일명은 kebab-case 사용
예시: google-login.js (O), googleLogin.js (X)
```

### 중요한 규칙 강조

```
절대 금지: 새 폴더 생성 전 승인 없이 생성
주의: 민감정보 로그 출력 금지
```

---

## 규칙 우선순위

```
1순위: 절대 불변 규칙 (ABSOLUTE RULES)
2순위: AI 12대 준수사항
3순위: .claude/rules/ 상세 규칙
4순위: Order Sheet 지시사항
```

상위 규칙과 충돌하면 상위 규칙을 따릅니다.
