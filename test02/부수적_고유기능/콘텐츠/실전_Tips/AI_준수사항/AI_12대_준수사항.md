# Claude Code AI가 지켜야 할 12대 준수사항

> 이 문서는 Claude Code가 작업 시 반드시 지켜야 할 핵심 규칙을 설명합니다.

---

## 12대 준수사항 요약

| # | 규칙 | 핵심 |
|---|------|------|
| 1 | 폴더 임의 생성 금지 | 승인 없이 폴더 만들지 않음 |
| 2 | 파일 위치 규칙 준수 | 이중 저장 필수 |
| 3 | Task ID 표기 | 파일 주석에 Task ID |
| 4 | 상태 업데이트 | Grid 상태 실시간 반영 |
| 5 | 검증 필수 | 작업 후 검증 에이전트 투입 |
| 6 | 문서화 필수 | work_logs, Reports 기록 |
| 7 | 민감정보 보호 | API 키 노출 금지 |
| 8 | 코드 품질 | 보안 취약점 방지 |
| 9 | 테스트 | 작업 후 테스트 실행 |
| 10 | 커밋 규칙 | 명확한 커밋 메시지 |
| 11 | 보고 | 완료 시 상세 보고 |
| 12 | 규칙 파일 참조 | .claude/rules/ 확인 |

---

## 1. 폴더 임의 생성 금지

```
✗ 승인 없이 새 폴더 생성
✓ 기존 폴더 확인 → 없으면 승인 요청
```

---

## 2. 파일 위치 규칙

```
코드 파일 → Stage 폴더 + Production 폴더 (이중 저장)
문서 파일 → 해당 Documentation 폴더
```

---

## 3. Task ID 표기

```javascript
/**
 * @task S2F1
 */
```

---

## 4. 상태 업데이트

```
Pending → In Progress → Executed → Completed
(Grid에 실시간 반영)
```

---

## 5. 검증 필수

```
작업 완료 → 검증 에이전트 투입 → 결과 기록
```

---

## 6. 문서화 필수

```
work_logs/current.md → 작업 내역
Reports/ → 결과 JSON
```

---

## 7. 민감정보 보호

```
✗ API 키 하드코딩
✓ 환경변수 사용
```

---

## 8-12. 기타 규칙

- 보안 취약점 방지 (XSS, SQL Injection 등)
- 작업 후 테스트 실행
- 커밋 메시지에 Task ID 포함
- 완료 시 상세 보고
- 작업 전 규칙 파일 확인
