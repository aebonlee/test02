# Task 계획 전에 User Flow를 먼저 정의하기

> 이 문서는 Task 계획을 세우기 전에 User Flow를 먼저 정의하는 방법을 설명합니다.

---

## 왜 User Flow가 먼저인가

| 순서 | 이유 |
|------|------|
| User Flow 먼저 | 사용자 관점에서 필요한 기능 파악 |
| Task 나중에 | Flow 기반으로 구현 작업 분할 |

---

## User Flow란

사용자가 목표를 달성하기 위해 거치는 단계들입니다.

```
회원가입 Flow:
1. 이메일 입력 → 2. 비밀번호 입력 → 3. 인증 코드 확인 → 4. 완료
```

---

## User Flow 정의 요청

```
"이 기능의 User Flow 정의해줘"
"사용자가 어떤 단계를 거치는지 정리해줘"
```

---

## User Flow 예시: 로그인

```
1. 로그인 페이지 접속
2. 이메일/비밀번호 입력 또는 Google 로그인
3. 인증 처리
4. 성공 시 대시보드 이동
5. 실패 시 에러 메시지 표시
```

---

## Flow → Task 변환

| User Flow 단계 | Task |
|---------------|------|
| 로그인 페이지 | S2F1: 로그인 UI |
| Google 로그인 | S2S1: OAuth 처리 |
| 인증 처리 | S2BA1: 로그인 API |
| 에러 표시 | S2F2: 에러 핸들링 |

---

## Flow 검토

| 확인 항목 | 질문 |
|----------|------|
| 완결성 | 모든 경로가 목표에 도달하는가? |
| 에러 처리 | 실패 케이스가 정의되어 있는가? |
| 사용성 | 단계가 너무 많지 않은가? |
