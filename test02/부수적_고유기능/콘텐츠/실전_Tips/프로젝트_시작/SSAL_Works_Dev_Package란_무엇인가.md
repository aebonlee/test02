# Dev Package란 무엇인가

> 풀스택 웹사이트 개발 프로젝트를 Claude Code와 함께 처음부터 끝까지 체계적으로 수행할 수 있도록 설계된 통합 개발 환경

---

## Package의 정체성

Dev Package는 풀스택 웹사이트 개발 프로젝트를 Claude Code와 함께 처음부터 끝까지 체계적으로 수행할 수 있도록 설계된 통합 개발 환경입니다. 다운로드하고 압축을 풀면, 프로젝트에 필요한 모든 것이 준비되어 있습니다.

일반적으로 새 프로젝트를 시작하려면 폴더 구조를 설계하고, 규칙을 정하고, 자동화 스크립트를 만들어야 합니다. 이 과정만 해도 몇 주가 걸릴 수 있습니다. 경험이 없다면 어디서부터 시작해야 할지 막막합니다.

Dev Package는 이 모든 것을 미리 준비해 두었습니다. 수백 시간의 시행착오를 거쳐 검증된 구조입니다. 다운로드하면 바로 프로젝트를 시작할 수 있습니다. 초기 설정에 시간을 낭비하지 않아도 됩니다.

---

## 무엇이 들어있는가

Package에는 크게 다섯 가지가 포함되어 있습니다.

첫째, 프로젝트 디렉토리 구조입니다. P0부터 S5까지 10단계로 나뉜 폴더 구조가 준비되어 있습니다. P 단계는 기획 과정이고, S 단계는 개발 과정입니다. 어떤 파일을 어디에 저장해야 하는지 고민할 필요가 없습니다. 정해진 위치에 저장하면 됩니다. 파일이 흩어지는 문제가 원천적으로 차단됩니다.

둘째, AI 협업 설정입니다. Claude Code가 프로젝트 규칙을 자동으로 인식하도록 CLAUDE.md 파일이 준비되어 있습니다. 7대 작업 규칙이 명시되어 있습니다. 서브에이전트 설정, 스킬 설정, 슬래시 명령어까지 모두 포함되어 있습니다. AI가 규칙을 어기지 않도록 Compliance 문서도 있습니다.

셋째, Order Sheet 템플릿입니다. Claude Code에게 작업을 지시할 때 사용하는 표준 양식이 32개 준비되어 있습니다. 기획 단계부터 개발 완료까지, 모든 단계에 맞는 템플릿이 있습니다. 템플릿을 선택하고 내용만 채우면 됩니다. 매번 지시문을 새로 작성할 필요가 없습니다.

넷째, 자동화 스크립트입니다. 코드를 저장하면 자동으로 배포 폴더에 복사되는 Pre-commit Hook이 설정되어 있습니다. 수작업으로 파일을 복사할 필요가 없습니다. git commit만 하면 자동으로 동기화됩니다. 휴먼 에러가 발생할 여지를 없앱니다.

다섯째, SAL Grid 시스템입니다. 프로젝트의 모든 Task를 추적하는 관리 체계입니다. 어떤 작업이 완료되었는지, 어떤 작업이 남았는지 한눈에 볼 수 있습니다. 진행 상황을 시각화하는 대시보드도 포함되어 있습니다. 프로젝트 전체를 조감할 수 있습니다.

---

## 왜 필요한가

Claude Code는 강력한 도구입니다. 코드를 작성하고, 파일을 수정하고, 복잡한 작업을 수행합니다. 하지만 체계적인 프로젝트 관리 없이 사용하면 문제가 생깁니다.

파일이 여기저기 흩어집니다. 어떤 작업을 했는지 추적하기 어려워집니다. 나중에 수정이 필요할 때 어디서부터 손대야 할지 모르게 됩니다. 프로젝트가 커질수록 혼란은 더 심해집니다.

이런 문제는 AI의 능력과 관계없습니다. 아무리 똑똑한 AI라도 규칙 없이 일하면 결과물이 엉망이 됩니다. 사람과 AI가 함께 일하려면 명확한 프로세스가 필요합니다. 그래야 일관된 품질이 나옵니다.

SSALWorks 방법론은 이 문제를 해결합니다. 모든 파일은 정해진 위치에 저장됩니다. 모든 작업은 기록됩니다. AI가 실수하지 않도록 규칙이 명확히 정의되어 있습니다. 검증 절차가 있어서 문제를 조기에 발견합니다.

Dev Package는 이 방법론을 바로 사용할 수 있게 Package로 만든 것입니다. 방법론을 배우는 데 시간을 쓸 필요 없이, 다운로드하면 바로 적용됩니다.

---

## 어떻게 사용하는가

사용법은 간단합니다. 세 단계면 됩니다.

첫 번째 단계입니다. Package를 다운로드하고 압축을 풉니다. 폴더 이름을 자신의 프로젝트명으로 바꿉니다. 이것만으로 폴더 구조가 완성됩니다.

두 번째 단계입니다. 터미널을 열고 scripts 폴더로 이동합니다. install-hooks.bat 파일을 실행합니다. Windows가 아니라면 install-hooks.sh를 실행합니다. Pre-commit Hook이 설치됩니다. 이제 자동화 준비가 완료됩니다.

세 번째 단계입니다. Claude Code를 실행합니다. AI가 CLAUDE.md를 읽고 프로젝트 규칙을 자동으로 인식합니다. 파일 저장 위치, 명명 규칙, 검증 절차를 모두 알고 있습니다. 사용자가 별도로 설명하지 않아도 됩니다.

이후부터는 Order Sheet를 활용합니다. 원하는 작업에 맞는 템플릿을 선택합니다. 내용을 채워서 Claude Code에게 전달합니다. AI가 규칙에 따라 작업을 수행합니다.

작업이 완료되면 git commit을 합니다. Pre-commit Hook이 자동으로 배포 폴더에 파일을 복사합니다. Vercel에 연결해두면 자동으로 배포까지 됩니다. 수동 작업이 최소화됩니다.

---

## 누구를 위한 것인가

이 Package는 비개발자에게 특히 유용합니다. 코딩을 몰라도 AI와 함께 웹 서비스를 만들 수 있습니다. 프로세스를 따라가면 됩니다. 복잡한 설정은 이미 되어 있습니다. 기술적 지식이 없어도 결과물을 만들어낼 수 있습니다.

개발자에게도 유용합니다. 프로젝트 초기 설정에 드는 시간을 절약할 수 있습니다. 검증된 구조를 바로 사용할 수 있습니다. 사이드 프로젝트를 빠르게 시작할 수 있습니다. 반복되는 설정 작업에서 해방됩니다.

팀에게도 유용합니다. 표준화된 구조 덕분에 누가 작업해도 일관된 결과물이 나옵니다. 인수인계가 쉬워집니다. 새 팀원도 구조를 빠르게 파악할 수 있습니다. 협업의 효율이 크게 올라갑니다.

---

## 다른 방법과의 차이

빈 폴더에서 시작하면 모든 것을 처음부터 만들어야 합니다. 구조 설계, 규칙 정의, 자동화 설정까지 직접 해야 합니다. 시간이 오래 걸리고, 시행착오가 많습니다. 체계 없이 시작하면 나중에 수습하기 어렵습니다.

다른 보일러플레이트를 사용할 수도 있습니다. 하지만 대부분 코드 템플릿만 제공합니다. AI 협업 규칙이나 프로젝트 관리 체계는 포함되어 있지 않습니다. AI와 함께 일하려면 AI를 위한 설정이 필요합니다.

Dev Package는 AI 협업에 최적화되어 있습니다. 코드뿐 아니라 프로세스까지 포함합니다. Claude Code와 함께 사용할 때 최대 효과를 발휘합니다. AI 시대의 새로운 개발 방식입니다.

---

## 결론

Dev Package는 단순한 템플릿이 아닙니다. 프로젝트 구조, AI 협업 규칙, 작업 지시 템플릿, 자동화 스크립트, Task 관리 시스템이 하나로 통합된 개발 환경입니다.

Claude Code만으로는 체계적인 프로젝트 관리가 어렵습니다. SSALWorks Dev Package를 사용하면 처음부터 올바른 구조로 시작할 수 있습니다.

AI와 협업하는 시대입니다. 도구만 좋다고 되는 게 아닙니다. 도구를 효과적으로 사용하는 체계가 필요합니다. Dev Package가 그 체계를 제공합니다. 검증된 구조로 시작하세요.

다운로드하고, 압축 풀고, 시작하세요. 프로젝트의 성공은 올바른 시작에서 결정됩니다.

수백 시간의 시행착오를 거쳐 만들어진 검증된 시스템입니다. 직접 겪지 않아도 됩니다.

이미 정리된 체계를 그대로 사용하면 됩니다. AI와 함께하는 프로젝트, Dev Package로 시작하세요.
