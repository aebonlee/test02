# Project_Status.md로 진행 상황과 다음 할 일 기록하기

> 이 문서는 Project_Status.md 파일을 만들어 프로젝트 전체 진행 상황을 추적하는 방법을 설명합니다.

---

## 왜 진행 상황 문서가 필요한가

프로젝트가 어디까지 진행됐는지 한눈에 파악할 수 있습니다. Claude Code도 이 문서를 참조하여 현재 상태를 이해하고 다음 작업을 제안합니다.

---

## 문서 저장 위치

```
P0_프로젝트_구조/
├── Project_Status.md            ← 여기
└── Project_Directory_Structure.md
```

---

## 문서에 포함할 내용

| 섹션 | 내용 |
|------|------|
| 전체 진행률 | 단계별 완료 현황 |
| 완료된 단계 | 끝난 단계와 주요 산출물 |
| 현재 진행 중 | 작업 중인 Task 목록 |
| 다음 작업 | 앞으로 해야 할 일 |
| 블로커/이슈 | 막힌 부분 |

---

## 상태 표시 기호

```
완료     - 작업 끝남
진행중   - 현재 작업 중
대기     - 아직 시작 안 함
블로커   - 진행 불가
```

---

## 문서 예시

```markdown
# Project Status

> 최종 업데이트: 2025-01-15

## 전체 진행률

| Stage | 상태 | 진행률 |
|-------|------|--------|
| S1 개발 준비 | 완료 | 100% |
| S2 개발-1차 | 진행중 | 60% |
| S3 개발-2차 | 대기 | 0% |

## 현재 진행 중
- S2F1: 로그인 UI (진행중)
- S2BA1: 로그인 API (대기)

## 블로커
- Google OAuth 설정 대기 중

## 다음 작업
- S2F2: 회원가입 UI
```

---

## Project_Status vs work_logs 비교

| 항목 | Project_Status.md | work_logs/current.md |
|------|-------------------|---------------------|
| 목적 | 전체 진행 현황 | 상세 작업 기록 |
| 범위 | 프로젝트 전체 | 최근 세션 |
| 상세도 | 요약 | 상세 |
| 대상 | 전체 팀원 | 개발자/AI |

둘 다 함께 사용하면 가장 효과적입니다.
