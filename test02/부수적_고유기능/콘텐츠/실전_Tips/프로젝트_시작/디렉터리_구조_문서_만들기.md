# Project_Directory_Structure.md 파일로 전체 폴더 구조 문서화하기

> 이 문서는 프로젝트의 폴더 구조를 문서화하여 파일 저장 위치를 명확하게 하는 방법을 설명합니다.

---

## 왜 구조 문서가 필요한가

폴더 구조가 문서화되어 있으면 파일을 어디에 저장해야 하는지 명확합니다. Claude Code도 이 문서를 참조하여 올바른 위치에 파일을 저장합니다.

---

## 문서 저장 위치

```
P0_프로젝트_구조/
├── Project_Directory_Structure.md   ← 여기
└── Project_Status.md
```

---

## 문서에 포함할 내용

| 섹션 | 내용 |
|------|------|
| 전체 폴더 트리 | 한눈에 보이는 구조도 |
| 각 폴더 설명 | 용도와 저장할 파일 유형 |
| 파일 저장 규칙 | 어떤 파일을 어디에 저장 |
| 네이밍 규칙 | 파일명/폴더명 작성법 |

---

## 폴더 트리 예시

```
프로젝트/
├── P0_프로젝트_구조/       # 기초 문서, 프로젝트 상태
├── P1_시장조사/           # 시장 분석, 경쟁사 조사
├── P2_기획/               # 기능 기획, 화면 설계
├── S1_개발_준비/          # 개발 환경 세팅
├── S2_개발-1차/           # 핵심 기능 개발
├── S3_개발-2차/           # 고급 기능 개발
├── S4_개발-3차/           # 최적화, 테스트
├── S5_개발_마무리/               # 배포, 운영
├── Production/            # 배포용 최신 코드
└── .claude/               # Claude Code 설정
```

---

## 파일 저장 규칙 예시

| 파일 유형 | Stage 폴더 | Production 폴더 |
|----------|------------|-----------------|
| HTML 페이지 | S?_*/Frontend/ | Production/Frontend/ |
| API 코드 | S?_*/Backend_API/ | Production/Backend_API/ |
| SQL 파일 | S?_*/Database/ | - (Production에 안 넣음) |
| 테스트 코드 | S?_*/Testing/ | - |

---

## 네이밍 규칙 예시

```markdown
## 파일명 규칙

### 코드 파일
- kebab-case 사용: `google-login.js`, `subscription-cancel.js`
- 기능-동작 형태: `email-send.js`, `user-profile.html`

### 폴더명
- P1~P3: 예비단계 (한글+숫자)
- S1~S5: 실행단계 (한글+숫자)
- 하위 폴더: 영문 (Frontend, Backend_API)
```
