# 프로젝트 시작 시 디렉터리 구조부터 먼저 생성하기

> 이 문서는 프로젝트 시작 시 체계적인 디렉터리 구조를 먼저 설정하는 방법을 설명합니다.

---

## 왜 먼저 구조를 만드는가

| 이점 | 설명 |
|------|------|
| 파일 위치 명확 | 어디에 저장할지 고민 불필요 |
| 일관성 유지 | 모든 파일이 규칙에 따라 저장 |
| 협업 용이 | 다른 사람도 구조 파악 쉬움 |
| 관리 편의 | 나중에 정리할 필요 없음 |

---

## Claude Code에게 요청

```
"프로젝트 디렉터리 구조 먼저 만들어줘"
"이 프로젝트에 맞는 폴더 구조 제안해줘"
```

---

## 기본 구조 예시

```
프로젝트/
├── Production/           # 배포용 파일
│   ├── Frontend/         # 프론트엔드
│   └── api/              # 백엔드 API
├── Documentation/        # 문서
├── Database/             # DB 스키마, SQL
├── Testing/              # 테스트 코드
└── .claude/              # Claude Code 설정
    └── work_logs/        # 작업 기록
```

---

## 구조 문서화

디렉터리 구조를 문서로 남기기:
```
"현재 폴더 구조를 Project_Directory_Structure.md에 저장해줘"
```

---

## 단계별 진행

1. **구조 설계**: 필요한 폴더 목록 작성
2. **폴더 생성**: 빈 폴더들 생성
3. **문서화**: 구조 문서 작성
4. **CLAUDE.md 연결**: 규칙 파일에 참조 추가

---

## 구조 설계 시 고려사항

| 고려사항 | 예시 |
|----------|------|
| 프로젝트 유형 | 웹앱, API, 라이브러리 |
| 기술 스택 | React, Node.js, Python |
| 팀 규모 | 혼자 또는 팀 |
| 배포 환경 | Vercel, AWS, 로컬 |

---

## 흔한 실수

| 실수 | 해결 |
|------|------|
| 구조 없이 시작 | 첫 작업 전 구조 설계 |
| 너무 복잡한 구조 | 단순하게 시작, 필요시 확장 |
| 문서화 안 함 | 구조 문서 필수 작성 |
