# Claude Code 설치 전에 Node.js와 Git을 설치하는 이유

> Claude Code를 설치하기 전에 Node.js와 Git을 먼저 설치해야 합니다. 왜 이 두 가지가 필요한지 이해하면 개발 환경을 더 잘 활용할 수 있습니다.

---

## Claude Code의 실행 환경, Node.js

Claude Code는 JavaScript라는 프로그래밍 언어로 만들어져 있습니다. 원래 JavaScript는 웹 브라우저에서만 실행할 수 있었습니다. 하지만 Node.js라는 프로그램이 등장하면서 컴퓨터에서도 JavaScript를 실행할 수 있게 되었습니다.

쉽게 비유하면, Node.js는 Claude Code라는 앱을 실행하는 운영체제 같은 역할입니다. 스마트폰에서 앱을 실행하려면 안드로이드나 iOS가 필요한 것처럼, Claude Code를 실행하려면 Node.js가 필요합니다.

Node.js를 설치하면 npm이라는 패키지 관리자도 함께 설치됩니다. npm은 다양한 프로그램을 손쉽게 설치하고 관리할 수 있게 해주는 도구입니다. Claude Code도 npm을 통해 설치합니다.

설치 명령어는 간단합니다. 터미널에서 `npm install -g @anthropic-ai/claude-code`를 실행하면 됩니다. 이 명령어가 작동하려면 Node.js가 먼저 설치되어 있어야 합니다.

Node.js 버전은 18 이상을 권장합니다. 너무 오래된 버전은 Claude Code가 제대로 작동하지 않을 수 있습니다.

---

## 코드 관리의 핵심, Git

Git은 코드의 버전을 관리하는 프로그램입니다. 문서 작업할 때 "최종.docx", "최종_수정.docx", "진짜최종.docx"처럼 파일을 여러 개 만들어본 경험이 있을 겁니다. Git은 이런 혼란을 해결해줍니다.

Git을 사용하면 파일의 변경 이력이 자동으로 기록됩니다. 언제, 무엇을, 왜 바꿨는지 모두 추적할 수 있습니다. 실수로 코드를 잘못 수정해도 이전 상태로 되돌릴 수 있습니다.

Claude Code는 Git과 긴밀하게 연동됩니다. 코드를 작성하면서 git status로 변경 사항을 확인하고, git diff로 수정 전후를 비교합니다. 작업이 끝나면 git commit으로 저장하고, git push로 GitHub에 업로드합니다.

Git이 없어도 Claude Code 자체는 실행됩니다. 파일을 읽고 쓰는 기본적인 작업은 가능합니다. 하지만 Git 없이 작업하면 변경 이력을 관리할 수 없고, 실수했을 때 복구가 어렵습니다.

---

## Git 프로그램과 .git 폴더는 다른 것

프로젝트 폴더에서 `.git`이라는 숨김 폴더가 생기는 것을 보신 적이 있을 겁니다. 이것과 Git 프로그램을 혼동하기 쉬운데, 둘은 전혀 다른 것입니다.

Git 프로그램은 시스템 전체에 설치됩니다. Windows에서는 `C:\Program Files\Git\` 폴더에 설치됩니다. 한 번만 설치하면 어느 폴더에서든 git 명령어를 사용할 수 있습니다.

반면 .git 폴더는 각 프로젝트마다 생성됩니다. 프로젝트의 모든 버전 기록을 저장하는 저장소입니다. 카메라와 사진첩에 비유할 수 있습니다. Git 프로그램은 카메라(도구)이고, .git 폴더는 사진첩(기록물)입니다. 카메라는 하나지만 사진첩은 여러 개 만들 수 있습니다.

.git 폴더는 git init 명령어를 실행하거나, GitHub에서 프로젝트를 clone할 때 생성됩니다. Claude Code로 작업을 시작할 때도 버전 관리를 위해 자동으로 생성되기도 합니다.

.git 폴더 안에는 모든 커밋 기록, 브랜치 정보, 원격 저장소(GitHub) 연결 정보가 저장됩니다. 이 폴더가 있어야 git log, git diff 같은 명령어가 작동합니다.

중요한 점은, .git 폴더를 삭제하면 해당 프로젝트의 모든 버전 기록이 사라진다는 것입니다. 실수로 삭제하지 않도록 주의해야 합니다.

## SSALWorks 방법론에서 Git이 필수인 이유

SSALWorks Dev Package는 Git을 기반으로 설계되어 있습니다. 체계적인 프로젝트 관리를 위해 Git이 반드시 필요합니다.

Dev Package에는 Pre-commit Hook이라는 기능이 포함되어 있습니다. 코드를 커밋할 때 자동으로 검증이 실행되고, Stage 폴더의 파일이 배포 폴더로 복사됩니다. 이 모든 자동화는 Git이 있어야 작동합니다.

GitHub Pages를 통한 배포도 Git이 필수입니다. 프로젝트를 GitHub에 푸시하면 자동으로 웹사이트가 배포됩니다. SAL Grid Viewer를 웹에서 확인하려면 이 과정이 필요합니다.

Claude Code만으로는 파일을 이곳저곳에 흩어놓게 되기 쉽습니다. 하지만 SSALWorks 방법론은 Git을 활용해 모든 변경 사항을 체계적으로 관리합니다. 나중에 "이 파일을 언제 왜 만들었지?" 하는 혼란을 방지할 수 있습니다.

## 설치 순서 정리

개발 환경을 처음부터 구축한다면 다음 순서로 설치하면 됩니다.

첫째, Git을 설치합니다. git-scm.com에서 다운로드할 수 있습니다. 설치 후 터미널에서 `git --version`을 실행해 설치가 잘 되었는지 확인합니다.

둘째, Node.js를 설치합니다. nodejs.org에서 LTS 버전을 다운로드합니다. LTS는 Long Term Support의 약자로, 안정적인 버전을 의미합니다. 설치 후 `node --version`과 `npm --version`으로 확인합니다.

셋째, Claude Code를 설치합니다. 터미널에서 `npm install -g @anthropic-ai/claude-code`를 실행합니다. 설치 후 `claude --version`으로 확인합니다.

넷째, Claude Code를 실행합니다. 작업할 폴더로 이동한 후 `claude`를 입력하면 됩니다. 처음 실행하면 로그인 과정을 거칩니다.

---

## 결론

Node.js는 Claude Code의 실행 환경이고, Git은 코드 버전 관리 도구입니다. 둘 다 Claude Code를 설치하기 전에 먼저 설치해야 하는 필수 요소입니다.

특히 SSALWorks 방법론을 따른다면 Git은 선택이 아닌 필수입니다. Dev Package의 자동화 기능, 체계적인 버전 관리, GitHub 배포 모두 Git을 기반으로 작동합니다.

처음에는 설치할 것이 많아 보일 수 있지만, 한 번 설정해두면 이후 작업이 훨씬 수월해집니다. Git 설치 → Node.js 설치 → Claude Code 설치 순서만 기억하면 됩니다.
