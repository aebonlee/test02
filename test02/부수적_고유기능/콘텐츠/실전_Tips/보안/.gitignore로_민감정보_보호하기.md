# .gitignore로 민감 정보가 GitHub에 올라가지 않게 보호하기

> 이 문서는 .gitignore를 사용하여 민감한 정보를 Git 저장소에서 제외하는 방법을 설명합니다.

---

## 왜 .gitignore가 필요한가

| 보호 대상 | 노출 시 위험 |
|----------|-------------|
| API 키 | 과금 발생, 악용 |
| DB 비밀번호 | 데이터 유출 |
| 개인정보 | 법적 문제 |
| 환경변수 | 서버 해킹 |

---

## .gitignore 기본 설정

```gitignore
# 환경변수
.env
.env.local
.env.production

# 의존성
node_modules/

# 빌드 결과물
dist/
build/

# 로그
*.log

# 운영체제 파일
.DS_Store
Thumbs.db
```

---

## Claude Code에게 요청

```
".gitignore 파일 확인해줘"
".env 파일이 .gitignore에 있는지 확인해줘"
"민감한 파일이 git에 추적되고 있는지 확인해줘"
```

---

## 이미 커밋된 파일 제외

```bash
# git에서 추적 중단 (파일은 유지)
git rm --cached .env
git commit -m "Remove .env from tracking"
```

---

## 자주 제외하는 파일

| 파일 | 이유 |
|------|------|
| .env* | API 키, 비밀번호 |
| node_modules/ | 용량 큼, 재설치 가능 |
| *.log | 불필요한 로그 |
| dist/, build/ | 빌드 결과물 |
| .DS_Store | macOS 시스템 파일 |

---

## 확인 방법

```bash
git status  # 추적 중인 파일 확인
git ls-files  # 저장소에 있는 모든 파일
```

---

## .gitignore 작성 규칙

| 패턴 | 의미 |
|------|------|
| *.log | 모든 .log 파일 |
| /dist | 루트의 dist 폴더만 |
| build/ | 모든 위치의 build 폴더 |
| !important.log | important.log는 제외하지 않음 |

---

## 실수 방지

- 프로젝트 시작 시 .gitignore 먼저 설정
- 민감 파일 생성 전 .gitignore 확인
- 커밋 전 git status로 확인
