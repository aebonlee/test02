# Claude Code 세션이 끊어졌을 때 이전 작업 이어서 진행하기

> 이 문서는 세션이 끊어졌을 때 이전 작업을 복구하고 이어서 진행하는 방법을 설명합니다.

---

## 세션이 끊어지는 상황

| 상황 | 원인 |
|------|------|
| 터미널 종료 | 실수로 창 닫음 |
| 컴퓨터 재시작 | 업데이트, 전원 문제 |
| 네트워크 끊김 | 인터넷 연결 불안정 |
| 타임아웃 | 오랜 시간 응답 없음 |

---

## 복구 방법

**1. 터미널 다시 열기**
- 프로젝트 폴더에서 터미널 열기
- `claude` 명령어 실행

**2. 이전 작업 파악**
```
"work_logs/current.md 읽고 이전 작업 상황 파악해줘"
```

**3. 이어서 진행**
```
"[마지막 작업]까지 완료됐고, [다음 작업]부터 이어서 진행해줘"
```

---

## work_logs 활용

work_logs/current.md에 작업 내역이 기록되어 있으면 복구가 쉽습니다.

```
"오늘 작업한 내용 요약해서 work_logs/current.md에 저장해줘"
```

---

## 복구 시 확인할 것

| 확인 항목 | 방법 |
|----------|------|
| 마지막 저장 파일 | git status 또는 파일 탐색기 |
| 완료된 작업 | work_logs 확인 |
| 미완료 작업 | 이전 대화 기억 기반 |

---

## 복구 요청 템플릿

```
"세션이 끊어졌어. 상황:
- 마지막 작업: [작업 내용]
- 완료된 것: [파일 목록]
- 남은 것: [작업 내용]

이어서 진행해줘."
```

---

## 자동 복구 정보

Claude Code는 최근 대화 히스토리를 저장합니다.

```
"이전 대화 내용 요약해줘"
```

---

## 복구 후 확인

| 확인 | 방법 |
|------|------|
| 파일 상태 | 수정된 파일 목록 확인 |
| 코드 동작 | 테스트 실행 |
| 누락 여부 | 계획과 비교 |

---

## 예방책

- 중요한 작업 완료 시마다 work_logs 업데이트
- 정기적으로 git commit
- 긴 작업은 단계별로 나누기

---

## 완전 복구 불가 시

```
"처음부터 다시 시작해야 해.
목표: [최종 목표]
현재 상태: [파일 상태]
진행해줘."
```
