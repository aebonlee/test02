# 세션 중 삽질하면 미련 없이 세션 정리하고 새로 시작하기

> 이 문서는 작업 중 막혔을 때 세션을 정리하고 새로 시작하는 방법을 설명합니다.

---

## 왜 세션을 정리하는가

AI가 잘못된 방향으로 진행 중일 때 계속 수정을 요청하면 상황이 더 복잡해질 수 있습니다. 차라리 깨끗하게 새로 시작하는 것이 효율적입니다.

---

## 세션 정리가 필요한 신호

| 신호 | 의미 |
|------|------|
| 같은 에러 반복 | 근본 원인을 못 찾고 있음 |
| 점점 복잡해짐 | 해결책이 더 문제를 만듦 |
| 5분 이상 삽질 | 방향을 잘못 잡았을 가능성 |
| AI가 혼란스러워 보임 | 컨텍스트가 꼬임 |

---

## 세션 정리 방법

**1. 현재 상태 기록**
```
"지금까지 한 작업을 요약해줘"
```

**2. work_logs에 저장**
```
"요약 내용을 work_logs/current.md에 저장해줘"
```

**3. 새 세션 시작**
- Claude Code 종료 (Ctrl+C 또는 /exit)
- 터미널에서 다시 `claude` 실행

---

## 새 세션에서 할 일

```
"work_logs/current.md 읽고 이전 작업 파악해줘"
"[문제 상황]에서 막혔어. 다른 방법으로 접근해보자"
```

---

## 새로 시작할 때 명확히 할 것

| 항목 | 예시 |
|------|------|
| 최종 목표 | "로그인 기능 구현" |
| 현재 상태 | "UI는 완료, API 연동에서 막힘" |
| 이전에 시도한 것 | "fetch로 했는데 CORS 에러" |
| 원하는 접근법 | "다른 방식으로 해결해줘" |

---

## 세션 정리 없이 방향 전환

```
"이 방향은 안 되는 것 같아.
지금까지 한 건 무시하고, 완전히 다른 방법으로 접근해보자.
[새로운 요구사항]"
```

---

## 미련 없이 버릴 것

| 버려도 됨 | 저장 필요 |
|----------|----------|
| 실패한 시도 | 성공한 코드 |
| 삽질 과정 | 핵심 발견 사항 |
| 임시 파일 | 중요 설정 |

---

## 정리 후 확인

- 새 세션에서 컨텍스트를 빠르게 파악할 수 있는가?
- 이전 실수를 반복하지 않을 정보가 있는가?
