# /resume 커맨드로 이전 세션 자동 복구하기

> Claude Code와 Gemini CLI에서 세션이 끊어졌을 때 `/resume` 커맨드로 자동으로 이전 작업을 이어서 진행하는 방법

---

## 핵심 요약

세션이 끊어져도 걱정 없습니다. `/resume` 커맨드 하나면 이전 세션을 자동으로 찾아서 완벽하게 복구합니다.

```
/resume
```

---

## /resume이란?

**자동 세션 복구 커맨드**입니다.

| 항목 | 설명 |
|------|------|
| 기능 | 이전 세션을 찾아서 자동 복구 |
| 지원 | Claude Code, Gemini CLI |
| 복구 범위 | 전체 컨텍스트, 작업 기록, 대화 내용 |
| 사용법 | `/resume` 입력 |

---

## 기존 수동 복구 vs /resume 자동 복구

### ❌ 기존 방식 (수동)

```
1. 터미널 다시 열기
2. claude 명령어 실행
3. "work_logs/current.md 읽고 이전 작업 상황 파악해줘"
4. "마지막 작업까지 완료됐고, 다음 작업부터 이어서 진행해줘"
```

**문제점:**
- 여러 단계 필요
- 수동으로 상황 설명
- 컨텍스트 일부 손실 가능

### ✅ /resume 방식 (자동)

```
/resume
```

**장점:**
- ✅ 단 한 줄로 완벽 복구
- ✅ 이전 세션 자동 검색
- ✅ 전체 컨텍스트 완벽 유지
- ✅ 작업 연속성 보장

---

## 사용 시나리오

### 시나리오 1: 터미널이 꺼진 경우

**상황:**
- 실수로 터미널 창 닫음
- 작업 중이던 내용 날아감

**해결:**
```bash
# 터미널 다시 열기
claude

# 세션 복구
/resume
```

**결과:**
- ✅ 이전 작업 내용 그대로 복구
- ✅ 대화 히스토리 유지
- ✅ 바로 이어서 작업 가능

---

### 시나리오 2: 컴퓨터 재시작

**상황:**
- Windows 업데이트로 강제 재시작
- 작업하던 세션 모두 종료

**해결:**
```bash
# 프로젝트 폴더에서 터미널 열기
cd C:\MyProject
claude

# 세션 복구
/resume
```

**결과:**
- ✅ 재시작 전 작업 내용 완벽 복구
- ✅ 파일 상태, 대화 내역 모두 유지

---

### 시나리오 3: 네트워크 끊김

**상황:**
- 인터넷 연결 불안정
- Claude Code 연결 끊김

**해결:**
```bash
# 인터넷 연결 복구 후
/resume
```

**결과:**
- ✅ 끊기기 전 상태 그대로 복구
- ✅ 진행 중이던 작업 이어서 계속

---

### 시나리오 4: 다음 날 작업 재개

**상황:**
- 어제 작업하다가 퇴근
- 오늘 다시 시작

**해결:**
```bash
# 프로젝트 폴더에서
claude
/resume
```

**결과:**
- ✅ 어제 마지막 작업 상태 그대로
- ✅ "어제 뭐 했더라?" 고민 필요 없음

## /resume의 강력한 점

### 1. 완벽한 컨텍스트 유지

**복구되는 정보:**
- 전체 대화 내역
- 작업 중이던 파일 목록
- 진행 상황
- 다음 할 일
- 사용자 선호도 및 설정

### 2. work_logs와의 시너지

`/resume`과 `work_logs/current.md`를 함께 사용하면 더욱 강력합니다.

```
/resume
→ Claude가 자동으로 work_logs/current.md 확인
→ 이전 작업 완벽 파악
→ 바로 이어서 진행
```

### 3. Claude Code & Gemini CLI 공통 지원

**Claude Code:**
```bash
claude
/resume
```

**Gemini CLI:**
```bash
gemini
/resume
```

**동일한 사용법으로 양쪽 모두 지원!**

```bash
# 터미널 열자마자
claude
/resume  # 👈 습관화!
```

### Tip 2: work_logs 자동 업데이트와 병행

```
작업 완료 시마다:
"work_logs/current.md에 지금까지 작업 내용 정리해줘"

다음 날:
/resume
→ work_logs 자동 참조
→ 완벽한 복구
```

### Tip 3: 중요 작업 전후로 사용

**작업 전:**
```
/resume  # 이전 상태 확인
```

**작업 중 문제 발생 시:**
```
/resume  # 마지막 안정 상태로 돌아가기
```

`/resume`과 함께 사용하면 좋은 커맨드들:

| 커맨드 | 기능 | 사용 예시 |
|--------|------|----------|

**조합 활용 예시:**
```bash
/resume      # 세션 복구
/test        # 테스트 실행
/commit      # 커밋
/deploy      # 배포
```

## /resume이 작동하는 원리

**내부 동작:**

```
1. /resume 입력
   ↓
2. Claude Code가 이전 세션 기록 검색
   - 로컬 세션 히스토리 파일
   - 대화 내역 캐시
   - work_logs 자동 참조
   ↓
3. 가장 최근 세션 찾기
   ↓
4. 전체 컨텍스트 로드
   - 대화 내역
   - 파일 상태
   - 작업 진행 상황
   ↓
5. 세션 복구 완료
   - "이전 작업 이어서 진행합니다"
```

## 주의사항

### 1. 여러 프로젝트 작업 시

```
프로젝트 A 세션:
cd C:\ProjectA
/resume  ✅ ProjectA 세션 복구

프로젝트 B 세션:
cd C:\ProjectB
/resume  ✅ ProjectB 세션 복구
```

**프로젝트별로 별도 세션 유지됨**

### 2. 오래된 세션

```
# 너무 오래된 세션은 복구 안 될 수 있음
# 이 경우 수동 설명 필요

"3일 전 작업하던 [기능명] 이어서 해줘.
마지막 상태: [설명]"
```

### 3. 세션 히스토리 저장 위치

**Claude Code:**
- Windows: `%USERPROFILE%\.claude\`
- Mac/Linux: `~/.claude/`

**자동 관리되므로 직접 건드릴 필요 없음**

## 문제 해결

### Q1: /resume이 작동하지 않아요

**확인사항:**
1. Claude Code 버전 확인 (최신 버전인가?)
2. 프로젝트 폴더 위치 확인 (올바른 폴더인가?)
3. 세션 히스토리 존재 여부 (처음 실행인가?)

**해결:**
```bash
# Claude Code 업데이트
npm install -g @anthropics/claude-code

# 수동 복구로 대체
"work_logs/current.md 읽고 상황 파악해줘"
```

### Q2: 다른 세션이 복구됐어요

**해결:**
```
"아니야, [프로젝트명]의 [작업명] 이어서 해줘"
```

### Q3: 복구된 내용이 너무 오래됐어요

**해결:**
```
"최신 상태로 업데이트해줘:
- 현재 파일 상태: [git status 결과]
- 작업 목표: [목표]"
```

## 실전 활용 예시

### 예시 1: 긴급 중단 후 복구

```
[오후 3시] API 개발 중 긴급 회의
→ 터미널 닫음

[오후 5시] 회의 끝나고 복귀
claude
/resume

Claude: "API 개발 작업 이어서 진행하겠습니다.
마지막으로 signup API 완료하셨고,
다음은 login API 작업 예정이셨습니다."

✅ 바로 작업 재개!
```

### 예시 2: 다음 날 작업 시작

```
[어제] 오후 6시 퇴근
→ 데이터베이스 스키마 작업 중

[오늘] 오전 9시 출근
claude
/resume

Claude: "어제 데이터베이스 스키마 작업하셨습니다.
users, posts 테이블 완료,
comments 테이블부터 이어서 하시겠습니까?"

✅ 어제 뭐 했는지 고민할 필요 없음!
```

### 예시 3: 여러 세션 관리

```
[프로젝트 A]
cd C:\ProjectA
/resume
→ ProjectA 작업

[잠시 프로젝트 B 확인]
cd C:\ProjectB
/resume
→ ProjectB 작업

[다시 프로젝트 A]
cd C:\ProjectA
/resume
→ ProjectA 원래 작업 그대로 복구

✅ 프로젝트별 세션 완벽 분리!
```
