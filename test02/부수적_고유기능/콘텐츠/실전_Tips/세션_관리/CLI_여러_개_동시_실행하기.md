# Claude Code 여러 개 동시 실행하여 병렬 작업하기

> 이 문서는 여러 터미널에서 Claude Code를 동시에 실행하여 병렬 작업하는 방법을 설명합니다.

---

## 왜 여러 개를 실행하는가

| 상황 | 이점 |
|------|------|
| 서로 다른 기능 개발 | 독립적으로 진행 가능 |
| 한 쪽에서 오래 걸리는 작업 | 다른 쪽에서 다른 작업 진행 |
| A/B 비교 테스트 | 두 가지 접근법 동시 시도 |

---

## 실행 방법

**터미널 여러 개 열기:**
1. 첫 번째 터미널: `claude` 실행
2. 두 번째 터미널 열기 (새 탭 또는 새 창)
3. 같은 프로젝트 폴더에서 `claude` 실행

---

## VS Code에서

1. 터미널 패널 우측 상단 + 버튼 클릭
2. 새 터미널에서 `claude` 실행
3. 터미널 간 전환: 드롭다운 메뉴 또는 Ctrl+PageUp/PageDown

---

## 주의사항

| 주의 | 설명 |
|------|------|
| 같은 파일 동시 수정 금지 | 충돌 발생 가능 |
| 작업 영역 구분 | 각 세션이 다른 파일 담당 |
| 커밋 충돌 주의 | 한 쪽에서만 커밋 |

---

## 권장 사용 패턴

| 세션 1 | 세션 2 |
|--------|--------|
| Frontend 개발 | Backend 개발 |
| 코드 작성 | 테스트 실행 |
| 메인 기능 | 버그 수정 |

---

## 세션 구분하기

각 세션 시작 시:
```
"이 세션에서는 Frontend 작업만 할 거야"
"이 세션에서는 Backend API만 작업할 거야"
```

---

## 파일 충돌 방지

| 방법 | 설명 |
|------|------|
| 영역 구분 | Frontend vs Backend |
| 파일 구분 | login.html vs api.js |
| 시간 구분 | 한 쪽 완료 후 다른 쪽 시작 |

---

## 동기화

작업 완료 후:
```
"방금 수정한 파일 목록 알려줘"
```
다른 세션에서:
```
"이 파일들이 변경됐으니 참고해줘: [파일 목록]"
```

## 자원 고려

- Claude Code는 API 호출 비용 발생
- 동시 실행 시 비용 2배
- 필요할 때만 병렬 실행 권장
