# Google Drive 동기화로 인한 파일 충돌 해결하기

> 이 문서는 Google Drive 동기화 폴더에서 작업할 때 발생하는 파일 충돌 문제를 해결하는 방법을 설명합니다.

---

## 문제 상황

Google Drive 동기화 폴더에서 작업하면 파일 수정 시 충돌이 발생할 수 있습니다.

```
Error: File has been unexpectedly modified
```

---

## 왜 문제가 생기는가

| 원인 | 설명 |
|------|------|
| 동기화 타이밍 | Claude가 파일 수정 중 Drive가 동기화 시도 |
| 파일 잠금 | Drive가 파일을 잠그고 있어 수정 불가 |
| 버전 충돌 | 로컬과 클라우드 버전 불일치 |

---

## 해결 방법 1: 동기화 일시 중지

1. Google Drive 트레이 아이콘 클릭
2. 설정 → 동기화 일시중지
3. 작업 완료 후 동기화 재개

---

## 해결 방법 2: 동기화 폴더 외부로 이동

프로젝트 폴더를 동기화 폴더 밖으로 이동:
```bash
# 예: C:\Projects\ 로 이동
mv ~/Google\ Drive/MyProject C:\Projects\MyProject
```

---

## 해결 방법 3: 선택적 동기화

Google Drive 설정에서 프로젝트 폴더만 동기화 제외:
1. Google Drive 설정 열기
2. 동기화할 폴더 선택
3. 프로젝트 폴더 체크 해제

---

## Claude Code에서 대처

파일 충돌 발생 시:
```
"방금 파일 수정이 안 됐어. 다시 시도해줘"
```

또는:
```
"파일을 다시 읽고 수정해줘"
```

---

## 권장 설정

| 항목 | 권장 |
|------|------|
| 프로젝트 위치 | 로컬 드라이브 (C:\Projects) |
| 백업 | Git 사용 |
| 동기화 | 작업 완료 후 수동 복사 |

---

| Claude Code 호환 | 낮음 | 높음 |

## 동기화 상태 확인

Google Drive 트레이 아이콘에서:
- 체크 표시: 동기화 완료
- 화살표: 동기화 진행 중
- 느낌표: 문제 발생
