# 서브에이전트로 복잡한 작업을 병렬 처리하기

> 이 문서는 Claude Code의 서브에이전트를 활용하여 복잡한 작업을 효율적으로 처리하는 방법을 설명합니다.

---

## 서브에이전트란

Main Agent가 복잡한 작업을 처리할 때 별도의 서브에이전트를 생성하여 특정 작업을 위임하는 기능입니다.

---

## 언제 사용되는가

| 상황 | 설명 |
|------|------|
| 복잡한 검색 | 코드베이스 전체 탐색 |
| 병렬 작업 | 여러 파일 동시 처리 |
| 전문 작업 | 코드 리뷰, 보안 검사 등 |

---

## 서브에이전트 종류

| 에이전트 | 역할 |
|----------|------|
| Explore | 코드베이스 탐색 |
| Plan | 구현 계획 수립 |
| code-reviewer | 코드 리뷰 |
| qa-specialist | 품질 검증 |
| security-auditor | 보안 검사 |

---

## 사용자가 요청하는 방법

```
"서브에이전트로 코드 리뷰해줘"
"병렬로 여러 파일 처리해줘"
"탐색 에이전트로 관련 파일 찾아줘"
```

---

## 서브에이전트 개수 제한

한 번에 너무 많은 서브에이전트를 실행하면 비효율적입니다.

| 권장 | 설명 |
|------|------|
| 동시 3개 이하 | 리소스 효율성 |
| 순차 처리 | 의존성 있는 작업 |
| 병렬 처리 | 독립적인 작업 |

---

## 서브에이전트 결과 확인

```
"서브에이전트 작업 결과 알려줘"
"검증 결과 요약해줘"
```

---

## 효율적인 사용 팁

| 팁 | 설명 |
|-----|------|
| 명확한 지시 | 구체적인 작업 범위 지정 |
| 적절한 분할 | 너무 크지도 작지도 않게 |
| 결과 통합 | 서브에이전트 결과 종합 |

---

## 서브에이전트 vs 직접 처리

| 직접 처리 | 서브에이전트 |
|----------|------------|
| 간단한 작업 | 복잡한 탐색 |
| 단일 파일 | 다중 파일 |
| 빠른 응답 필요 | 철저한 검사 필요 |
