# Pre-commit Hook으로 반복작업 자동화하기

> Git 커밋할 때마다 자동으로 실행되는 스크립트로 반복 작업을 없애는 방법

---

## 매번 같은 작업을 반복하고 있나요?

개발을 하다 보면 매번 같은 작업을 반복하게 됩니다. 마크다운 파일을 수정하면 자바스크립트로 번들링해야 하고, 코드를 수정하면 다른 폴더에 복사해야 합니다. 이런 작업을 잊어버리면 배포할 때 문제가 생깁니다.

사람은 반복 작업을 잊어버리기 쉽습니다. 10번 중 1번은 실수합니다. 그래서 자동화가 필요합니다.

---

## Pre-commit Hook이란 무엇인가

Pre-commit Hook은 Git에서 커밋하기 직전에 자동으로 실행되는 스크립트입니다. "커밋과 동시에" 원하는 작업들이 자동으로 실행됩니다.

레스토랑에 비유하면, 주문을 받으면 자동으로 주방에 전달되는 것과 같습니다. 웨이터가 매번 주방으로 달려갈 필요가 없습니다. 시스템이 알아서 처리합니다.

Pre-commit Hook도 마찬가지입니다. 개발자가 커밋만 하면, 필요한 모든 작업이 자동으로 실행됩니다.

---

## 어떤 작업을 자동화할 수 있나

SSALWorks에서는 8가지 작업을 Pre-commit Hook으로 자동화합니다.

첫째, Order Sheet 마크다운 파일들을 자바스크립트로 번들링합니다. 마크다운을 수정하면 자동으로 웹에서 사용할 수 있는 형태로 변환됩니다.

둘째, Briefings 안내문들을 자바스크립트로 번들링합니다. 상황별 안내문을 마크다운으로 작성하면 자동으로 웹용 파일이 생성됩니다.

셋째, 외부 연동 설정 가이드를 자바스크립트로 번들링합니다. Google OAuth, Stripe 같은 외부 서비스 설정 가이드가 자동 변환됩니다.

넷째, 서비스 소개 모달 내용을 메인 페이지에 삽입합니다. 마크다운으로 서비스 소개를 작성하면 index.html에 자동으로 반영됩니다.

다섯째, SAL Grid 매뉴얼을 HTML로 변환합니다. 마크다운 매뉴얼이 깔끔한 HTML 페이지가 됩니다.

여섯째, 빌더 계정 매뉴얼을 HTML로 변환합니다. 사용자용 매뉴얼이 자동으로 웹페이지가 됩니다.

일곱째, P0부터 S5까지의 진행률을 자동으로 계산합니다. 폴더 구조와 SAL Grid 데이터를 분석하여 JSON 파일로 저장합니다.

여덟째, Stage 폴더의 코드를 배포 폴더로 복사합니다. 개발 폴더에서 작업한 내용이 자동으로 배포 폴더에 동기화됩니다.

---

## 왜 SSALWorks 방법론이 필요한가

Claude Code만 사용하면 이런 자동화를 직접 설정해야 합니다. Pre-commit Hook이 뭔지 알아야 하고, 스크립트를 직접 작성해야 합니다. Git 설정 파일을 수정해야 하고, 오류가 나면 디버깅해야 합니다.

SSALWorks 방법론은 이 모든 것이 이미 설정되어 있습니다. 패키지를 받으면 바로 사용할 수 있습니다. Git을 설치하고 커밋만 하면 7가지 자동화가 작동합니다.

개발자가 아닌 사람도 쉽게 사용할 수 있습니다. 원리를 몰라도 됩니다. 그냥 커밋하면 됩니다.

---

## 설정 없이 바로 사용하기

SSALWorks 패키지에는 Pre-commit Hook이 미리 설정되어 있습니다. 별도의 설정이 필요 없습니다.

Git을 설치하고 프로젝트 폴더에서 작업을 시작하면 됩니다. 파일을 수정하고 커밋하면 8가지 자동화가 작동합니다.

커밋할 때 터미널에 "Pre-commit Hook 실행 중"이라는 메시지가 나타납니다. 잠시 기다리면 모든 작업이 완료됩니다. 성공하면 커밋이 진행되고, 실패하면 무엇이 문제인지 알려줍니다.

---

## 마크다운으로 작성하면 모든 것이 자동화된다

SSALWorks에서는 마크다운으로 콘텐츠를 작성합니다. 마크다운은 간단한 문법으로 문서를 작성하는 방식입니다. 특별한 프로그램 없이 메모장으로도 작성할 수 있습니다.

마크다운으로 Order Sheet를 작성하면 자동으로 웹에서 사용할 수 있는 형태가 됩니다. 안내문을 작성하면 자동으로 변환됩니다. 매뉴얼을 작성하면 자동으로 웹페이지가 됩니다.

개발 지식이 없어도 콘텐츠를 만들 수 있습니다. 마크다운만 작성하면 나머지는 시스템이 처리합니다.

---

## 실수를 방지하는 안전장치

Pre-commit Hook은 실수를 방지하는 안전장치 역할도 합니다. 빌드에 실패하면 커밋이 중단됩니다. 문제가 있는 상태로 저장소에 올라가는 것을 막아줍니다.

예를 들어 마크다운 파일에 문법 오류가 있으면 번들링이 실패합니다. 커밋이 중단되고 어떤 파일에 문제가 있는지 알려줍니다. 문제를 수정하고 다시 커밋하면 됩니다.

이렇게 하면 항상 정상 작동하는 코드만 저장소에 올라갑니다. 팀으로 작업할 때 특히 유용합니다.

---

## 결론

Pre-commit Hook은 반복 작업을 자동화하는 강력한 도구입니다. SSALWorks 방법론에서는 8가지 자동화가 미리 설정되어 있습니다. Git만 설치하면 바로 사용할 수 있습니다.

마크다운으로 콘텐츠를 작성하고 커밋하면 모든 것이 자동으로 처리됩니다. 번들링, 변환, 복사를 일일이 할 필요가 없습니다. 실수할 걱정도 없습니다.

개발에 집중하고 반복 작업은 시스템에 맡기세요. 그것이 SSALWorks 방법론의 핵심입니다.
