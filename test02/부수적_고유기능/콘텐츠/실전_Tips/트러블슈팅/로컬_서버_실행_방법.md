# npx serve로 HTML 파일 로컬 서버에서 실행하고 포트 충돌 해결하기

> 이 문서는 HTML 파일을 로컬 서버로 실행하는 방법과 포트 충돌 해결 방법을 설명합니다.

---

## 왜 서버가 필요한가?

파일을 더블클릭해서 열면 주소가 `file://`로 시작합니다. 이 방식은 보안 제한 때문에 많은 기능이 작동하지 않습니다.

| 기능 | file:// | http:// (서버) |
|------|:-------:|:--------------:|
| CORS 요청 | X | O |
| ES 모듈 import | X | O |
| fetch API | X | O |
| Supabase SDK | X | O |
| Cookie 접근 | X | O |

---

## 실행 방법 비교

| 방법 | 설치 필요 | 자동 새로고침 | 명령어 |
|------|:--------:|:------------:|--------|
| npx serve | X | X | `npx serve 폴더명` |
| Python | X | X | `python -m http.server` |
| Live Server | 확장 설치 | O | VS Code에서 클릭 |

---

## 기본 명령어

```bash
# npx serve (권장) - 폴더와 포트 지정
npx serve Production/Frontend -l 3000

# 현재 폴더를 서버로 실행
npx serve .

# Python - 현재 폴더를 8000번 포트로
python -m http.server 8000

# Python - 특정 폴더 지정
python -m http.server -d Production/Frontend 3000
```

---

## 포트 충돌이란?

포트 번호는 한 번에 하나의 프로그램만 사용할 수 있습니다. 이미 3000번을 다른 프로그램이 쓰고 있으면 에러가 발생합니다.

```
Error: listen EADDRINUSE: address already in use :::3000
```

---

## 포트 충돌 해결

```bash
# 방법 1: 다른 포트 사용 (가장 간단)
npx serve -l 8080

# 방법 2: Windows에서 프로세스 종료
netstat -ano | findstr :3000    # PID 확인
taskkill /PID 12345 /F          # 해당 PID 종료

# 방법 3: Mac/Linux에서 프로세스 종료
lsof -i :3000                   # PID 확인
kill -9 12345                   # 해당 PID 종료
```

---

## 자주 쓰는 포트 번호

| 포트 | 용도 |
|------|------|
| 3000 | React, Node.js 기본 |
| 5173 | Vite 기본 |
| 8000 | Python 기본 |
| 8080 | 대체 포트 |
