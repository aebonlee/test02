# SQL 주석이란 무엇인가

> 코드 속에 사람을 위한 메모를 남기는 방법, 그것이 주석입니다.
> 컴퓨터는 무시하고 지나가지만, 나중에 코드를 보는 사람에게는 큰 도움이 됩니다.

---

## 주석이 뭔가요?

프로그래밍에서 주석(Comment)은 코드에 덧붙이는 설명입니다. 컴퓨터는 주석을 완전히 무시합니다. 실행되지 않습니다. 오직 코드를 읽는 사람만을 위한 것입니다.

비유하자면, 책에 포스트잇을 붙여두는 것과 같습니다. 책의 내용 자체는 바뀌지 않지만, 나중에 다시 볼 때 "여기가 중요해!"라고 표시해둔 포스트잇이 있으면 이해가 빠릅니다. 주석도 마찬가지입니다.

---

## SQL에서 주석 쓰는 법

SQL에서는 두 가지 방법으로 주석을 작성합니다.

첫 번째는 한 줄 주석입니다. 두 개의 대시(--)로 시작합니다. 대시 뒤에 쓴 모든 내용은 그 줄 끝까지 주석으로 처리됩니다.

```sql
-- 이것은 한 줄 주석입니다
SELECT * FROM users;  -- 모든 사용자 조회
```

두 번째는 여러 줄 주석입니다. `/*`로 시작해서 `*/`로 끝납니다. 그 사이의 모든 내용은 주석입니다. 여러 줄에 걸쳐 긴 설명을 쓸 때 유용합니다.

```sql
/*
이것은 여러 줄 주석입니다.
긴 설명을 쓸 때 사용합니다.
세 줄이든 열 줄이든 상관없습니다.
*/
SELECT * FROM users;
```

## 다른 프로그래밍 언어에서는?

주석은 모든 프로그래밍 언어에 있지만, 기호가 조금씩 다릅니다.

JavaScript나 Java, C 계열 언어에서는 슬래시 두 개(//)가 한 줄 주석입니다. 여러 줄 주석은 SQL과 같이 `/* */`를 사용합니다.

HTML에서는 `<!-- -->`를 사용합니다. 웹 페이지의 코드를 보면 이런 형태의 주석을 자주 볼 수 있습니다.

Python에서는 샵 기호(#)가 한 줄 주석입니다. 따옴표 세 개(""")로 감싸면 여러 줄 주석이 됩니다.

기호는 달라도 목적은 같습니다. 코드를 읽는 사람에게 설명을 남기는 것입니다.

---

## 왜 주석을 쓰나요?

주석을 쓰는 이유는 여러 가지입니다.

첫째, 코드의 목적을 설명합니다. 무슨 작업을 하는 코드인지 한눈에 알 수 있게 해줍니다. 복잡한 로직 앞에 "이 부분은 사용자 인증을 처리합니다"라고 적어두면 코드를 이해하기 훨씬 쉬워집니다.

둘째, 작성자와 날짜를 기록합니다. 누가 언제 이 코드를 작성했는지 남겨두면, 나중에 문제가 생겼을 때 담당자를 찾기 쉽습니다.

셋째, 나중에 해야 할 일을 표시합니다. TODO라는 표시를 많이 씁니다. "TODO: 에러 처리 추가 필요"처럼 적어두면 까먹지 않습니다.

넷째, 복잡한 로직을 설명합니다. 왜 이렇게 코드를 작성했는지, 어떤 의도로 이 방식을 선택했는지 적어둡니다.

---

## 실제 예시: SSALWorks 프로젝트에서

SSALWorks 프로젝트의 SQL 파일을 보면 주석이 체계적으로 사용되고 있습니다.

파일 상단에는 파일 정보가 있습니다. 작성일, 목적, 실행 순서 등을 적어둡니다. 이 SQL 파일이 무엇을 하는 파일인지 파일을 열자마자 알 수 있습니다.

각 섹션 앞에는 구분선과 제목이 있습니다. 긴 SQL 파일에서 원하는 부분을 빠르게 찾을 수 있게 해줍니다.

중요한 결정 사항에는 이유를 적어둡니다. 왜 이 컬럼에 UNIQUE 제약을 걸었는지, 왜 이 인덱스가 필요한지 설명합니다.

## 좋은 주석 vs 나쁜 주석

모든 주석이 좋은 것은 아닙니다. 좋은 주석과 나쁜 주석이 있습니다.

좋은 주석은 "왜"를 설명합니다. "한 사용자가 여러 계정을 가질 수 있으므로 닉네임 중복 허용"처럼 결정의 이유를 설명합니다.

나쁜 주석은 코드를 그대로 반복합니다. "사용자 조회"라고만 쓰고 그 아래에 `SELECT * FROM users`가 있다면, 이 주석은 별로 도움이 안 됩니다. 코드만 봐도 알 수 있는 내용이니까요.

가장 좋은 코드는 주석 없이도 이해되는 코드입니다. 변수명과 함수명을 명확하게 지으면 주석이 줄어듭니다. 하지만 복잡한 비즈니스 로직이나 특별한 결정 사항은 주석으로 남겨야 합니다.

---

## SSALWorks 방법론에서의 주석

SSALWorks 방법론에서는 주석을 문서화의 일부로 봅니다. 코드와 문서를 분리하지 않고, 코드 안에 설명을 함께 담습니다.

Claude Code가 SQL 파일을 생성할 때도 항상 주석을 포함합니다. 파일 상단에 목적과 날짜를 적고, 각 섹션마다 구분선을 넣습니다. 이렇게 하면 나중에 그 파일을 다시 열었을 때 무슨 파일인지 바로 알 수 있습니다.

비개발자인 Project Owner도 주석을 통해 코드의 의도를 파악할 수 있습니다. 완벽하게 이해하지 못해도, 대략 무슨 작업인지 짐작할 수 있습니다.

---

## 정리

주석은 코드 속의 메모입니다. 컴퓨터는 무시하지만 사람에게는 중요합니다.

SQL에서는 `--`로 한 줄 주석, `/* */`로 여러 줄 주석을 씁니다. 다른 언어에서는 `//`, `#`, `<!-- -->` 등을 사용합니다.

주석은 "무엇을" 보다 "왜"를 설명해야 합니다. 코드만 봐도 알 수 있는 내용은 주석으로 쓸 필요가 없습니다.

SSALWorks 방법론에서는 주석을 코드 문서화의 핵심으로 활용합니다. 파일마다 상단에 목적과 날짜를 적고, 중요한 결정에는 이유를 남깁니다.
