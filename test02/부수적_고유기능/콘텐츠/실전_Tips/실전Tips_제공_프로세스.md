# Tips 콘텐츠 작성 및 배포 프로세스

> **Claude Code 필독**: Tips 콘텐츠 작성 요청을 받으면 이 문서를 따라 MD 파일 작성부터 viewer.html 연동까지 완료하세요.

---

## 1. 전체 프로세스 요약 (자동화)

```
[1단계] 카테고리 확인
    ↓
[2단계] MD 파일 작성 (100줄, 설명식)
    ↓
[3단계] tips-list.json에 항목 추가
    ↓
[4단계] node sync-tips.js 실행 → 3개 파일 자동 업데이트
    ↓
[5단계] 완료 보고
```

**핵심**: `tips-list.json` 하나만 수정하면 스크립트가 3개 파일 자동 동기화!

## 3. 2단계: MD 파일 작성

### 3.1 저장 위치

```
부수적_고유기능/콘텐츠/Tips/{카테고리명}/{파일명}.md
```

**예시**:
```
부수적_고유기능/콘텐츠/Tips/웹개발_기초개념/풀스택_웹사이트란_무엇인가.md
```

### 3.2 파일명 규칙

- 한글 사용 가능
- 공백 대신 언더스코어(_) 사용
- 특수문자 최소화
- 내용을 알 수 있는 명확한 이름

### 3.3 콘텐츠 유형 (3가지)

Tips 콘텐츠는 3가지 유형으로 구분되며, 유형에 따라 분량과 작성 방식이 다릅니다.

| 유형 | 특징 | 제목 패턴 | 분량 |
|------|------|----------|------|
| **개념 설명형** | "~란 무엇인가" 형식, 개념 이해 목적 | 풀스택 웹사이트란, SQL 주석이란 | 80~100줄 |
| **방법 가이드형** | "~하는 방법" 형식, 단계별 안내 | 설치 방법, 설정 방법, 활용하기 | 100~120줄 |
| **문제 해결형** | "~문제 해결" 형식, 트러블슈팅 | 오류 해결, 복구하기 | 60~80줄 |

**유형별 작성 포인트:**

- **개념 설명형**: 비유와 예시를 풍부하게 사용. "레스토랑에 비유하면..."
- **방법 가이드형**: 단계별 흐름을 명확히. 하지만 번호 나열만 하지 말고 각 단계를 설명
- **문제 해결형**: 문제 상황 → 원인 → 해결책 순서로 간결하게

### 3.4 작성 양식 규칙 (필수 준수!)

#### 📏 분량: 유형별 권장 분량 준수

콘텐츠 유형에 따라 적정 분량이 다릅니다.

- **개념 설명형**: 80~100줄
- **방법 가이드형**: 100~120줄 (단계별 설명 필요)
- **문제 해결형**: 60~80줄 (핵심만 간결히)

너무 짧으면 내용이 부실하고, 너무 길면 읽기 어렵습니다.

#### 📝 스타일: 설명식 문장 위주

**표나 목록보다 문장으로 설명합니다.**

```
❌ 나쁜 예 (표 남용):
| 항목 | 설명 |
|------|------|
| 프론트엔드 | 화면 |
| 백엔드 | 서버 |

✅ 좋은 예 (설명식):
프론트엔드는 사용자가 보는 화면입니다.
백엔드는 보이지 않는 서버에서 데이터를 처리합니다.
```

#### 📊 표: 최소화

표는 꼭 필요한 경우에만 사용합니다. 비교가 필요하거나 정보를 한눈에 봐야 할 때만 표를 씁니다. 설명할 수 있으면 문장으로 씁니다.

#### 💻 코드블록: 최소화

비개발자 대상이므로 코드 블록은 가급적 피합니다. 꼭 필요한 경우에도 최소한으로만 사용합니다.

#### 🌾 SSALWorks 방법론 강조

Tips 콘텐츠는 SSALWorks 방법론의 가치를 전달해야 합니다. "Claude Code만으로는 부족하고, SSALWorks 방법론이 필요하다"는 메시지를 자연스럽게 포함합니다.

**강조 패턴 예시:**

- **차별화 패턴**: "Claude Code만으로는 ~가 어렵습니다. SSALWorks 방법론에서는..."
- **비교 패턴**: "일반적인 방법은 ~이지만, SSALWorks에서는..."
- **가치 패턴**: "SSALWorks Dev Package를 사용하면 이 모든 것이 자동으로..."
- **결론 연결**: 결론 섹션에서 "SSALWorks 방법론을 따르면..." 문구 포함

**구체적 문장 예시:**

```
"Claude Code는 강력한 도구이지만, 체계적인 프로젝트 관리 없이는
파일이 흩어지고 작업 이력을 추적하기 어렵습니다.
SSALWorks 방법론은 이 문제를 해결합니다."

"SSALWorks Dev Package에는 이 설정이 이미 포함되어 있어
별도 작업 없이 바로 사용할 수 있습니다."
```

#### 🎯 비유/예시 활용

추상적인 개념은 비유나 실생활 예시로 설명합니다. 레스토랑 비유, 건축 비유 등을 활용하면 비개발자도 쉽게 이해합니다.

## 4. 3단계: tips-list.json에 항목 추가

> **자동화**: JSON 파일 하나만 수정하면 스크립트가 3개 파일 자동 업데이트!

### 4.1 파일 위치

```
부수적_고유기능/콘텐츠/Tips/tips-list.json
```

### 4.2 기존 카테고리에 파일 추가

해당 카테고리의 `files` 배열에 추가:

```json
{
  "id": "웹개발_기초개념",
  "name": "웹개발 기초개념",
  "icon": "🌐",
  "description": "웹 개발의 기본 개념 이해",
  "files": [
    { "name": "풀스택 웹사이트란 무엇인가", "file": "풀스택_웹사이트란_무엇인가.md" },
    { "name": "새로운 팁 제목", "file": "새로운_팁.md" }
  ]
}
```

**필드 설명:**
- `name`: 뷰어에 표시될 제목
- `file`: 파일명만 (경로 제외, `.md` 포함)

### 4.3 새 카테고리 추가

`categories` 배열에 새 객체 추가:

```json
{
  "id": "새카테고리명",
  "name": "표시될 카테고리 이름",
  "icon": "🆕",
  "description": "카테고리 설명",
  "files": [
    { "name": "첫 번째 팁 제목", "file": "첫번째_팁.md" }
  ]
}
```

**⚠️ 주의:**
- `id`는 폴더명과 동일해야 함 (언더스코어 사용)
- `name`은 표시용 (공백, 한글 가능)

## 5. 4단계: 동기화 스크립트 실행

### 5.1 스크립트 실행

**프로젝트 루트에서 실행:**
```bash
node 부수적_고유기능/콘텐츠/Tips/sync-tips.js
```

**또는 Tips 폴더로 이동 후 실행:**
```bash
cd 부수적_고유기능/콘텐츠/Tips && node sync-tips.js
```

### 5.2 실행 결과

```

📝 파일 업데이트...

  ✅ 업데이트: tips-viewer.html
  ✅ 업데이트: index.html
```

### 5.3 자동 업데이트되는 파일

## 8. 시스템 구조 (참고)

```
GitHub 저장소
    ↓ (push)
jsdelivr CDN (자동 반영)
    ↓ (fetch)
viewer.html (Marked.js로 렌더링)
    ↓
사용자 화면
```

### 파일 관계

```
Tips/
├── tips-list.json       ← 단일 소스 (이것만 수정!)
├── sync-tips.js         ← 동기화 스크립트
├── viewer.html          ← 자동 업데이트됨
├── Tips_제공_프로세스.md ← 이 문서
└── {카테고리}/
    └── {파일}.md        ← 콘텐츠
```

### CDN URL 형식

```
https://cdn.jsdelivr.net/gh/SUNWOONGKYU/SSALWorks@master/부수적_고유기능/콘텐츠/Tips/{경로}
```
