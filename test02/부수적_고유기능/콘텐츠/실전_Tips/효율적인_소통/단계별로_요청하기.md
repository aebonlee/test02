# 큰 작업은 한 번에 시키지 말고 작은 단계로 나눠서 요청하기

> 이 문서는 큰 작업을 단계별로 나눠서 요청하여 품질을 높이고 문제를 즉시 대응하는 방법을 설명합니다.

---

## 왜 단계별로 요청하는가

한 번에 모든 것을 요청하면 중간에 문제가 생겨도 어디서 잘못됐는지 파악하기 어렵습니다. 단계별로 진행하면 각 단계에서 품질을 확인하고 문제를 즉시 해결할 수 있습니다.

---

## 한 번에 vs 단계별

**잘못된 방법:**
```
"회원가입, 로그인, 결제 다 만들어줘"
→ 중간에 문제 발생 시 어디서 잘못됐는지 파악 어려움
→ 나중에 전체를 다시 확인해야 함
```

**올바른 방법:**
```
"1단계: 회원가입 만들어줘" → 완료 확인
"2단계: 로그인 만들어줘" → 완료 확인
"3단계: 결제 만들어줘" → 완료 확인
→ 각 단계별 품질 보장
```

---

## 작업 분할 예시

```
전체: 로그인 기능

1단계: UI 구현 (폼, 버튼)
2단계: API 연동 (Supabase Auth)
3단계: 에러 핸들링
4단계: 테스트
```

---

## 요청 방법

```
"회원 관리 기능을 구현하려고 해.
다음 순서로 진행하자:
1. 회원가입 폼 UI
2. 회원가입 API 연동
3. 로그인 기능
4. 비밀번호 재설정

우선 1단계부터 시작해줘"
```

---

## 단계 완료 확인

```
"1단계 완료된 것 같아. 확인해줘:
- 모든 파일 저장됐어?
- 테스트 통과해?
- 다음 단계로 넘어가도 돼?"
```

---

## 문제 발생 시

```
"2단계에서 문제가 생겼어.
API 연동이 안 되는데, 먼저 이 문제 해결하고 진행하자."
```

---

## 단계 스킵하기

```
"4단계(테스트)는 나중에 하자.
먼저 3단계까지 완료하고 다른 기능으로 넘어가자."
```

---

## 진행 상황 요약 요청

```
"지금까지 뭘 완료했고 뭐가 남았는지 정리해줘"
```
