# 구현 방법이 여러 가지일 때 3가지 이상 대안 요청하기

> 이 문서는 Claude Code에게 여러 구현 방안을 요청하여 최선의 선택을 하는 방법을 설명합니다.

---

## 왜 대안을 요청하는가

| 이점 | 설명 |
|------|------|
| 더 나은 선택 | 비교 후 최적 방안 선택 |
| 학습 효과 | 다양한 접근법 이해 |
| 트레이드오프 파악 | 각 방법의 장단점 인식 |
| 유연성 확보 | 나중에 변경 시 참고 가능 |

---

## 요청 방법

```
"로그인 기능 구현 방법 3가지 제안해줘.
각각 장단점 설명해줘."
```

---

## 비교 표 요청

```
"3가지 방법을 표로 비교해줘:
- 구현 난이도
- 성능
- 유지보수성
- 비용"
```

---

## 대안 요청 예시

**인증 방식:**
```
"사용자 인증 방법 3가지:
1. 이메일/비밀번호
2. OAuth (Google)
3. Magic Link

각각 장단점 알려줘"
```

**데이터 저장:**
```
"상태 관리 방법 비교해줘:
1. React useState
2. Context API
3. Zustand
4. Redux"
```

---

## 선택 후 요청

```
"2번 방법(OAuth)으로 진행해줘"
"1번과 2번 조합해서 구현해줘"
```
