# AI 답변이 이해 안 되면 이해될 때까지 계속 질문하기

> 이 문서는 Claude Code의 답변을 완전히 이해할 때까지 계속 질문하여 학습 효과를 극대화하는 방법을 설명합니다.

---

## 왜 이해해야 하는가

이해 없이 "알겠어, 해줘"라고 하면 비슷한 상황에서 스스로 해결할 수 없습니다. 개념을 이해하면 응용할 수 있고, 다음에 같은 문제가 생겨도 스스로 해결할 수 있습니다.

---

## 이해 없이 vs 이해 후

**이해 없이:**
```
"알겠어 해줘"
→ 비슷한 상황에서 스스로 해결 못함
→ 같은 질문 반복
```

**이해 후:**
```
"useCallback이 뭐야? 왜 필요해?"
→ 개념 습득
→ 응용 가능
→ 스스로 해결 가능
```

---

## 질문 유형

| 유형 | 예시 질문 |
|------|----------|
| 왜 | "왜 이렇게 해야 해?" |
| 어떻게 | "이게 어떻게 동작해?" |
| 뭐야 | "이 용어가 뭐야?" |
| 재설명 | "더 쉽게 설명해줘" |
| 비교 | "A랑 B 차이가 뭐야?" |
| 예시 | "실제 예시 보여줘" |

---

## 질문 연속 예시

```
1차: "useEffect가 뭐야?"
2차: "의존성 배열이 뭐야?"
3차: "빈 배열이랑 배열 없는 거 뭐가 달라?"
4차: "언제 어떤 걸 써야 해? 예시 보여줘"
5차: "내가 이해한 게 맞는지 확인해줘"
```

---

## 이해 확인 방법

```
"내가 이해한 게 맞는지 확인해줘:
[이해한 내용]
맞아?"

"결론만 세 줄로 요약해줘"

"실제로 코드에 적용하면 어떻게 돼?"
```

---

## 뭘 물어야 할지 모를 때

```
"솔직히 뭘 모르는지도 모르겠어.
기초부터 다시 설명해줘"

"비개발자도 이해하게 쉽게 설명해줘"

"비유로 설명해줘"
```

---

## 질문하기 부끄러울 때

```
"기초적인 질문인데, 변수가 뭐야?"
"바보 같은 질문일 수 있는데, 왜 이렇게 해야 해?"
→ 기초 질문도 괜찮습니다. 이해하는 게 중요합니다.
```
