# 애매한 질문보다 상황, 기대 결과, 제한사항을 구체적으로 말하기

> 이 문서는 Claude Code에게 정확한 답변을 받기 위해 구체적으로 질문하는 방법을 설명합니다.

---

## 왜 구체적인 질문이 중요한가

"안 돼요"라고만 하면 AI가 추측해야 합니다. 상황, 시도한 것, 오류 메시지를 포함하면 정확한 원인을 파악하고 빠르게 해결할 수 있습니다.

---

## 좋은 질문의 구성 요소

| 요소 | 예시 |
|------|------|
| 상황 | "로그인 기능 구현하려고 했습니다" |
| 시도 | "onClick 이벤트 추가했습니다" |
| 결과 | "클릭해도 반응 없습니다" |
| 오류 | "콘솔에 TypeError 표시됩니다" |

---

## 질문 개선 예시

**나쁜 질문:**
```
"로그인 안 돼요"
```

**좋은 질문:**
```
"로그인 테스트 중입니다.
- 이메일: test@example.com 입력
- 버튼 클릭 시 '401 Unauthorized' 응답
- 콘솔에 'Invalid credentials' 메시지
원인이 무엇일까요?"
```

---

## 오류 메시지 공유 방법

```
"다음 오류가 발생합니다:

TypeError: Cannot read property 'map' of undefined
    at ProductList (ProductList.js:15:23)
    at renderWithHooks (react-dom.development.js:14985:18)"
```

---

## 질문 개선 공식

| 모호한 질문 | 구체적인 질문 |
|-------------|---------------|
| "안 돼요" | "A를 하려고 B를 시도했는데 C 오류 발생" |
| "이상해요" | "예상 결과는 A인데 실제로는 B가 나옵니다" |
| "왜 그래요" | "X 상황에서 Y를 했는데 Z가 됐어요. 왜 그럴까요?" |

---

## 스크린샷/파일 공유

```
"이 파일에서 15번째 줄에 에러가 나요:
[파일 경로]: Production/Frontend/login.js

에러 메시지:
[에러 내용 복사]"
```

---

## 질문 템플릿

```
"[문제 상황]

1. 하려고 한 것: [목표]
2. 시도한 방법: [시도한 내용]
3. 결과: [실제 결과]
4. 오류 메시지: [콘솔/터미널 출력]

어떻게 해결하면 될까요?"
```
