# 3편 | Claude 모델과 접근방식 개요

---

Claude를 사용하는 방법은 여러 가지가 있다. 웹 브라우저에서 간단히 대화할 수도 있고, 데스크톱 앱을 설치하거나, 개발 환경에 통합하거나, API로 자신의 서비스에 연결할 수도 있다. 이 편에서는 어떤 모델이 있고, 어떤 방식으로 Claude에 접근할 수 있는지 전체 그림을 살펴본다.

## 1. Claude 모델 체계

### 1-1 모델 등급 구조

Claude는 세 가지 등급의 모델을 제공한다. 각 등급은 성능, 속도, 비용이 다르므로 상황에 맞게 선택해야 한다.

**Opus (오푸스)**: 최상위 모델이다. 가장 뛰어난 추론 능력과 창의성을 제공하지만, 응답 속도가 상대적으로 느리고 비용이 높다. 복잡한 분석, 고난도 문제 해결, 전문적인 콘텐츠 생성에 적합하다.

**Sonnet (소네트)**: 중간 등급 모델이다. 성능과 속도, 비용의 균형이 좋아 가장 널리 사용된다. 일반적인 업무 대부분에 충분한 성능을 제공하면서도 빠른 응답이 가능하다.

**Haiku (하이쿠)**: 경량 모델이다. 가장 빠르고 저렴하다. 간단한 질문 응답, 텍스트 분류, 대량 처리 작업에 적합하다. 복잡한 추론이 필요하지 않은 경우 비용 효율적인 선택이다.

### 1-2 버전 번호 이해

모델명에는 버전 번호가 붙는다. 예를 들어 "Claude Sonnet 4.5"에서 4는 세대(generation)를, .5는 세부 버전을 의미한다.

- Claude 3 → Claude 3.5 → Claude 3.7: 같은 세대 내 점진적 개선
- Claude 3 → Claude 4: 새로운 세대, 큰 폭의 성능 향상

일반적으로 숫자가 높을수록 더 최신이고 더 뛰어난 성능을 제공한다. 2025년 12월 현재 최신 모델은 Claude Opus 4.5와 Claude Sonnet 4.5이다.

### 1-3 모델 선택 가이드

상황별 추천 모델은 다음과 같다:

| 작업 유형 | 추천 모델 | 이유 |
|----------|----------|------|
| 복잡한 분석/연구 | Opus | 최고 수준의 추론 능력 |
| 일반 업무/글쓰기 | Sonnet | 성능과 속도의 균형 |
| 코드 작성/리뷰 | Sonnet | 빠른 응답과 충분한 품질 |
| 간단한 질문 | Haiku | 빠르고 저렴 |
| 대량 텍스트 처리 | Haiku | 비용 효율성 |
| 창의적 콘텐츠 | Opus | 높은 창의성 |

확실하지 않다면 Sonnet으로 시작하는 것을 추천한다. 대부분의 작업에 충분하며, 필요에 따라 Opus나 Haiku로 전환할 수 있다.

## 2. 접근방식 5가지

Claude에 접근하는 방법은 크게 5가지로 나눌 수 있다. 각 방식은 고유한 장점과 적합한 사용 시나리오가 있다.

### 2-1 웹 + 모바일 앱

**웹 (claude.ai)**: 가장 기본적인 접근 방식이다. 웹 브라우저에서 claude.ai에 접속하면 즉시 Claude와 대화할 수 있다. 별도 설치가 필요 없고, 계정만 있으면 어디서든 사용 가능하다.

**모바일 앱 (iOS/Android)**: 스마트폰용 전용 앱이다. 이동 중에도 Claude를 사용할 수 있으며, 음성 입력, 카메라로 촬영한 이미지 분석 등 모바일 특화 기능을 제공한다.

웹과 모바일은 동일한 계정으로 동기화되어, 한쪽에서 시작한 대화를 다른 쪽에서 이어갈 수 있다.

**주요 기능**:
- 텍스트 대화
- 파일 업로드 및 분석 (PDF, 이미지 등)
- Projects 기능으로 관련 대화 그룹화
- Artifacts로 코드, 문서 등 실시간 생성

### 2-2 데스크톱 앱

Windows, macOS, Linux용 독립 애플리케이션이다. 웹 버전의 모든 기능에 더해 다음과 같은 추가 기능을 제공한다:

**키보드 단축키**: 전역 단축키로 어떤 작업 중에도 빠르게 Claude를 호출할 수 있다.

**MCP (Model Context Protocol)**: Claude가 사용자 컴퓨터의 파일, 다른 프로그램, 외부 서비스에 접근할 수 있게 하는 기능이다. 예를 들어 로컬 파일을 읽거나, GitHub 저장소에 접근하거나, 데이터베이스를 조회할 수 있다.

**오프라인 기능**: 인터넷 연결이 불안정해도 이전 대화 내용을 확인하고 준비할 수 있다.

데스크톱 앱은 Claude를 주력 도구로 사용하는 경우 추천한다.

### 2-3 Claude Code

터미널(명령줄)에서 실행하는 개발자 도구이다. 2025년 5월 정식 출시되어 빠르게 인기를 얻었다.

**핵심 특징**:
- 프로젝트 전체 코드베이스를 이해하고 작업
- 파일 읽기, 쓰기, 수정 직접 수행
- 터미널 명령어 실행
- Git 연동으로 버전 관리
- 테스트 작성 및 실행

Claude Code는 단순히 코드를 추천해주는 것이 아니라, 실제로 파일을 수정하고 명령을 실행한다. 개발자가 지시하면 Claude Code가 직접 작업을 수행하는 방식이다.

개발자뿐 아니라 비개발자도 Claude Code를 활용할 수 있다. 문서 작성, 데이터 정리, 파일 관리 등 다양한 작업을 터미널 환경에서 효율적으로 수행할 수 있다.

### 2-4 IDE 결합 (VS Code)

Visual Studio Code 같은 코드 편집기에 Claude를 통합하는 방식이다.

**GitHub Copilot 연동**: GitHub Copilot의 백엔드로 Claude를 선택하여 코드 자동완성, 제안 기능을 사용할 수 있다.

**확장 프로그램**: Claude를 VS Code 내에서 직접 호출하는 확장 프로그램들이 있다. 코드를 선택하고 Claude에게 설명을 요청하거나, 리팩토링을 요청할 수 있다.

**Claude Code + VS Code**: Claude Code와 VS Code를 동시에 사용하는 워크플로우도 있다. Claude Code가 파일을 수정하면 VS Code에서 실시간으로 변경 사항을 확인할 수 있다.

IDE 결합은 코딩 작업이 주 업무인 개발자에게 적합하다.

### 2-5 API 연결

자신의 프로그램이나 서비스에 Claude를 연결하는 방식이다.

**API란**: Application Programming Interface의 약자로, 프로그램들이 서로 통신하는 방법이다. Claude API를 사용하면 자신만의 앱이나 서비스에 Claude의 능력을 추가할 수 있다.

**활용 예시**:
- 고객 서비스 챗봇 구축
- 문서 자동 요약 시스템
- 콘텐츠 생성 자동화
- 데이터 분석 파이프라인

**비용 구조**: API는 사용한 만큼 비용을 지불하는 방식이다. 입력과 출력 토큰 수에 따라 요금이 계산된다. 모델별로 단가가 다르며, Haiku가 가장 저렴하고 Opus가 가장 비싸다.

API는 프로그래밍 지식이 필요하지만, Claude Code를 활용하면 비개발자도 API 연동 코드를 생성할 수 있다.

## 3. 접근방식 비교

| 접근방식 | 대상 사용자 | 설치 필요 | 주요 장점 |
|---------|-----------|----------|----------|
| 웹 | 모든 사용자 | 없음 | 즉시 사용 가능 |
| 모바일 앱 | 이동 중 사용자 | 앱 설치 | 음성 입력, 카메라 |
| 데스크톱 앱 | 파워 유저 | 앱 설치 | MCP, 단축키 |
| Claude Code | 개발자/파워 유저 | CLI 설치 | 직접 실행, 자동화 |
| IDE 결합 | 개발자 | 확장 설치 | 코딩 워크플로우 통합 |
| API | 개발자/기업 | 개발 필요 | 서비스 통합 |

## 4. 선택 가이드

### 4-1 처음 시작하는 경우

**웹 (claude.ai)**을 추천한다. 설치 없이 바로 사용할 수 있고, Claude의 기본 기능을 모두 경험할 수 있다. 웹에서 충분히 익숙해진 후 필요에 따라 다른 방식으로 확장하면 된다.

### 4-2 업무에 본격적으로 활용하는 경우

**데스크톱 앱**을 추천한다. 키보드 단축키로 빠르게 호출할 수 있고, MCP를 통해 로컬 파일이나 외부 서비스와 연동할 수 있다.

### 4-3 개발 작업이 주인 경우

**Claude Code**를 추천한다. 코드베이스 전체를 이해하고, 직접 파일을 수정하며, 터미널 명령을 실행할 수 있어 개발 생산성이 크게 향상된다.

### 4-4 자체 서비스를 만드는 경우

**API**를 사용해야 한다. 사용량에 따른 비용 관리가 가능하고, 자신의 서비스에 맞게 Claude를 커스터마이징할 수 있다.
