# 10편 | Claude 내부 기능

---

내부 기능은 Claude 모델이 태어날 때부터 가진 본질적 능력이다. 별도의 설정이나 활성화 없이 기본으로 작동한다. 이 편에서는 Claude가 무엇을 할 수 있는지, 그 핵심 능력들을 상세히 살펴본다.

## 1. 내부 기능이란

### 1-1 모델의 본질적 능력

내부 기능은 Claude 모델 자체에 내장된 능력이다. 학습 과정에서 습득한 것으로, 추가 도구나 외부 연결 없이도 작동한다. 텍스트를 이해하고 생성하는 것, 이미지를 분석하는 것, 논리적으로 추론하는 것 등이 여기에 해당한다.

사용자가 질문을 하면 Claude는 내부 기능만으로 답변을 생성한다. 웹 검색이나 외부 도구 없이도 대부분의 대화와 작업이 가능한 이유다.

### 1-2 내부 도구와의 차이

내부 기능과 내부 도구는 다르다.

| 구분 | 내부 기능 | 내부 도구 |
|------|----------|----------|
| 정의 | 모델 자체의 능력 | Anthropic이 추가로 제공하는 기능 |
| 활성화 | 항상 작동 | 선택적 사용 |
| 예시 | 텍스트 생성, 이미지 분석 | 웹 검색, Research, 심층 사고 |
| 플랜 제한 | 없음 | 일부 Pro 이상 |

내부 기능은 Claude의 "두뇌"라면, 내부 도구는 그 두뇌에 연결된 "추가 장비"라고 생각하면 된다.

## 2. 핵심 처리 능력

### 2-1 텍스트 이해

Claude는 자연어를 깊이 있게 이해한다. 단순히 단어를 인식하는 수준이 아니라, 문맥과 의도, 뉘앙스까지 파악한다.

**이해할 수 있는 것들:**
- 일상 대화와 질문
- 기술 문서와 논문
- 법률 문서와 계약서
- 문학 작품과 시
- 프로그래밍 코드
- 여러 언어 (한국어, 영어, 일본어, 중국어 등)

**문맥 이해 예시:**
"그거 괜찮아?"라는 질문에 대해 Claude는 이전 대화에서 무엇을 논의했는지에 따라 "그거"가 무엇인지 파악하고 적절히 답변한다.

### 2-2 텍스트 생성

이해한 내용을 바탕으로 새로운 텍스트를 생성한다. 단순 답변부터 긴 문서까지 다양한 형태의 글을 작성할 수 있다.

**생성할 수 있는 것들:**
- 이메일, 보고서, 기획서
- 블로그 글, 마케팅 카피
- 소설, 시, 대본
- 프로그래밍 코드
- 번역문
- 요약문

**스타일 조절:**
같은 내용이라도 요청에 따라 공식적으로, 친근하게, 간결하게, 상세하게 등 다양한 스타일로 작성할 수 있다.

### 2-3 분석과 추론

주어진 정보를 분석하고 논리적으로 추론하여 결론을 도출한다.

**분석 능력:**
- 데이터에서 패턴과 추세 발견
- 장단점 비교 분석
- 원인과 결과 파악
- 문제점 식별

**추론 능력:**
- 주어진 전제에서 논리적 결론 도출
- 가설 검증
- 시나리오별 결과 예측
- 의사결정 지원

복잡한 문제에 대해 단계별로 사고 과정을 보여주며 결론에 도달할 수 있다.

### 2-4 요약과 번역

긴 문서를 핵심만 추려 요약하거나, 한 언어에서 다른 언어로 번역한다.

**요약:**
- 긴 문서를 몇 문장으로 압축
- 핵심 포인트 추출
- 원하는 길이로 조절 가능

**번역:**
- 100개 이상의 언어 지원
- 단순 직역이 아닌 의역
- 문화적 맥락 고려
- 전문 용어 적절히 처리

## 3. 멀티모달 능력

Claude는 텍스트뿐만 아니라 다양한 형태의 정보를 처리할 수 있다.

### 3-1 이미지 분석

사진, 차트, 다이어그램, 스크린샷 등 이미지를 이해하고 설명한다.

**할 수 있는 것:**
- 이미지 내용 설명
- 차트/그래프 데이터 해석
- 다이어그램 구조 분석
- 텍스트 추출 (OCR)
- 이미지 속 문제 식별

**활용 예시:**
- "이 차트가 보여주는 추세를 설명해줘"
- "이 스크린샷에서 오류 메시지가 뭐야?"
- "이 사진 속 제품의 특징을 분석해줘"

※ Claude Code에서는 이미지를 직접 붙여넣기할 수 없고, 파일 경로로 제공해야 한다.

### 3-2 문서 분석

PDF, Word, Excel 등 다양한 형식의 문서를 읽고 분석한다.

**지원 형식:**
- PDF 문서
- Word 문서 (.docx)
- Excel 스프레드시트 (.xlsx)
- CSV 파일
- 텍스트 파일
- Markdown 파일

**할 수 있는 것:**
- 문서 내용 요약
- 특정 정보 추출
- 데이터 분석
- 문서 간 비교

### 3-3 코드 분석

프로그래밍 코드를 읽고 이해하며 분석한다.

**지원 언어:**
대부분의 프로그래밍 언어를 지원한다. JavaScript, Python, Java, C++, Go, Rust, Ruby, PHP, Swift, Kotlin 등.

**할 수 있는 것:**
- 코드 동작 설명
- 버그 발견 및 수정 제안
- 코드 리뷰
- 리팩토링 제안
- 성능 개선 제안
- 보안 취약점 식별

## 4. Memory 기능

### 4-1 세션 간 기억이란

기본적으로 Claude는 대화가 끝나면 그 내용을 잊는다. 새 대화를 시작하면 이전 대화에서 나눈 내용을 기억하지 못한다.

Memory 기능은 이 한계를 넘어 세션 간에 중요한 정보를 기억하게 한다. 사용자가 매번 같은 설명을 반복할 필요가 없어진다.

### 4-2 무엇을 기억하나

**기억하는 것:**
- 사용자 이름과 선호도
- 프로젝트 맥락
- 자주 사용하는 용어나 스타일
- 이전에 내린 결정사항
- 작업 진행 상황

**기억하지 않는 것:**
- 민감한 개인정보 (자동 필터링)
- 일회성 정보
- 사용자가 기억하지 말라고 요청한 것

사용자가 직접 "이건 기억해줘" 또는 "이건 잊어줘"라고 지시할 수도 있다.

### 4-3 접근방식별 지원

※ Memory 기능은 웹과 데스크톱 앱에서만 지원된다.

| 접근방식 | Memory 지원 | 대안 |
|----------|:-----------:|------|
| 웹 | ✅ | - |
| 데스크톱 | ✅ | - |
| 모바일 | ✅ | - |
| Claude Code | ❌ | CLAUDE.md 파일 |
| API | ❌ | 직접 구현 필요 |

Claude Code에서는 CLAUDE.md 파일이 Memory 역할을 대신한다. 프로젝트에 대한 정보, 작업 규칙, 선호도 등을 이 파일에 기록해두면 Claude Code가 매 세션마다 참조한다.

## 5. Context Compaction

### 5-1 긴 대화의 문제

Claude는 한 번에 처리할 수 있는 텍스트 양에 제한이 있다. 이를 "컨텍스트 윈도우"라고 한다. 대화가 길어지면 이 한계에 도달하게 된다.

과거에는 한계에 도달하면 오래된 대화 내용이 잘려나가 맥락을 잃는 문제가 있었다.

### 5-2 자동 요약 방식

Context Compaction은 이 문제를 해결한다. 대화가 길어지면 오래된 부분을 자동으로 요약하여 압축한다.

**작동 방식:**
1. 대화가 컨텍스트 한계에 근접
2. 오래된 대화 내용을 핵심만 남기고 요약
3. 요약본으로 교체하여 공간 확보
4. 새로운 대화 계속 가능

사용자가 직접 개입할 필요 없이 자동으로 처리된다.

### 5-3 맥락 유지 원리

단순히 앞부분을 잘라내는 것이 아니라, 대화의 핵심 맥락을 보존한다.

**보존되는 것:**
- 대화의 주제와 목적
- 중요한 결정사항
- 핵심 정보와 데이터
- 진행 중인 작업 상태

**압축되는 것:**
- 중복된 설명
- 부수적인 대화
- 이미 해결된 논의

덕분에 아주 긴 대화도 맥락을 잃지 않고 이어갈 수 있다.

## 6. 내부 기능의 한계

내부 기능만으로는 할 수 없는 것들이 있다. 이 한계를 넘으려면 내부 도구나 확장 생태계가 필요하다.

### 6-1 실시간 정보 없음

Claude의 지식은 학습 시점까지로 제한된다. 오늘의 뉴스, 현재 주가, 실시간 날씨 등은 알 수 없다.

**해결책:** 웹 검색 도구 사용 (11편에서 다룸)

### 6-2 외부 시스템 접근 불가

내부 기능만으로는 외부 데이터베이스, 클라우드 서비스, 로컬 파일 시스템 등에 접근할 수 없다.

**해결책:** MCP, Connectors 사용 (13편에서 다룸)

### 6-3 파일 생성 제한

Claude는 텍스트를 생성할 수 있지만, 그것을 파일로 저장하는 것은 접근방식에 따라 다르다.

| 접근방식 | 파일 생성 |
|----------|----------|
| 웹/모바일 | Artifacts로 미리보기, 다운로드 |
| 데스크톱 | Artifacts + MCP로 직접 저장 |
| Claude Code | 직접 파일 생성/수정 가능 |
| API | 애플리케이션에서 처리 |

Claude Code는 파일 시스템에 직접 접근할 수 있어 파일 생성과 수정이 자유롭다.

## 7. 정리

내부 기능은 Claude의 기본 능력이다. 별도 설정 없이 모든 접근방식에서 사용할 수 있다.

| 기능 | 설명 |
|------|------|
| 텍스트 이해/생성 | 자연어 처리의 핵심 |
| 분석과 추론 | 논리적 사고와 문제 해결 |
| 멀티모달 | 이미지, 문서, 코드 분석 |
| Memory | 세션 간 기억 (웹/데스크톱/모바일) |
| Context Compaction | 긴 대화 맥락 유지 |

다음 편에서는 이 기본 능력 위에 Anthropic이 추가로 제공하는 내부 도구들을 살펴본다.
