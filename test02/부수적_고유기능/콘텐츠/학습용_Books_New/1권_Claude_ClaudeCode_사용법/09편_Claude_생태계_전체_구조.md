# 9편 | Claude 생태계 전체 구조

---

지금까지 Part 1에서 Claude의 탄생과 철학을, Part 2에서 다양한 접근방식을 살펴보았다. 이제 Part 3에서는 관점을 바꿔 Claude 생태계 전체를 구성요소별로 정리한다. 개별 기능을 하나씩 배우는 것도 중요하지만, 전체 구조를 이해하면 각 기능의 역할과 관계가 명확해진다.

## 1. 왜 생태계 관점이 필요한가

### 1-1 개별 기능 vs 전체 구조

Claude를 처음 접하면 수많은 기능과 용어에 압도당하기 쉽다. 웹 검색, Research, MCP, Skills, Connectors, Projects, Artifacts... 이것들이 각각 무엇이고 어떻게 다른지 혼란스럽다.

개별 기능을 하나씩 배우는 접근법도 있지만, 먼저 전체 구조를 파악하면 각 기능이 어디에 속하는지, 왜 존재하는지 이해하기 쉬워진다. 마치 도시를 처음 방문했을 때 지도를 먼저 보는 것과 같다.

### 1-2 7가지 구성요소

Claude 생태계는 크게 7가지 구성요소로 나눌 수 있다.

| # | 구성요소 | 핵심 질문 | 설명 |
|---|----------|----------|------|
| 1 | 모델 | 누가 일하나? | Opus, Sonnet, Haiku |
| 2 | 접근방식 | 어디서 만나나? | 모바일, 웹, 데스크톱, CLI, IDE, API |
| 3 | 내부 기능 | 무엇을 할 수 있나? | 모델의 본질적 능력 |
| 4 | 내부 도구 | 무엇을 추가로 쓸 수 있나? | Anthropic이 제공하는 추가 기능 |
| 5 | 작업 환경 | 어디서 작업하나? | Projects, Artifacts |
| 6 | 확장 생태계 | 어떻게 확장하나? | MCP, Connectors, Skills, Plugins |
| 7 | 구독 및 요금 | 얼마를 내나? | Free, Pro, Max, Team, Enterprise, API |

이 7가지를 이해하면 Claude의 모든 기능을 체계적으로 파악할 수 있다.

## 2. 구성요소 1: 모델

Claude는 세 가지 모델 라인업을 제공한다. 각 모델은 성능과 속도, 비용의 균형점이 다르다.

| 모델 | 특징 | 적합한 용도 |
|------|------|------------|
| Opus | 최고 성능, 복잡한 추론 | 연구, 전략 수립, 고난도 코딩 |
| Sonnet | 균형잡힌 성능 | 일반 업무, 코딩, 분석 |
| Haiku | 빠른 속도, 저비용 | 간단한 질문, 대량 처리 |

2025년 현재 최신 버전은 Opus 4.5, Sonnet 4.5, Haiku 4.5이다. 모델에 대한 상세한 내용은 03편에서 다루었다.

## 3. 구성요소 2: 접근방식

Claude를 만나는 방법은 6가지가 있다.

| 접근방식 | 특징 | 상세 편 |
|----------|------|--------|
| 모바일 앱 | 이동 중 사용, 음성 입력 | 05편 |
| 웹 (claude.ai) | 브라우저에서 바로 사용 | 05편 |
| 데스크톱 앱 | MCP 연결, 로컬 파일 접근 | 06편 |
| Claude Code (CLI) | 터미널에서 개발 작업 | 07편 |
| IDE 결합 | VS Code 등과 통합 | 08편 |
| API | 자체 애플리케이션에 통합 | 08편 |

각 접근방식에 따라 사용 가능한 기능이 조금씩 다르다. Part 2에서 상세히 다루었으므로 여기서는 요약만 한다.

## 4. 구성요소 3: 내부 기능

내부 기능은 Claude 모델이 태어날 때부터 가진 본질적 능력이다. 별도의 설정이나 활성화 없이 모든 접근방식에서 동일하게 작동한다.

### 4-1 핵심 처리 능력

**텍스트 이해**: 문서, 코드, 대화 등 모든 형태의 텍스트를 이해한다.

**텍스트 생성**: 글쓰기, 번역, 요약, 코드 작성 등 다양한 형태의 텍스트를 생성한다.

**분석과 추론**: 데이터를 분석하고, 논리적 추론을 수행하며, 문제를 해결한다.

### 4-2 멀티모달 능력

**이미지 분석**: 사진, 차트, 다이어그램 등을 이해하고 설명한다.

**문서 분석**: PDF, Word, Excel 등 다양한 형식의 문서를 읽고 분석한다.

**코드 분석**: 프로그래밍 코드를 이해하고, 버그를 찾고, 개선점을 제안한다.

### 4-3 고급 기능

**Memory**: 세션 간에 중요한 정보를 기억한다. 사용자의 선호도, 프로젝트 맥락 등을 저장하여 다음 대화에서 활용한다.

**Context Compaction**: 긴 대화에서 이전 내용을 자동으로 요약하여 맥락을 유지하면서도 효율적으로 처리한다.

내부 기능에 대한 상세한 내용은 10편에서 다룬다.

## 5. 구성요소 4: 내부 도구

내부 도구는 Anthropic이 Claude 인터페이스에 추가로 내장한 기능이다. 사용자가 선택적으로 활성화하거나 사용할 수 있으며, 일부는 특정 플랜에서만 제공된다.

### 5-1 대화/검색 도구

**웹 검색**: 실시간으로 인터넷을 검색하여 최신 정보를 가져온다. Pro 플랜 이상에서 사용 가능하다.

**Research**: 주제에 대해 심층적으로 조사하고 종합적인 보고서를 작성한다.

**심층 사고 (Extended Thinking)**: 복잡한 문제에 대해 더 깊이 생각하고 단계별로 추론한다.

**스타일 (Styles)**: 응답의 톤과 형식을 설정한다. 공식적, 친근한, 간결한 등 다양한 스타일을 선택할 수 있다.

### 5-2 개발/작업 도구

**Slash Commands**: 자주 사용하는 작업을 단축 명령어로 실행한다.

**Hooks**: 특정 시점에 자동으로 실행되는 동작을 설정한다.

**Subagents**: 전문화된 하위 에이전트를 병렬로 실행하여 복잡한 작업을 분담한다.

**Anthropic Skills**: Excel, PowerPoint, Word, PDF 등을 생성하는 기본 제공 Skills이다.

**Checkpoints**: 작업 상태를 자동으로 저장하고 필요시 이전 상태로 복원한다.

**Desktop Extensions**: 로컬 파일 시스템, 스크린샷, 클립보드 등에 접근하는 기능이다.

내부 도구에 대한 상세한 내용은 11편과 12편에서 다룬다.

## 6. 구성요소 5: 작업 환경

작업 환경은 대화와 산출물을 조직하고 관리하는 공간이다.

### 6-1 Projects

관련된 대화들을 하나의 프로젝트로 묶어서 관리한다. 프로젝트에 파일을 추가하면 해당 프로젝트의 모든 대화에서 그 파일들을 참조할 수 있다. 프로젝트별로 기본 지시사항을 설정할 수도 있다.

### 6-2 Artifacts

Claude가 생성한 코드, 문서, 차트 등을 별도의 패널에서 표시하고 편집할 수 있는 공간이다. 단순히 텍스트로 보여주는 대신 실제로 실행되거나 렌더링된 형태로 표시된다.

작업 환경에 대한 상세한 내용은 13편에서 다룬다.

## 7. 구성요소 6: 확장 생태계

확장 생태계는 외부에서 Claude의 능력을 확장하는 시스템이다. Anthropic뿐만 아니라 파트너사나 사용자도 확장 기능을 만들 수 있다.

### 7-1 MCP (Model Context Protocol)

외부 도구와 데이터 소스를 Claude에 연결하는 개방형 프로토콜이다. 2024년에 출시되었고, 2025년 12월에 Linux Foundation에 기부되어 업계 표준으로 자리잡고 있다.

### 7-2 Connectors/Integrations

Notion, Canva, Stripe, Jira 등 외부 클라우드 서비스와 원클릭으로 연결한다. claude.ai/directory에서 사용 가능한 Connectors를 확인하고 연결할 수 있다.

### 7-3 외부 Skills

Skills는 재사용 가능한 지시사항, 스크립트, 리소스의 패키지이다. Anthropic이 기본 제공하는 Skills 외에도 파트너사나 사용자가 직접 만들 수 있다. 2025년 12월에 Open Standard로 공개되었다.

### 7-4 Plugins

Claude Code 전용으로, Slash Commands, Subagents, MCP Servers, Hooks를 하나의 패키지로 묶어 배포하는 방식이다.

확장 생태계에 대한 상세한 내용은 13편에서 다룬다.

## 8. 구성요소 7: 구독 및 요금

Claude는 6가지 플랜을 제공한다.

| 플랜 | 월 요금 | 대상 | 주요 특징 |
|------|--------|------|----------|
| Free | $0 | 누구나 | 기본 기능, 제한된 사용량 |
| Pro | $20 | 개인 | 웹 검색, Projects, 더 많은 사용량 |
| Max 5x | $100 | 헤비 유저 | Pro의 5배 사용량 |
| Max 20x | $200 | 파워 유저 | Pro의 20배 사용량 |
| Team | $30/인 | 팀 | 팀 협업 기능, 관리 도구 |
| Enterprise | 문의 | 기업 | 맞춤형 보안, 규모 |

API는 별도의 종량제 요금이 적용된다. 모델별로 입력 토큰과 출력 토큰에 대한 가격이 다르다.

구독 및 요금에 대한 상세한 내용은 14편에서 다룬다.

## 9. 구성요소 간 관계

7가지 구성요소는 서로 연결되어 작동한다.

```
[모델] ─────────────────────────────────────────────────────
   │
   ├─ [내부 기능] ── 모델의 본질적 능력
   │
   ├─ [내부 도구] ── Anthropic이 추가한 기능
   │
   ├─ [작업 환경] ── 작업 공간 (Projects, Artifacts)
   │
   └─ [확장 생태계] ── 외부 연결 (MCP, Connectors, Skills)

[접근방식] ─── 모델을 만나는 창구 (웹, 앱, CLI 등)

[구독/요금] ─── 사용 범위 결정
```

**모델**이 핵심이고, 그 위에 **내부 기능**이 기본으로 탑재되어 있다. **내부 도구**는 Anthropic이 추가로 제공하는 것이고, **확장 생태계**는 외부에서 연결하는 것이다. **작업 환경**은 이 모든 것을 조직하는 공간이다. **접근방식**은 이 생태계를 만나는 창구이고, **구독/요금**은 어디까지 사용할 수 있는지를 결정한다.
