# 2편 | Claude의 탄생과 진화

---

Claude는 2023년 첫 등장 이후 빠르게 발전해왔다. 단순한 대화형 AI에서 시작하여, 이제는 복잡한 업무를 자율적으로 수행하는 에이전트 수준에 도달했다. 이 편에서는 Claude가 어떻게 탄생했고, 어떤 과정을 거쳐 현재의 모습이 되었는지 살펴본다.

## 1. Claude 1세대 (2023년)

### 1-1 첫 등장

Claude 1.0은 2023년 3월에 처음 공개되었다. ChatGPT가 2022년 11월에 등장하여 AI 대화 서비스 시장을 열었고, 약 4개월 뒤 Anthropic이 Claude로 경쟁에 뛰어든 것이다.

초기 Claude의 특징은 "정직함"이었다. 다른 AI들이 사용자가 원하는 답변을 하려고 노력하는 동안, Claude는 모르는 것은 모른다고 말하고, 확실하지 않은 정보는 그렇다고 밝히는 경향이 강했다. 이는 Anthropic의 Constitutional AI 철학이 반영된 결과였다.

### 1-2 Claude 2의 발전

2023년 7월, Claude 2가 출시되었다. 주요 개선 사항은 다음과 같았다:

**컨텍스트 확장**: 처리할 수 있는 텍스트 양이 크게 늘어났다. 이전에는 짧은 문서만 다룰 수 있었지만, Claude 2는 긴 보고서나 책 한 권 분량의 텍스트도 한 번에 분석할 수 있게 되었다.

**코딩 능력 향상**: 프로그래밍 관련 질문에 대한 답변 품질이 크게 개선되었다. 코드 작성, 디버깅, 설명 모두 이전보다 정확해졌다.

**할루시네이션 감소**: AI가 사실이 아닌 정보를 그럴듯하게 답변하는 문제가 절반 수준으로 줄었다.

2023년 11월에는 Claude 2.1이 출시되어 200K 토큰의 컨텍스트 윈도우를 지원하게 되었다. 토큰은 AI가 텍스트를 처리하는 단위로, 200K 토큰은 대략 15만 단어, 즉 500페이지 분량의 책을 한 번에 읽을 수 있다는 의미이다.

## 2. Claude 3 패밀리 (2024년)

### 2-1 세 가지 모델의 등장

2024년 3월, Anthropic은 Claude 3 패밀리를 발표하며 큰 변화를 가져왔다. 하나의 모델이 아닌, 용도에 따라 선택할 수 있는 세 가지 모델을 동시에 출시한 것이다.

**Claude 3 Opus**: 가장 강력한 모델이다. 복잡한 분석, 고난도 추론, 창의적 작업에 최적화되어 있다. 처리 속도는 상대적으로 느리고 비용도 높지만, 최고 수준의 품질이 필요할 때 사용한다.

**Claude 3 Sonnet**: 성능과 속도의 균형을 추구하는 모델이다. 대부분의 업무에 적합하며, 기업 환경에서 가장 많이 사용된다. Opus보다 빠르고 저렴하면서도 충분히 뛰어난 성능을 제공한다.

**Claude 3 Haiku**: 가장 빠르고 가벼운 모델이다. 간단한 질문 응답, 빠른 분류 작업, 대량 처리가 필요한 경우에 적합하다. 비용 효율성이 가장 높다.

모델 이름은 모두 시의 형식에서 따왔다. Opus(오푸스)는 대작을 의미하고, Sonnet(소네트)은 14행으로 구성된 정형시, Haiku(하이쿠)는 짧은 일본 시를 뜻한다.

### 2-2 Claude 3.5 Sonnet의 등장

2024년 6월, Claude 3.5 Sonnet이 출시되었다. 이 모델은 Claude 3 Opus를 능가하는 성능을 보여주면서도 Sonnet의 속도와 가격을 유지했다. 즉, 더 싸고 빠른 모델이 더 비싸고 느린 모델보다 뛰어난 성능을 보인 것이다.

이 시점에서 Anthropic은 중요한 결정을 내렸다. 무조건 큰 모델이 좋은 것이 아니라, 효율적인 모델이 실용적이라는 것을 보여준 것이다. Claude 3.5 Sonnet은 출시 직후 가장 인기 있는 AI 모델 중 하나가 되었다.

함께 출시된 Artifacts 기능도 주목할 만하다. 대화 중에 코드, 문서, 차트 등을 별도의 창에서 실시간으로 생성하고 수정할 수 있는 기능으로, AI와의 협업 방식을 크게 개선했다.

## 3. Claude 4 시대 (2025년)

### 3-1 Claude 4의 출시

2025년 5월 22일, Anthropic은 Claude 4를 공식 출시했다. Claude Opus 4와 Claude Sonnet 4 두 가지 모델이 동시에 발표되었다.

Claude 4의 가장 큰 특징은 "에이전트 능력"이다. 단순히 질문에 답하는 것을 넘어, 복잡한 작업을 자율적으로 수행할 수 있게 되었다. 고객 테스트에서 Claude Opus 4는 사람의 개입 없이 7시간 동안 연속으로 작업을 수행하면서 일관된 성능을 유지했다.

코딩 능력도 크게 향상되었다. 특히 Claude Code라는 터미널 기반 도구가 함께 출시되어, 개발자들이 코드 작성, 디버깅, 리팩토링을 AI와 함께 수행할 수 있게 되었다.

### 3-2 확장 사고 모드

Claude 3.7 Sonnet(2025년 2월)부터 도입된 "확장 사고(Extended Thinking)" 모드는 Claude 4에서 더욱 발전했다.

일반 모드에서 Claude는 빠르게 답변한다. 대부분의 일상적인 질문에는 이것으로 충분하다. 그러나 복잡한 수학 문제, 논리적 추론, 전략적 분석이 필요한 경우에는 확장 사고 모드를 사용할 수 있다.

확장 사고 모드에서 Claude는 답변하기 전에 문제를 여러 각도에서 분석하고, 자신의 추론 과정을 점검하며, 더 깊이 있는 결론을 도출한다. 시간은 더 걸리지만, 복잡한 문제에서 정확도가 크게 향상된다.

### 3-3 최신 모델 현황 (2025년 12월)

2025년 하반기에도 빠른 발전이 이어졌다:

**Claude Opus 4.1** (2025년 8월): 에이전트 작업, 실제 코딩, 추론 능력이 강화된 업그레이드 버전이다.

**Claude Sonnet 4.5** (2025년 9월): 코딩, 에이전트 작업, 컴퓨터 사용에서 당시 최고 성능을 기록했다.

**Claude Opus 4.5** (2025년 11월): 현재 가장 최신 모델이다. 전반적인 성능이 크게 향상되었으며, 특히 복잡한 추론과 창의적 작업에서 뛰어난 결과를 보여준다.

## 4. 진화의 방향성

### 4-1 크기보다 효율

Claude의 발전 과정에서 주목할 점은 "무조건 큰 모델"을 추구하지 않는다는 것이다. Claude 3.5 Sonnet이 Claude 3 Opus를 능가한 사례처럼, Anthropic은 모델의 크기보다 효율성과 실용성을 중시한다.

이 철학은 실제 수치로도 확인된다. Claude 3.5 Sonnet은 Claude 3 Opus 대비 응답 속도가 2배 빠르면서도 대부분의 벤치마크에서 더 높은 점수를 기록했다. 사용자 입장에서는 더 빠르고, 더 저렴하고, 더 뛰어난 모델을 사용할 수 있게 된 것이다.

### 4-2 안전성과 성능의 동시 추구

모델이 발전할수록 안전 장치도 함께 강화되었다. Claude 4는 이전 버전보다 강력하지만, 동시에 더 신중하고 정직하게 답변한다. 유해한 콘텐츠 생성을 거부하는 능력도 향상되었다.

Anthropic은 모델 출시 전 광범위한 레드팀 테스트를 수행한다. 외부 전문가들이 모델의 취약점을 찾아내고, 발견된 문제를 수정한 뒤에야 공개한다. 이러한 과정이 출시 속도를 늦출 수 있지만, Anthropic은 안전성을 타협하지 않는다.

### 4-3 에이전트로의 진화

단순 대화에서 시작하여, 이제 Claude는 복잡한 작업을 자율적으로 수행하는 에이전트로 발전하고 있다. 파일 읽기, 코드 실행, 웹 검색 등 다양한 도구를 활용하여 실제 업무를 처리할 수 있다.

Claude Code가 대표적인 예이다. 개발자가 "이 버그를 수정해줘"라고 요청하면, Claude Code는 코드베이스를 분석하고, 문제를 찾아내고, 수정 코드를 작성하고, 테스트까지 실행한다. 사람이 할 일은 결과를 확인하고 승인하는 것뿐이다.

### 4-4 사용자 피드백 반영

Anthropic은 사용자 피드백을 적극적으로 반영한다. Claude 3.5 Sonnet 출시 후 개발자들이 "코딩 작업에서 더 적극적으로 행동해달라"는 요청이 많았다. 이를 반영하여 Claude 4에서는 파일 수정, 명령 실행 등 더 능동적인 작업이 가능해졌다.
