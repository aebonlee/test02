# 13편 | 작업 환경과 확장 생태계

---

이전 편까지 Claude의 내부 기능과 내부 도구를 살펴보았다. 이번 편에서는 작업을 조직하는 공간인 작업 환경과, 외부에서 Claude의 능력을 확장하는 확장 생태계를 다룬다.

## 1. 개요

### 1-1 작업 환경이란

작업 환경은 대화와 산출물을 조직하고 관리하는 공간이다. 여러 대화를 하나의 프로젝트로 묶거나, 생성된 콘텐츠를 별도 패널에서 관리할 수 있다.

- **Projects**: 관련 대화들을 묶어 관리
- **Artifacts**: 생성된 콘텐츠를 별도 공간에서 표시

### 1-2 확장 생태계란

확장 생태계는 외부에서 Claude의 능력을 확장하는 시스템이다. Anthropic뿐 아니라 파트너사나 사용자도 확장 기능을 만들 수 있다.

- **MCP**: 외부 도구와 데이터 소스 연결
- **Connectors**: 클라우드 서비스 연결
- **외부 Skills**: 파트너/사용자가 만든 Skills
- **Plugins**: Claude Code용 확장 패키지

### 1-3 내부 도구와의 차이

| 구분 | 내부 도구 | 작업 환경 | 확장 생태계 |
|------|----------|----------|-------------|
| 제공자 | Anthropic | Anthropic | Anthropic + 외부 |
| 목적 | 기능 추가 | 작업 조직 | 외부 연결 |
| 예시 | 웹 검색, Commands | Projects, Artifacts | MCP, Connectors |

## 2. 작업 환경

### 2-1 Projects

**Projects란**

Projects는 관련된 대화들을 하나로 묶어 관리하는 공간이다. 웹사이트 개발 프로젝트라면 기획, 디자인, 개발 관련 대화를 모두 한 프로젝트에 담을 수 있다.

프로젝트에 파일을 추가하면 해당 프로젝트의 모든 대화에서 그 파일들을 참조할 수 있다. 매번 같은 파일을 업로드할 필요가 없다.

**프로젝트 생성과 관리**

프로젝트 생성은 간단하다:
1. 새 프로젝트 생성
2. 프로젝트 이름 지정
3. 필요한 파일 추가
4. 대화 시작

프로젝트 내에서 여러 대화를 생성하고, 각 대화는 프로젝트의 맥락을 공유한다.

**파일 첨부와 참조**

프로젝트에 첨부할 수 있는 파일:
- 문서 파일 (PDF, Word, 텍스트)
- 코드 파일 (모든 프로그래밍 언어)
- 이미지 파일
- 데이터 파일 (CSV, JSON)

첨부된 파일은 프로젝트의 모든 대화에서 자동으로 참조된다. "첨부한 기획서를 바탕으로..."라고 말하면 Claude가 해당 파일을 찾아 활용한다.

**프로젝트 지시사항 설정**

프로젝트별로 기본 지시사항을 설정할 수 있다. 모든 대화에 적용되는 규칙이나 스타일을 미리 정의해두는 것이다.

예시:
- "이 프로젝트의 코드는 TypeScript로 작성해줘"
- "답변은 항상 한국어로 해줘"
- "마케팅 팀을 위한 프로젝트야, 비기술적으로 설명해줘"

※ Projects는 웹과 데스크톱 앱에서 사용할 수 있다.

### 2-2 Artifacts

**Artifacts란**

Artifacts는 Claude가 생성한 코드, 문서, 차트 등을 별도의 패널에서 표시하는 공간이다. 대화 내용과 분리되어, 생성된 콘텐츠를 독립적으로 확인하고 편집할 수 있다.

**생성되는 콘텐츠 유형**

Artifacts에 표시되는 콘텐츠:

| 유형 | 예시 |
|------|------|
| 코드 | JavaScript, Python, HTML/CSS |
| 문서 | Markdown, 텍스트 문서 |
| 다이어그램 | Mermaid 차트, 플로우차트 |
| 시각화 | SVG 그래픽, 차트 |
| 웹 컴포넌트 | React 컴포넌트 미리보기 |

**실시간 미리보기**

Artifacts의 강력한 점은 실시간 미리보기다. 코드를 작성하면 바로 실행 결과를 볼 수 있다.

- HTML/CSS → 렌더링된 웹페이지
- React 컴포넌트 → 실제 동작하는 UI
- Mermaid 코드 → 시각화된 다이어그램
- SVG 코드 → 그래픽 이미지

**편집과 다운로드**

Artifacts 패널에서 직접 편집할 수 있다:
- 코드 수정 후 즉시 결과 확인
- 수정한 내용으로 대화 계속
- 완성된 결과물 다운로드

※ Artifacts는 웹과 데스크톱 앱에서 사용할 수 있다.

## 3. 확장 생태계

### 3-1 MCP (Model Context Protocol)

**MCP란**

MCP(Model Context Protocol)는 외부 도구와 데이터 소스를 Claude에 연결하는 개방형 프로토콜이다. 2024년에 출시되었고, 2025년 12월에 Linux Foundation에 기부되어 업계 표준으로 자리잡고 있다.

쉽게 말해, Claude가 외부 세계와 소통할 수 있는 "번역기"다. 데이터베이스, API, 파일 시스템 등 다양한 외부 자원에 접근할 수 있게 해준다.

**MCP 서버 개념**

MCP는 서버-클라이언트 구조로 작동한다:
- **MCP 서버**: 외부 도구를 Claude가 이해할 수 있게 변환
- **MCP 클라이언트**: Claude가 MCP 서버와 통신

예를 들어, Supabase MCP 서버를 연결하면 Claude가 데이터베이스를 직접 조회하고 수정할 수 있다.

**연결 방법**

MCP 서버 연결 단계:
1. 원하는 MCP 서버 선택
2. 설정 파일에 서버 정보 추가
3. 인증 정보 설정 (필요시)
4. Claude 재시작

**활용 예시**

| MCP 서버 | 기능 |
|----------|------|
| Supabase MCP | 데이터베이스 조회/수정 |
| GitHub MCP | 저장소 관리 |
| Slack MCP | 메시지 전송/조회 |
| Filesystem MCP | 로컬 파일 접근 |

※ MCP는 Claude Code와 데스크톱 앱에서 사용할 수 있다.

### 3-2 Connectors (Integrations)

**Connectors란**

Connectors는 외부 클라우드 서비스와 원클릭으로 연결하는 기능이다. MCP보다 설정이 간단하다. claude.ai/integrations에서 사용 가능한 서비스를 확인하고 바로 연결할 수 있다.

**사용 가능한 서비스**

주요 Connectors:

| 서비스 | 기능 |
|--------|------|
| Google Drive | 문서 읽기/쓰기 |
| Notion | 페이지 접근 |
| Slack | 채널 연동 |
| Jira | 이슈 관리 |
| Canva | 디자인 작업 |

**연결 방법**

Connectors 연결은 간단하다:
1. claude.ai/integrations 접속
2. 원하는 서비스 선택
3. 서비스 계정으로 로그인
4. 권한 승인
5. 연결 완료

복잡한 설정 없이 계정 인증만으로 연결된다.

**활용 예시**

- Google Drive 연결 → "드라이브의 기획서를 분석해줘"
- Notion 연결 → "노션 페이지를 요약해줘"
- Slack 연결 → "이 내용을 팀 채널에 공유해줘"

### 3-3 외부 Skills

**외부 Skills란**

외부 Skills는 Anthropic이 아닌 파트너사나 사용자가 만든 Skills다. 12편에서 다룬 Anthropic Skills와 달리 별도로 설치해야 사용할 수 있다.

**Anthropic Skills와의 차이**

| 구분 | Anthropic Skills | 외부 Skills |
|------|------------------|-------------|
| 제공자 | Anthropic | 파트너/사용자 |
| 설치 | 기본 제공 | 별도 설치 |
| 검증 | Anthropic 검증 | 제공자 책임 |
| 예시 | Excel, PDF | 특화 도구 |

2025년 12월에 Skills가 Open Standard로 공개되면서, 누구나 자신만의 Skills를 만들고 공유할 수 있게 되었다.

**Skills 설치 방법**

외부 Skills 설치:
1. Skills 저장소에서 원하는 Skills 찾기
2. 설치 명령 실행
3. 필요한 설정 완료
4. Skills 사용 시작

**활용 예시**

외부 Skills로 가능한 작업:
- 특정 업계 전문 문서 생성
- 맞춤형 분석 도구
- 자동화된 워크플로우

### 3-4 Plugins

**Plugins란**

Plugins는 Claude Code 전용 확장 패키지다. Slash Commands, Subagents, MCP Servers, Hooks를 하나로 묶어 배포하는 방식이다.

여러 도구를 개별로 설정하는 대신, Plugin 하나를 설치하면 관련 기능이 모두 설정된다.

**Plugin 구성 요소**

하나의 Plugin에 포함될 수 있는 것:

| 구성 요소 | 역할 |
|-----------|------|
| Slash Commands | 단축 명령어 |
| Subagents | 전문 에이전트 |
| MCP Servers | 외부 연결 |
| Hooks | 자동화 규칙 |

예를 들어, "웹 개발 Plugin"은 프론트엔드/백엔드 Subagents, 배포 Commands, Git MCP 등을 한 번에 설치한다.

**Plugin 설치 방법**

Plugin 설치:
1. Plugin 저장소에서 선택
2. 설치 명령 실행
3. 필요한 인증 설정
4. 바로 사용 시작

**활용 예시**

유용한 Plugin 유형:
- **개발 Plugin**: Git, 테스트, 배포 자동화
- **문서 Plugin**: 기술 문서 작성 도구
- **분석 Plugin**: 데이터 분석 도구 모음

※ Plugins는 Claude Code에서 사용할 수 있다.

작업 환경과 확장 생태계는 Claude를 더 강력하고 유연하게 만든다.
