# 14편 | 구독 플랜과 요금 체계

---

Claude를 효과적으로 사용하려면 요금 체계를 이해해야 한다. 어떤 플랜을 선택할지, 비용을 어떻게 최적화할지 알면 더 경제적으로 Claude를 활용할 수 있다.

## 1. 개요

### 1-1 Claude 요금 체계 구조

Claude의 요금 체계는 크게 두 가지로 나뉜다.

**구독 플랜**: 월정액을 내고 일정량을 사용하는 방식이다. Free, Pro, Max, Team, Enterprise 플랜이 있다.

**API 요금**: 사용한 만큼 지불하는 종량제 방식이다. 토큰 단위로 과금된다.

대부분의 일반 사용자는 구독 플랜을, 자체 애플리케이션을 개발하는 경우 API를 사용한다.

### 1-2 플랜 선택 기준

플랜 선택 시 고려할 사항:
- 사용 빈도: 얼마나 자주 사용하는가?
- 사용 목적: 개인용인가, 팀/기업용인가?
- 필요 기능: 웹 검색, Research 등이 필요한가?
- 예산: 월 얼마까지 지출할 수 있는가?

## 2. 개인용 플랜

### 2-1 Free 플랜

**월 요금**: 무료

**포함 기능**:
- Claude Sonnet 모델 사용
- 기본 대화 기능
- 이미지/문서 분석
- Artifacts

**사용량 제한**:
- 일일 메시지 수 제한
- 사용량 많을 때 속도 제한

**적합한 사용자**:
- Claude를 처음 접하는 사람
- 가끔 간단한 질문만 하는 사람
- 유료 플랜 전 테스트하려는 사람

### 2-2 Pro 플랜 ($20/월)

**Free 대비 추가 기능**:
- Claude Opus 모델 사용 가능
- 웹 검색 기능
- Research 기능
- Projects 기능
- 더 긴 컨텍스트 윈도우

**사용량 확대**:
- 월간 토큰 할당량: 500만 토큰 (Sonnet 기준)
- 5시간당 약 45개 메시지

**5시간 리셋 시스템**:
Pro 플랜은 5시간마다 사용량이 초기화된다. 5시간 내 제한량을 모두 사용하면 다음 5시간까지 대기해야 한다.

**적합한 사용자**:
- 업무에 Claude를 활용하는 직장인
- 웹 검색/Research가 필요한 사람
- 프로젝트 단위로 작업을 관리하고 싶은 사람

### 2-3 Max 플랜

Pro보다 더 많은 사용량이 필요한 헤비 유저를 위한 플랜이다.

**Max 5x ($100/월)**:
- 월간 토큰 할당량: 2,500만 토큰 (Sonnet 기준)
- 5시간당 약 225개 메시지
- Pro의 5배 사용량

**Max 20x ($200/월)**:
- 월간 토큰 할당량: 1억 토큰 (Sonnet 기준)
- 5시간당 약 900개 메시지
- Pro의 20배 사용량

**적합한 사용자**:
- 하루 종일 Claude와 작업하는 개발자
- 대량의 문서를 처리해야 하는 사람
- Pro 플랜에서 자주 제한에 걸리는 사람

## 3. 팀/기업용 플랜

### 3-1 Team 플랜 ($30/인/월)

**팀 협업 기능**:
- 팀원 간 대화/프로젝트 공유
- 공유 Projects
- 팀 단위 사용량 관리

**관리 도구**:
- 관리자 대시보드
- 사용자별 권한 설정
- 사용량 모니터링
- 예산 알림 설정

**사용량**:
- 월간 토큰 할당량: 1,000만 토큰/인 (Sonnet 기준)
- 5시간 제한 없음 (관리자 설정에 따름)

**적합한 조직**:
- 5~50명 규모의 팀
- 협업이 필요한 프로젝트 팀
- 중앙에서 비용을 관리하고 싶은 조직

### 3-2 Enterprise 플랜

**맞춤형 기능**:
- 맞춤형 토큰 할당량
- 전용 인프라
- 맞춤형 모델 파인튜닝
- 우선 지원

**보안 및 규정 준수**:
- SSO (Single Sign-On)
- SAML 인증
- SOC 2 Type II 인증
- HIPAA 준수 (의료 분야)
- 데이터 처리 계약 (DPA)

**도입 절차**:
1. Anthropic 영업팀 문의
2. 요구사항 협의
3. 맞춤형 견적
4. 계약 및 도입

## 4. API 요금

### 4-1 토큰 기반 과금 구조

API는 토큰 단위로 과금된다. 토큰은 AI가 텍스트를 처리하는 최소 단위로, 대략 영어 3-5글자 또는 한글 2-3글자에 해당한다.

**입력 토큰과 출력 토큰**:
- 입력 토큰: 사용자가 보내는 텍스트 (프롬프트, 문서 등)
- 출력 토큰: Claude가 생성하는 텍스트 (답변, 코드 등)

출력 토큰이 입력 토큰보다 약 5배 비싸다. 긴 답변을 요청할수록 비용이 증가한다.

### 4-2 모델별 요금 (2025년 12월 기준)

**Claude Opus 4.5 (최고 성능)** ⭐ 가격 대폭 인하:
- 입력: $5 / 백만 토큰
- 출력: $25 / 백만 토큰
- 이전 대비 1/3 가격으로 인하되어 접근성 향상

**Claude Sonnet 4.5 (균형형)**:
- 입력: $3 / 백만 토큰
- 출력: $15 / 백만 토큰
- 200K 이상 장문 요청 시: $6 / $22.50

**Claude Haiku 4.5 (경량형)**:
- 입력: $1 / 백만 토큰
- 출력: $5 / 백만 토큰

| 모델 | 입력 (백만 토큰) | 출력 (백만 토큰) | 특징 |
|------|-----------------|-----------------|------|
| Opus 4.5 | $5 | $25 | 최고 성능, 가격 인하 |
| Sonnet 4.5 | $3 | $15 | 균형, 가장 많이 사용 |
| Haiku 4.5 | $1 | $5 | 빠른 속도, 간단한 작업 |

## 5. 비용 최적화 전략

### 5-1 토큰 이해와 사용량 확인

비용 최적화의 첫 단계는 사용량 파악이다.

**사용량 확인 방법**:
- console.anthropic.com에서 일일/월간 사용량 확인
- Claude Code에서 `/cost` 명령어로 확인
- VS Code 확장 프로그램으로 실시간 모니터링

### 5-2 프롬프트 최적화

효율적인 프롬프트 작성으로 토큰을 절약할 수 있다.

**간결화**:
- ❌ "안녕하세요 Claude! 정말 중요한 질문이 있는데요..."
- ✅ "다음 데이터를 요약해주세요:"

**구조화**:
```
작업: 고객 피드백 분석
입력: [텍스트]
출력 형식:
- 감정: 긍정/부정/중립
- 주요 키워드: 3개
```

**예시 활용**: 복잡한 설명보다 좋은 예시 하나가 효과적이다.

### 5-3 컨텍스트 관리

불필요한 컨텍스트가 누적되면 비용이 기하급수적으로 증가한다.

**컨텍스트 정리 전략**:
- 긴 대화는 중간중간 요약하여 압축
- 다음 대화에 필요한 정보만 유지
- 주제가 바뀌면 새로운 세션으로 시작

### 5-4 캐싱/배치 처리 활용

**프롬프트 캐싱**:
- 캐시 쓰기: 기본 가격의 1.25배
- 캐시 히트: 기본 가격의 0.1배 (90% 절약)
- 5분 TTL (Time To Live)

**배치 처리 (Batch API)**:
- 24시간 내 처리 조건으로 50% 할인
- Sonnet 기준: $1.50/$7.50 (정가의 절반)
- 최대 배치 크기: 10,000개 요청/배치

### 5-5 예산 관리 시스템

**예산 설정 가이드라인**:
- 일일 예산: $10-50 (개인)
- 월간 예산: $100-1000 (팀)
- 프로젝트별 예산 설정

**예산 관리 도구**:
- console.anthropic.com에서 예산 한도 설정
- 80%, 90%, 100% 도달 시 알림
- 자동 충전(auto-reload) 기능 관리
