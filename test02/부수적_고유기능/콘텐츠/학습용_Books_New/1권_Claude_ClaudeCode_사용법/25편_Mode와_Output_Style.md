# 25편 | Mode와 Output Style

---

Claude Code는 작업 방식을 조절하는 Mode와 응답 스타일을 조절하는 Output Style을 제공한다. 복잡한 작업 전에 계획을 세우거나, 응답의 길이와 형식을 조절할 수 있다. 이 편에서는 Mode와 Output Style 설정을 살펴본다.

## 1. Mode 개요

### 1-1 Mode란

Mode는 Claude Code의 작업 방식을 결정한다. 바로 실행하는 기본 Mode와 계획을 먼저 세우는 Plan Mode가 있다.

```
기본 Mode:   요청 → 바로 실행 → 결과
Plan Mode:   요청 → 계획 수립 → 승인 → 실행 → 결과
```

### 1-2 기본 Mode vs Plan Mode

| 구분 | 기본 Mode | Plan Mode |
|------|----------|-----------|
| 실행 방식 | 즉시 실행 | 계획 후 실행 |
| 승인 필요 | 없음 | 계획 승인 필요 |
| 적합한 작업 | 간단한 작업 | 복잡한 작업 |
| 수정 가능 | 실행 후 | 실행 전 |

### 1-3 Mode 전환

**Plan Mode 진입:**
- 복잡한 작업 요청 시 자동 진입
- 명시적으로 "계획 먼저 세워줘" 요청

**Plan Mode 종료:**
- 계획 승인 후 실행
- 계획 취소

## 2. Plan Mode

### 2-1 Plan Mode 진입

복잡한 구현 작업을 요청하면 Claude Code가 Plan Mode 진입을 제안한다.

```
사용자: 결제 시스템 구현해줘

Claude: 이 작업은 여러 단계가 필요합니다.
        계획을 먼저 세우겠습니다.

        [Plan Mode 진입]
```

**자동 진입 조건:**
- 여러 파일 수정이 필요한 작업
- 아키텍처 결정이 필요한 작업
- 여러 선택지가 있는 작업

### 2-2 계획 작성

Plan Mode에서 Claude Code는 구현 계획을 작성한다.

```
[Plan Mode]

결제 시스템 구현 계획:

1단계: 데이터베이스
- payments 테이블 생성
- subscriptions 테이블 생성

2단계: API 개발
- POST /api/payments/create
- POST /api/payments/confirm
- GET /api/payments/status

3단계: 프론트엔드
- 결제 폼 컴포넌트
- 결제 상태 표시

4단계: 테스트
- API 테스트
- 통합 테스트

이 계획으로 진행할까요?
```

**계획 수정:**
```
사용자: 2단계에 환불 API도 추가해줘

Claude: 계획을 수정합니다.

        2단계: API 개발
        - POST /api/payments/create
        - POST /api/payments/confirm
        - POST /api/payments/refund  ← 추가
        - GET /api/payments/status
```

### 2-3 계획 승인과 실행

**승인:**
```
사용자: 좋아, 진행해

Claude: [Plan Mode 종료]
        계획대로 구현을 시작합니다.

        1단계 진행 중...
```

**취소:**
```
사용자: 취소해

Claude: [Plan Mode 종료]
        계획을 취소했습니다.
```

## 3. Output Style

### 3-1 Output Style이란

Output Style은 Claude Code 응답의 형식과 길이를 조절한다.

```
간결한 스타일:  핵심만 짧게
상세한 스타일:  설명과 함께 길게
코드 중심:      설명 최소화, 코드 위주
```

### 3-2 스타일 요청 방법

응답 스타일은 자연어로 요청한다.

```
사용자: 간결하게 대답해줘
사용자: 자세히 설명해줘
사용자: 코드만 보여줘
```

CLAUDE.md나 Skills에 기본 스타일을 미리 정의할 수도 있다.

```markdown
# CLAUDE.md 예시
응답은 항상 간결하게 작성해주세요.
코드에는 주석을 포함하지 마세요.
```

### 3-3 스타일 옵션

**간결함 (Concise):**
```
사용자: 배열 정렬 함수 만들어줘

Claude: const sort = arr => arr.sort((a, b) => a - b);
```

**상세함 (Detailed):**
```
사용자: 배열 정렬 함수 만들어줘

Claude: 배열을 오름차순으로 정렬하는 함수를 만들겠습니다.

        JavaScript의 sort() 메서드는 기본적으로 문자열
        정렬을 하므로, 숫자 정렬을 위해서는 비교 함수가
        필요합니다.

        const sortNumbers = (arr) => {
          return arr.sort((a, b) => a - b);
        };

        사용 예시:
        sortNumbers([3, 1, 4, 1, 5])  // [1, 1, 3, 4, 5]
```

**코드 중심 (Code-focused):**
```
사용자: 배열 정렬 함수 만들어줘

Claude: const sortNumbers = (arr) => arr.sort((a, b) => a - b);

        // 내림차순
        const sortDesc = (arr) => arr.sort((a, b) => b - a);
```

## 4. 응답 커스터마이징

### 4-1 간결함 vs 상세함

**간결한 응답 요청:**
```
사용자: 간단히 설명해줘
사용자: 핵심만 알려줘
사용자: 코드만 보여줘
```

**상세한 응답 요청:**
```
사용자: 자세히 설명해줘
사용자: 왜 그런지도 알려줘
사용자: 초보자도 이해하게 설명해줘
```

### 4-2 코드 스타일

**특정 스타일 요청:**
```
사용자: TypeScript로 작성해줘
사용자: 함수형으로 작성해줘
사용자: 클래스 기반으로 작성해줘
사용자: 주석 포함해서 작성해줘
```

**포맷팅 요청:**
```
사용자: ESLint 규칙 따라서 작성해줘
사용자: Prettier 포맷 적용해줘
```

### 4-3 언어 설정

Claude Code는 기본적으로 사용자의 언어로 응답한다.

**언어 변경:**
```
사용자: 영어로 대답해줘

Claude: Sure, I'll respond in English from now on.
```

**코드 주석 언어:**
```
사용자: 코드 주석은 한글로 작성해줘

Claude: // 사용자 정보를 가져오는 함수
        const getUser = async (id) => {
          // 데이터베이스에서 조회
          return await db.users.findById(id);
        };
```

## 5. 정리

### Mode 비교

| 구분 | 기본 Mode | Plan Mode |
|------|----------|-----------|
| 진입 | 기본값 | 복잡한 작업 시 |
| 특징 | 즉시 실행 | 계획 → 승인 → 실행 |
| 장점 | 빠른 처리 | 실수 방지, 수정 용이 |
| 적합 | 간단한 작업 | 다단계 작업 |

### Output Style 요약

| 스타일 | 특징 | 적합한 상황 |
|--------|------|------------|
| 간결함 | 핵심만 | 빠른 답변 필요 시 |
| 상세함 | 설명 포함 | 학습, 이해 필요 시 |
| 코드 중심 | 코드 위주 | 구현에 집중 시 |

### 활용 팁

**복잡한 작업:**
```
"계획 먼저 세워줘" → Plan Mode로 안전하게
```

**간단한 질문:**
```
"간단히 알려줘" → 빠른 응답
```

**학습 목적:**
```
"왜 그런지 설명해줘" → 상세한 설명
```

**코딩 작업:**
```
"코드만 보여줘" → 설명 없이 코드만
```
