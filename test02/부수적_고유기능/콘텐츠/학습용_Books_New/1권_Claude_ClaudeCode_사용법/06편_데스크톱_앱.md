# 6편 | 데스크톱 앱

---

Claude 데스크톱 앱은 웹 버전의 모든 기능에 더해, 로컬 시스템과의 연동, 단축키, MCP(Model Context Protocol) 같은 고급 기능을 제공한다. Claude를 주력 업무 도구로 활용하려는 사용자에게 추천하는 접근 방식이다.

## 1. 설치 및 설정

### 1-1 다운로드와 설치

Claude 데스크톱 앱은 Windows, macOS, Linux를 지원한다. claude.ai/download에서 운영체제에 맞는 설치 파일을 다운로드할 수 있다.

**Windows**: .exe 설치 파일을 실행하고 안내에 따라 설치한다.
**macOS**: .dmg 파일을 열고 응용 프로그램 폴더로 드래그한다.
**Linux**: .deb 또는 .rpm 패키지를 사용하거나 AppImage를 직접 실행한다.

설치 후 웹에서 사용하던 계정으로 로그인하면 대화 기록, 프로젝트, 설정이 모두 동기화된다.

### 1-2 초기 설정

**테마 선택**: 밝은 테마, 어두운 테마, 시스템 설정 따르기 중 선택할 수 있다. 설정 메뉴에서 6가지 색상 테마를 제공한다.

**언어 설정**: 인터페이스 언어를 선택한다. Claude의 응답 언어는 대화 중에 지정하거나 프로젝트 설정에서 기본값을 지정할 수 있다.

**알림 설정**: 응답 완료 알림, 오류 알림 등을 설정한다.

## 2. 데스크톱 앱 고유 기능

### 2-1 전역 단축키

데스크톱 앱의 가장 편리한 기능 중 하나는 전역 단축키이다. 어떤 작업을 하고 있든 단축키 하나로 Claude를 즉시 호출할 수 있다.

**기본 단축키** (macOS):
- `Cmd + Space + Space`: Claude 빠른 입력창 열기
- `Cmd + Shift + C`: 새 대화 시작
- `Cmd + ,`: 설정 열기

**기본 단축키** (Windows):
- `Ctrl + Space + Space`: Claude 빠른 입력창 열기
- `Ctrl + Shift + C`: 새 대화 시작
- `Ctrl + ,`: 설정 열기

단축키는 설정에서 원하는 조합으로 변경할 수 있다.

**빠른 입력창**: 전역 단축키로 열리는 작은 입력창이다. 전체 앱을 열지 않고도 빠르게 질문하고 답변을 받을 수 있다. 복잡한 대화가 필요하면 "전체 앱에서 열기" 버튼으로 메인 앱으로 전환할 수 있다.

### 2-2 로컬 파일 접근

데스크톱 앱은 웹 버전보다 로컬 파일을 더 편리하게 다룰 수 있다.

**드래그앤드롭**: 파일 탐색기에서 파일을 드래그하여 대화창에 드롭하면 바로 업로드된다.

**클립보드 통합**: 스크린샷을 찍은 후 대화창에서 `Cmd/Ctrl + V`로 바로 붙여넣을 수 있다.

**최근 파일**: 최근에 업로드한 파일 목록을 제공하여 같은 파일을 반복해서 찾지 않아도 된다.

### 2-3 시스템 통합

**시작 프로그램 등록**: 컴퓨터를 켜면 자동으로 Claude가 백그라운드에서 실행되도록 설정할 수 있다. 필요할 때 단축키로 즉시 호출할 수 있다.

**메뉴바/시스템 트레이**: macOS에서는 메뉴바에, Windows에서는 시스템 트레이에 아이콘이 표시된다. 아이콘을 클릭하면 빠른 메뉴에 접근할 수 있다.

## 3. MCP (Model Context Protocol)

### 3-1 MCP란 무엇인가

MCP는 Claude가 외부 시스템과 통신할 수 있게 해주는 프로토콜이다. 쉽게 말해, Claude가 사용자의 컴퓨터에 있는 파일을 읽거나, 외부 서비스에 접근하거나, 다른 도구를 사용할 수 있게 해주는 기술이다.

웹 버전에서 Claude는 사용자가 직접 업로드한 파일만 볼 수 있다. 하지만 MCP가 연결된 데스크톱 앱에서는 Claude가 허용된 폴더의 파일을 직접 읽고, 외부 API를 호출하고, 데이터베이스를 조회할 수 있다.

### 3-2 내장 MCP 서버

데스크톱 앱에는 기본적으로 세 가지 MCP 서버를 연결할 수 있다.

**Filesystem MCP**: Claude가 사용자 컴퓨터의 특정 폴더에 접근할 수 있게 한다. 예를 들어, 프로젝트 폴더를 연결해두면 "이 폴더에 있는 모든 Python 파일을 분석해줘"라고 요청할 수 있다.

설정 방법:
1. 설정 → MCP 탭으로 이동
2. Filesystem MCP 활성화
3. 접근을 허용할 폴더 경로 지정
4. 읽기 전용 또는 읽기/쓰기 권한 선택

**Memory MCP**: Claude가 대화 간에 정보를 기억할 수 있게 한다. 일반적으로 새 대화를 시작하면 이전 대화 내용을 모르지만, Memory MCP를 활성화하면 중요한 정보를 저장해두고 나중에 참조할 수 있다.

활용 예시:
- 사용자 선호도 기억 (응답 스타일, 자주 사용하는 용어 등)
- 프로젝트 관련 정보 축적
- 자주 참조하는 레퍼런스 저장

**Anthropic MCP**: 여러 Claude 인스턴스를 동시에 활용하는 기능이다. 복잡한 작업을 여러 Claude가 나눠서 처리하게 할 수 있다.

### 3-3 외부 MCP 서버

내장 MCP 외에도 다양한 외부 MCP 서버를 설치하여 Claude의 능력을 확장할 수 있다.

**GitHub MCP**: GitHub 저장소에 접근하여 코드를 읽고, 이슈를 확인하고, PR을 분석할 수 있다.

**Google Drive MCP**: Google Drive의 문서에 접근할 수 있다. 드라이브에 있는 스프레드시트를 분석하거나 문서를 편집하는 작업이 가능하다.

**Slack MCP**: Slack 채널의 메시지를 읽고 분석할 수 있다. 특정 채널의 논의 내용을 요약하거나 관련 메시지를 찾는 작업에 활용할 수 있다.

**Database MCP**: PostgreSQL, MySQL 등의 데이터베이스에 연결하여 쿼리를 실행하고 결과를 분석할 수 있다.

**Browser MCP**: 웹 페이지를 읽고 분석할 수 있다. 특정 URL의 내용을 가져와서 요약하거나 정보를 추출하는 작업이 가능하다.

### 3-4 MCP 설정 방법

외부 MCP 서버 설치는 다음 단계로 진행한다:

1. 원하는 MCP 서버의 설치 가이드 확인
2. 필요한 패키지 또는 실행 파일 다운로드
3. 데스크톱 앱 설정에서 MCP 서버 추가
4. 서버 경로와 필요한 인증 정보 입력
5. 연결 테스트 후 활성화

각 MCP 서버마다 필요한 권한과 설정이 다르므로, 공식 문서를 참고하여 설정해야 한다.

### 3-5 보안 고려사항

MCP는 강력한 기능이지만, 보안에 대한 이해가 필요하다.

**권한 최소화**: 필요한 폴더와 기능에만 접근을 허용한다. 전체 시스템에 접근을 허용하는 것은 피해야 한다.

**민감 정보 주의**: MCP를 통해 Claude가 읽는 파일에 비밀번호, API 키, 개인정보가 포함되지 않도록 주의한다.

**로컬 처리**: MCP를 통해 읽은 데이터는 Claude의 응답 생성에 사용된다. 민감한 데이터는 가능하면 요약하거나 익명화하여 전달하는 것이 좋다.

**정기 검토**: 연결된 MCP 서버와 권한을 정기적으로 검토하고, 더 이상 필요 없는 연결은 비활성화한다.

## 4. 데스크톱 앱 활용 시나리오

### 4-1 프로젝트 중심 업무

특정 프로젝트를 진행할 때 관련 폴더를 Filesystem MCP로 연결하고, 프로젝트 설정에 기본 지시사항을 저장해둔다. 프로젝트 기간 동안 관련된 모든 대화가 같은 맥락에서 이루어진다.

### 4-2 데이터 분석

데이터 폴더를 MCP로 연결하고 "이 폴더에 있는 CSV 파일들을 분석해서 월별 매출 트렌드를 알려줘"라고 요청한다. Claude가 직접 파일을 읽고 분석 결과를 제공한다.

### 4-3 문서 작성 워크플로우

참고 자료 폴더를 연결해두고, 문서 작성 중에 "이 폴더의 이전 보고서들을 참고해서 이번 보고서의 서론을 작성해줘"라고 요청한다.
