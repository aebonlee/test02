# 7편 | Claude Code

---

Claude Code는 터미널에서 실행되는 AI 어시스턴트이다. 다른 접근방식들이 대화에 초점을 맞춘다면, Claude Code는 실제 작업 수행에 초점을 맞춘다. 시스템 명령을 실행하고, 파일을 직접 생성하고 편집하며, 복잡한 업무를 자동으로 처리할 수 있다. 비개발자와 개발자 모두를 위한 강력한 도구이다.

## 1. Claude Code란

### 1-1 기본 개념

Claude Code는 명령줄 인터페이스(CLI)에서 작동하는 Claude이다. 마우스와 그래픽 인터페이스 대신 텍스트 명령으로 조작한다. 처음에는 낯설게 느껴질 수 있지만, 한번 익숙해지면 다른 방식보다 훨씬 강력하고 효율적이다.

모바일 앱이나 웹에서의 Claude는 대화 상대에 가깝다. 질문하면 답하고, 요청하면 텍스트를 생성한다. 반면 Claude Code는 대화 상대이면서 동시에 실행자이다. "파일을 만들어줘"라고 하면 실제로 파일을 만들고, "이 프로그램을 설치해줘"라고 하면 실제로 설치를 진행한다.

### 1-2 Claude Code만의 특화 기능

기본적인 대화 기능은 모바일 앱과 동일하다. 다만 Artifacts는 지원하지 않으며, 이미지는 파일 경로로 제공해야 한다. Claude Code만의 특화 기능은 다음과 같다.

**파일 생성 및 편집**: Excel, Word, PowerPoint, PDF 등 모든 형식의 파일을 생성하고 편집하여 로컬에 저장할 수 있다.

**기술 문서 생성**: README.md, API 문서, 설치 가이드, 사용자 매뉴얼 등을 자동으로 작성한다.

**외부 MCP 서버 연동**: Google Drive, Slack, Jira 등 외부 서비스와 통합하여 사용할 수 있다.

**시스템 명령 실행**: Git, npm 등 모든 터미널 명령을 실행할 수 있다.

**프로젝트 관리**: 파일 구조를 분석하고 재구성할 수 있다.

**자동화 작업**: 프로그램 설치, 환경 변수 설정, 패키지 설치, 빌드 및 배포 과정을 자동화한다.

**코드 작성 및 디버깅**: 전체 프로그램을 처음부터 작성하고, 오류를 분석하여 수정한다.

**Git 워크플로우**: 커밋 생성, 병합 충돌 해결, 릴리스 노트 작성 등 Git 관련 작업을 처리한다.

### 1-3 제약사항

Claude Code에도 제약사항이 있다. 마우스 없이 키보드 입력을 통한 텍스트 명령어로만 조작해야 한다. 터미널 사용 경험이 없으면 처음에 진입 장벽을 느낄 수 있다. 또한 이미지나 그래프 등 시각적 결과물을 직접 표시하지 못한다.

## 2. 설치하기

### 2-1 사전 준비

Claude Code를 설치하려면 Node.js가 먼저 설치되어 있어야 한다. Node.js는 JavaScript 실행 환경으로, Claude Code가 이를 기반으로 동작한다.

Node.js 설치 여부 확인은 터미널에서 `node --version`을 입력하면 된다. 버전 번호가 표시되면 설치된 것이고, 오류가 나면 설치가 필요하다.

### 2-2 터미널 접속

Windows에서 터미널(명령 프롬프트)을 여는 방법은 여러 가지가 있다.

**방법 1**: Windows 키 + R을 누르고, `cmd`를 입력한 후 엔터를 친다.

**방법 2**: Windows 시작 버튼을 클릭하고 "명령 프롬프트"를 검색한 후 엔터를 친다.

검정색(또는 설정에 따라 다른 색) 바탕의 창이 나타나면 성공이다. 이제 명령어를 입력할 수 있다.

### 2-3 Claude Code 설치

터미널에서 다음 명령어를 입력한다.

```
npm install -g @anthropic-ai/claude-code
```

`-g` 옵션은 전역(global) 설치를 의미한다. 이 옵션을 사용하면 컴퓨터의 어느 디렉토리에서든 Claude Code를 실행할 수 있다.

설치에는 약 1-2분이 소요된다. 설치가 완료되면 `claude --version`을 입력하여 정상 설치 여부를 확인할 수 있다.

### 2-4 Claude Code 실행

터미널에서 `claude`를 입력하면 Claude Code가 실행된다. 어느 디렉토리에서든 실행 가능하다.

### 2-5 계정 연결

최초 실행 시 계정 연결 과정이 필요하다.

**바탕화면 색 선택**: 6가지 옵션 중 원하는 것을 선택한다.

**로그인 방식 선택**: 두 가지 옵션이 표시된다.
1. Claude account with subscription (Pro, Max, Team, or Enterprise)
2. Anthropic Console account (API usage billing)

기존 Claude 계정이 있다면 1번을 선택하는 것이 권장된다.

**인증 과정**: 1번 선택 후 엔터를 치면 웹 브라우저가 자동으로 열린다. "Claude Code가 귀하의 Claude chat account에 연결을 요청했습니다"라는 메시지와 함께 권한 정보가 표시된다. "승인" 버튼을 클릭하면 로그인이 완료된다.

로그인 성공 후 터미널에서 Enter 키를 누르면 다음 단계로 진행된다. 이러한 인증 과정은 최초 설치 시에만 진행되며, 이후에는 생략된다.

## 3. 기본 사용법

### 3-1 시작 화면

로그인 후 Claude Code의 시작 화면이 표시된다. "Welcome to Claude Code!"라는 환영 메시지와 함께 현재 작업 디렉토리(cwd)가 표시된다.

화면에는 시작하기 위한 팁도 표시된다. `/init`을 실행하면 프로젝트 설정 파일인 CLAUDE.md를 생성할 수 있다. 파일 분석, 편집, bash 명령어, git 작업 등에 Claude를 활용할 수 있으며, 다른 엔지니어에게 요청하듯 구체적으로 요청할수록 좋은 결과를 얻을 수 있다.

### 3-2 작업 디렉토리 이동

Claude Code는 현재 디렉토리를 기준으로 파일을 생성하고 수정한다. 따라서 프로젝트 작업을 시작하기 전에 해당 프로젝트 폴더로 이동하는 것이 좋다.

이동 명령어 예시: `cd G:\내 드라이브\MyTask`

### 3-3 자연어로 명령하기

Claude Code에게는 자연어로 원하는 작업을 요청할 수 있다.

- "오늘 업무일지 텍스트 파일 만들어줘"
- "이 CSV 파일의 데이터 요약해줘"
- "폴더 안의 이미지 파일만 새 폴더로 정리해줘"
- "React 컴포넌트 기본 구조 만들어줘"
- "테스트 코드 작성해줘"
- "오류 메시지를 분석하고 해결방법 제시해줘"

구체적이고 명확한 요청일수록 더 정확한 결과를 얻을 수 있다.

## 4. 주요 명령어

### 4-1 시스템(슬래시) 명령어

`/help`: 사용 가능한 모든 명령어와 사용법 표시
`/status`: 현재 설정 및 연결 상태 확인
`/init`: CLAUDE.md 파일 생성 (프로젝트 초기 설정)
`/exit` 또는 `Ctrl+C`: Claude Code 종료
`/clear`: 화면 지우기
`/tools`: 사용 가능한 도구 목록 보기
`/sessions`: 대화 세션 목록 보기
`/continue`: 가장 최근 대화 이어가기
`/resume [sessionId]`: 특정 세션 재개
`/model [name]`: 모델 전환 (sonnet, opus 등)
`/config`: 설정 관리
`/mcp`: MCP 서버 관리 및 상태 확인
`/memory`: 저장된 메모리 확인

### 4-2 파일 작업 명령어

`ls` 또는 `dir`: 현재 폴더의 파일 목록 보기
`cd [폴더명]`: 다른 폴더로 이동
`pwd`: 현재 작업 중인 폴더 경로 확인
`cat [파일명]`: 파일 내용 표시

### 4-3 대화 및 모델 명령어

`claude -c` 또는 `--continue`: 가장 최근 대화 이어가기
`claude -r [sessionId]` 또는 `--resume`: 특정 세션 재개
`claude --model sonnet`: Sonnet 모델로 시작
`claude --model opus`: Opus 모델로 시작

## 5. 작업 모드

Claude Code는 3가지 작업 모드를 제공한다. 작업 성격에 따라 적절한 모드를 선택해서 사용한다.

### 5-1 일반 모드 (Default Mode)

기본 모드로, 파일 수정 전에 매번 사용자 승인을 요청한다. 중요한 파일 작업이나 신중한 검토가 필요한 경우에 적합하다. 모든 변경사항을 사전에 확인할 수 있지만, 작업 속도는 느리다.

### 5-2 자동 수정 모드 (Accept Edits Mode)

파일 수정을 자동으로 승인하여 즉시 실행한다. 신뢰할 수 있는 작업이나 빠른 진행이 필요한 경우에 적합하다. 작업 속도가 매우 빠르지만, 실수 시 즉시 파일이 변경된다. `/permission accept`를 입력하여 활성화한다.

### 5-3 플래닝 모드 (Plan Mode)

작업 실행 전에 전체 계획을 먼저 제시하고 승인을 받는다. 복잡한 프로젝트나 여러 파일 수정, 구조적 변경이 필요한 경우에 적합하다. 전체 작업 흐름을 사전에 파악할 수 있지만, 초기 계획 수립 시간이 추가로 소요된다. `/permission plan`을 입력하여 활성화한다.

**Shift + Tab** 키를 누르면 모드를 순환하며 전환할 수 있다 (일반 → 자동 수정 → 플래닝 → 일반).

처음에는 일반 모드로 시작하고, Claude Code의 작업 방식에 익숙해지면 자동 수정 모드를 활용하는 것이 좋다. 대규모 리팩토링이나 새 기능 개발 시에는 플래닝 모드가 유용하다.

## 6. 주요 도구

Claude Code가 사용하는 주요 도구들은 `/tools` 명령어로 확인할 수 있다.

**Bash**: 시스템 명령어 실행 (프로그램 설치, 파일 작업 등)
**Read**: 파일 읽기 (텍스트, 이미지, PDF 등 모든 파일)
**Write**: 새 파일 생성 및 작성
**Edit**: 기존 파일 수정
**MultiEdit**: 파일의 여러 부분 동시 수정
**Grep**: 파일 내용 검색 (정규식 지원)
**Glob**: 파일 이름 패턴 검색 (예: *.txt, **/*.js)
**WebSearch**: 웹 검색 및 최신 정보 가져오기
**WebFetch**: 특정 웹페이지 내용 가져오기
**TodoWrite**: 작업 목록 관리 및 진행 상황 추적
**Task**: 복잡한 작업을 위한 서브 에이전트 실행

이러한 도구들은 Claude Code가 자동으로 선택하여 사용하므로, 사용자가 직접 지정할 필요는 없다.

## 7. 활용 예시

Claude Code의 활용 범위는 매우 넓다. 몇 가지 예시를 들면 다음과 같다.

"VS Code 설치해줘"라고 요청하면 자동으로 다운로드부터 설치까지 진행한다.

"Git 저장소들 일괄 업데이트하고 상태를 확인해줘"라고 하면 git 명령을 실행하여 처리한다.

ChatGPT와 Gemini API를 통해 동일한 주제로 시장분석을 요청하고, 각 AI의 분석 결과를 수집한 후 이를 종합하여 시장분석 보고서를 만들 수 있다.

여러 개의 전문화된 서브 에이전트를 투입하여 각자의 전문 영역에서 협업하여 책을 만들 수도 있다.

프론트엔드, 백엔드, 데이터베이스, API, 결제 시스템, 보안, 테스트 전문 서브 에이전트들이 병렬로 작업하여 완전한 온라인 쇼핑몰을 개발하는 것도 가능하다.

복잡한 명령어를 외울 필요는 없다. Claude Code는 자연어로 대화하듯 명령할 수 있기 때문이다.
