# 8편 | IDE 결합과 API 연결

---

Claude Code만으로도 강력한 작업이 가능하지만, VS Code와 결합하면 더욱 효율적인 작업 환경을 구축할 수 있다. 또한 API를 통해 Claude를 자신의 애플리케이션에 직접 통합할 수도 있다. 이 편에서는 IDE 결합과 API 연결, 두 가지 고급 접근방식을 다룬다.

## 1. VS Code란

### 1-1 기본 개념

VS Code(Visual Studio Code)는 Microsoft가 제공하는 무료 오픈소스 텍스트 편집기이다. 문서 작성부터 코딩까지 모든 텍스트 작성 작업을 지원한다. 수많은 확장 프로그램을 설치하여 기능을 확장할 수 있고, Claude Code와 결합하여 AI 지원 통합 작업 환경을 구성할 수 있다.

### 1-2 주요 기능

**파일 관리**: 프로젝트 탐색기로 폴더 구조를 한눈에 볼 수 있다. 전역 검색으로 프로젝트 내 모든 파일에서 특정 텍스트를 찾을 수 있다. 다중 창 분할로 여러 파일을 동시에 열어볼 수 있다.

**코드 편집**: 문법 강조로 코드를 읽기 쉽게 표시한다. 자동 완성 기능으로 타이핑을 줄여준다. 오류를 실시간으로 검출하고, 코드 리팩토링을 지원한다.

**IntelliSense**: 스마트 코드 완성 기능으로, 변수명이나 함수명을 입력할 때 자동으로 추천해준다.

**Git 통합**: 버전 관리 기능이 기본으로 내장되어 있다. 소스 제어 패널에서 변경 사항을 확인하고 커밋할 수 있다.

**통합 터미널**: 내장 터미널에서 명령어를 실행할 수 있다. 별도의 터미널 창을 열 필요 없이 VS Code 안에서 모든 작업을 처리할 수 있다.

**확장 프로그램**: 4만 개 이상의 플러그인으로 필요한 기능을 추가할 수 있다.

## 2. VS Code 설치

### 2-1 다운로드 및 설치

Claude Code에게 "VS Code 설치해줘"라고 요청하면 자동으로 다운로드부터 설치까지 모든 과정을 대신 처리해준다.

직접 설치하려면 VS Code 다운로드 사이트(https://code.visualstudio.com)에 접속하여 Windows x64 버전을 다운로드한 후 설치 파일을 실행한다.

### 2-2 설치 옵션

설치 시 다음 옵션들을 모두 체크하는 것이 권장된다.

- "바탕 화면에 바로가기 만들기"
- "'Code(으)로 열기' 작업을 Windows 탐색기 파일의 상황에 맞는 메뉴에 추가"
- "'Code(으)로 열기' 작업을 Windows 탐색기 디렉터리의 상황에 맞는 메뉴에 추가"
- "Code를 지원되는 파일 형식에 대한 편집기로 등록"
- "PATH에 추가(다시 시작한 후 사용 가능)"

PATH에 추가 옵션은 일반 터미널에서 `code` 명령어를 사용하기 위해 필수이다.

### 2-3 초기 설정

**Copilot 설정**: GitHub Copilot 설정 화면이 나타나면 건너뛰기를 선택한다.

**테마 선택**: "Browse Color Themes"를 클릭하여 원하는 바탕 색상을 선택한다.

**자동저장 설정**: 파일 → 자동저장을 선택하면 작업 손실을 방지할 수 있다.

**한국어 설정**: 확장 프로그램에서 Korean Language Pack을 설치하고 재시작한다.

## 3. 확장 프로그램

### 3-1 확장 프로그램이란

VS Code의 확장 프로그램(Extension)은 기본 에디터에 새로운 기능을 추가하는 플러그인이다. 왼쪽 사이드바의 확장 프로그램 아이콘(네모 4개 모양)을 클릭한 후 검색하여 설치한다.

### 3-2 공통 도구

**pkief.material-icon-theme**: 파일 종류별로 아이콘을 표시한다.

**gruntfuggly.todo-tree**: '할 일' 목록을 한눈에 정리해서 보여준다.

**eamodio.gitlens**: 파일 변경 이력과 작성자를 확인할 수 있다.

**yzhang.markdown-all-in-one**: 마크다운 문서 작성 시 자동완성과 미리보기를 제공한다.

**tomoki1207.pdf**: PDF 파일을 VS Code에서 바로 열어볼 수 있다.

**humao.rest-client**: API 테스트를 VS Code 안에서 바로 실행할 수 있다.

### 3-3 개발 도구

**christian-kohler.path-intellisense**: 파일 경로 입력 시 자동완성을 제공한다.

**ms-python.python**: Python 코드 실행과 디버깅을 지원한다.

**ritwickdey.liveserver**: HTML 파일을 실시간으로 브라우저에서 확인할 수 있다.

## 4. Claude Code와 함께 사용하기

### 4-1 통합 터미널에서 실행

VS Code에서 Claude Code를 실행하려면 `Ctrl + `` (백틱) 키를 눌러 통합 터미널을 연다. 또는 좌측 상단 메뉴에서 터미널 → 새 터미널을 선택한다. 터미널 창이 열리면 `claude`를 입력한다.

### 4-2 작업 폴더 열기

VS Code에서 파일 > 폴더 열기를 선택하여 작업할 폴더를 연다. 터미널에서 원하는 폴더로 이동한 후 `code .` 명령어를 입력하면 현재 폴더를 VS Code에서 바로 열 수 있다.

### 4-3 시너지 효과

VS Code에서 파일을 편집하면서 동시에 터미널에서 Claude Code의 도움을 받으면 작업 효율이 크게 향상된다. Claude Code가 코드를 생성하면 VS Code의 편집 창에서 바로 확인하고 수정할 수 있다.

## 5. 통합 작업환경 구성

모든 도구를 설치하면 강력한 통합 작업환경을 구성할 수 있다.

**Claude Code**: 터미널 기반 AI 업무 도우미로, 파일 시스템 직접 조작, Git 명령 실행, 프로그램 자동 설치를 담당한다.

**Claude 데스크톱 앱**: MCP 서버 연결로 파일 관리와 고급 기능을 활용한다. Artifacts로 문서 작성 및 미리보기가 가능하다.

**VS Code**: 다양한 확장 프로그램으로 전문적인 코드 편집이 가능하다. Claude Code가 생성한 코드를 즉시 편집하고 디버깅할 수 있다.

4개 도구가 유기적으로 연결되어 개발부터 일반 사무까지 AI 기반으로 처리할 수 있다.

## 6. API 연결

### 6-1 Claude API란

Anthropic API는 개발자가 자신의 애플리케이션에 Claude를 직접 통합할 수 있도록 제공하는 프로그래밍 인터페이스이다. 웹이나 앱에서 대화하는 것이 아니라, 코드를 통해 Claude의 기능을 호출하는 방식이다.

지금까지 살펴본 접근방식들(모바일 앱, 웹, 데스크톱 앱, Claude Code, IDE 결합)은 모두 Anthropic이 제공하는 인터페이스를 통해 Claude를 사용한다. API 연결은 이와 달리 자신만의 인터페이스를 만들어 Claude를 사용하는 방식이다.

### 6-2 주요 기능

**Messages API**: 기본적인 대화 기능을 제공한다. 메시지를 보내고 응답을 받는다.

**Batch API**: 여러 요청을 한 번에 처리할 수 있다. 대량의 데이터를 처리할 때 효율적이다.

**Prompt Caching**: 반복적으로 같은 시스템 프롬프트를 사용할 때 캐싱 기능을 활용하면 토큰 사용량을 90%까지 줄일 수 있다.

**Tool Use**: Claude가 외부 도구를 호출할 수 있게 한다. 예를 들어 날씨 API를 호출하거나 데이터베이스를 조회하는 등의 작업이 가능하다.

**Vision**: 이미지를 분석하는 기능을 제공한다.

**Computer Use (Beta)**: Claude가 화면을 보고 마우스와 키보드를 제어할 수 있는 실험적 기능이다.

### 6-3 시작 방법

API를 사용하려면 다음 단계를 거친다.

**1. Anthropic Console 접속**: https://console.anthropic.com 에 접속하여 Claude 계정으로 로그인한다.

**2. API 키 발급**: Settings → API Keys → Create Key를 클릭하여 API 키를 발급받는다. 발급된 키는 `sk-ant-api03-`로 시작한다. 이 키는 비밀번호처럼 안전하게 보관해야 한다. 다른 사람과 공유하거나 GitHub 같은 공개된 장소에 노출되지 않도록 주의한다.

**3. SDK 설치**: 사용하는 프로그래밍 언어에 맞는 SDK를 설치한다. Python의 경우 `pip install anthropic` 명령어를 사용한다.

### 6-4 API 요금

API는 사용한 만큼 과금되는 방식이다. 웹이나 앱의 구독 요금과는 별도로 청구된다. 모델별로 입력 토큰과 출력 토큰에 대한 가격이 다르다.

Prompt Caching을 활용하면 토큰 사용량을 크게 줄일 수 있고, Batch API를 사용하면 50% 할인된 가격으로 처리할 수 있다.

### 6-5 API vs 다른 접근방식

**API가 적합한 경우**:
- 자신만의 애플리케이션에 AI 기능을 넣고 싶을 때
- 대량의 데이터를 자동으로 처리해야 할 때
- 다른 시스템과 연동이 필요할 때
- 사용량을 세밀하게 제어하고 싶을 때

**다른 접근방식이 적합한 경우**:
- 대화형으로 AI를 사용하고 싶을 때
- 개발 없이 바로 사용하고 싶을 때
- 구독 요금제가 더 경제적일 때

### 6-6 공식 문서

API에 대한 자세한 내용은 공식 문서(https://docs.anthropic.com)에서 확인할 수 있다. 각 기능별 상세 설명, 코드 예시, 요금 정보 등이 제공된다.

## 7. Claude Code와 데스크톱 앱의 차이점

### 7-1 작업 환경의 근본적 차이

**Claude Code**: 창조적 구현 환경이다. 새로운 코드 작성, 복잡한 시스템 설계, 예측하지 못한 오류 해결에 적합하다. 시스템 명령 실행, Git 통합, 프로그램 자동 설치 등 개발 작업에 필요한 모든 권한을 보유한다.

**데스크톱 앱**: 구조적 정리 환경이다. 안정적이고 예측 가능한 작업 환경, GUI 기반 상호작용을 제공한다. 문서 작성 및 편집, 데이터 분석, 아이디어 정리에 적합하다.

### 7-2 오류 처리와 위험도

**Claude Code에서의 오류**: 잘못된 명령 하나로 시스템 파일이 손상될 수 있다. Git 작업 실수로 코드 손실 가능성이 있다.

**데스크톱 앱에서의 오류**: 대부분 수정 가능한 텍스트 오류에 그친다. 시스템에 영향을 주지 않는 안전한 환경이다.

이러한 차이점을 이해하면 각자의 작업 상황과 목적에 따라 적절한 도구를 선택할 수 있다. 때로는 여러 도구를 상호 보완적으로 활용하는 것이 가장 효과적이다.
