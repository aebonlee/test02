# 11편 | Claude 내부 도구 (1) - 대화/검색

---

이전 편에서 Claude의 내부 기능, 즉 모델이 태어날 때부터 가진 본질적 능력을 살펴보았다. 이번 편에서는 Anthropic이 그 위에 추가로 제공하는 내부 도구 중 대화와 검색에 관련된 것들을 다룬다.

## 1. 내부 도구란

### 1-1 내부 기능과의 차이

내부 기능은 Claude 모델 자체에 내장된 능력이다. 텍스트 이해, 이미지 분석, 논리적 추론 등 모든 대화에서 자동으로 작동한다.

내부 도구는 다르다. Anthropic이 Claude 인터페이스에 추가로 내장한 기능으로, 사용자가 선택적으로 활성화하거나 사용할 수 있다. 일부는 특정 플랜에서만 제공된다.

| 구분 | 내부 기능 | 내부 도구 |
|------|----------|----------|
| 제공자 | 모델 자체 | Anthropic 추가 |
| 활성화 | 항상 작동 | 선택적 사용 |
| 플랜 제한 | 없음 | 일부 있음 |

### 1-2 대화/검색 도구 개요

내부 도구는 크게 두 가지로 나눌 수 있다. 대화와 검색을 돕는 도구, 그리고 개발과 작업을 돕는 도구다. 이 편에서는 전자를 다룬다.

**대화/검색 도구:**
- 웹 검색: 실시간 정보 검색
- Research: 심층 조사와 보고서 작성
- 심층 사고: 복잡한 문제에 대한 깊은 추론
- 스타일: 응답의 톤과 형식 설정

## 2. 웹 검색

### 2-1 웹 검색이란

Claude의 지식은 학습 시점까지로 제한된다. 오늘의 뉴스, 현재 주가, 최신 기술 동향 등은 기본적으로 알 수 없다. 웹 검색 도구는 이 한계를 넘어 실시간으로 인터넷에서 정보를 가져온다.

웹 검색이 활성화되면 Claude는 필요할 때 자동으로 검색을 수행한다. 사용자가 명시적으로 "검색해줘"라고 말하지 않아도, 최신 정보가 필요하다고 판단되면 검색한다.

### 2-2 언제 사용하나

웹 검색이 자동으로 작동하는 상황:

**최신 정보 요청:**
- "오늘 환율이 얼마야?"
- "이번 주 주요 뉴스 알려줘"
- "최신 아이폰 스펙이 어떻게 돼?"

**시간에 민감한 질문:**
- "현재 서울 날씨는?"
- "지금 열리고 있는 컨퍼런스가 있어?"
- "올해 노벨상 수상자가 누구야?"

**검증이 필요한 정보:**
- 특정 기업의 현재 CEO
- 법률이나 정책의 최신 상태
- 기술 문서의 현재 버전

### 2-3 검색 결과 활용

Claude는 검색 결과를 그대로 전달하지 않는다. 여러 출처의 정보를 종합하고, 분석하여 사용자 질문에 맞는 답변을 구성한다.

**검색 결과 처리 과정:**
1. 관련 웹페이지 검색
2. 여러 출처에서 정보 수집
3. 정보 종합 및 검증
4. 사용자 질문에 맞게 재구성
5. 출처 표시와 함께 답변 제공

검색 결과에 대한 출처를 함께 제공하므로, 사용자가 원본을 직접 확인할 수도 있다.

### 2-4 한계와 주의점

웹 검색에도 한계가 있다:

**접근 불가 콘텐츠:**
- 로그인이 필요한 페이지
- 유료 구독 콘텐츠
- 검색 엔진에 인덱싱되지 않은 페이지

**정보의 정확성:**
- 검색 결과가 항상 정확한 것은 아니다
- 여러 출처가 상충할 수 있다
- 중요한 결정은 직접 확인 권장

※ 웹 검색은 Pro 플랜 이상에서 사용할 수 있다.

## 3. Research

### 3-1 Research란

Research는 웹 검색의 확장판이다. 단순히 정보를 찾는 것을 넘어, 주제에 대해 심층적으로 조사하고 종합적인 보고서를 작성한다.

일반적인 웹 검색이 "빠른 답변"이라면, Research는 "깊은 분석"이다. 수십 개의 출처를 탐색하고, 정보를 분류하고, 통찰을 도출한다.

### 3-2 웹 검색과의 차이

| 구분 | 웹 검색 | Research |
|------|---------|----------|
| 목적 | 빠른 정보 확인 | 심층 조사 |
| 소요 시간 | 몇 초 | 수 분 |
| 출처 수 | 몇 개 | 수십 개 |
| 결과물 | 간단한 답변 | 구조화된 보고서 |
| 적합한 질문 | 사실 확인 | 복잡한 주제 분석 |

### 3-3 활용 시나리오

Research가 적합한 상황:

**시장 조사:**
- "한국 SaaS 시장 현황을 분석해줘"
- "경쟁사 3곳의 가격 정책을 비교해줘"

**기술 조사:**
- "React와 Vue의 2024년 현황을 비교해줘"
- "최신 AI 개발 도구 트렌드를 정리해줘"

**의사결정 지원:**
- "원격근무 도입의 장단점을 조사해줘"
- "클라우드 서비스 선택 기준을 정리해줘"

### 3-4 결과물 형태

Research의 결과물은 구조화된 보고서 형태다:

**보고서 구성:**
- 요약: 핵심 발견사항
- 본문: 주제별 상세 분석
- 출처: 참고한 자료 목록
- 결론: 종합적인 통찰

진행 중에는 조사 상황을 실시간으로 보여주어, 어떤 정보를 탐색하고 있는지 확인할 수 있다.

※ Research는 Pro 플랜 이상에서 사용할 수 있다.

## 4. 심층 사고 (Extended Thinking)

### 4-1 심층 사고란

심층 사고는 Claude가 복잡한 문제에 대해 더 깊이, 더 오래 생각하도록 하는 기능이다. 일반적인 응답이 직관적 사고라면, 심층 사고는 단계별로 신중하게 추론하는 과정이다.

활성화하면 Claude는 답변하기 전에 충분한 시간을 들여 문제를 분석한다. 생각 과정이 길어지지만, 그만큼 더 정확하고 깊이 있는 답변을 얻을 수 있다.

### 4-2 일반 응답과의 차이

| 구분 | 일반 응답 | 심층 사고 |
|------|----------|----------|
| 응답 속도 | 빠름 | 느림 |
| 사고 과정 | 숨겨짐 | 표시됨 |
| 적합한 문제 | 단순한 질문 | 복잡한 추론 |
| 답변 깊이 | 기본 | 심층 |

### 4-3 적합한 사용 상황

심층 사고가 효과적인 경우:

**복잡한 수학/논리 문제:**
- 여러 단계의 계산이 필요한 문제
- 논리적 추론이 필요한 퍼즐
- 증명이 필요한 문제

**코드 설계:**
- 복잡한 알고리즘 설계
- 아키텍처 결정
- 성능 최적화 방안

**전략적 분석:**
- 여러 요소를 고려한 의사결정
- 시나리오별 장단점 분석
- 복잡한 상황의 문제 해결

### 4-4 사고 과정 확인

심층 사고의 특징은 Claude의 생각 과정을 볼 수 있다는 점이다. 어떤 단계를 거쳐 결론에 도달했는지 확인할 수 있어, 추론의 타당성을 검증할 수 있다.

**확인할 수 있는 것:**
- 문제를 어떻게 분해했는지
- 각 단계에서 어떤 판단을 했는지
- 여러 선택지 중 왜 특정 결론을 택했는지

이를 통해 Claude의 답변을 더 잘 이해하고, 필요하면 특정 부분에 대해 추가 질문할 수 있다.

## 5. 스타일 (Styles)

### 5-1 스타일이란

스타일은 Claude의 응답 톤과 형식을 설정하는 기능이다. 같은 내용이라도 상황에 따라 다른 방식으로 전달해야 할 때가 있다. 스타일을 설정하면 모든 응답에 일관된 톤이 적용된다.

예를 들어, 업무용 이메일을 작성할 때는 공식적인 스타일이, 친구에게 설명할 때는 친근한 스타일이 적합하다.

### 5-2 기본 제공 스타일

Anthropic이 기본으로 제공하는 스타일:

**Formal (공식적):**
- 비즈니스 문서에 적합
- 존칭 사용, 정중한 표현
- 구조화된 형식

**Concise (간결한):**
- 핵심만 전달
- 불필요한 수식어 제거
- 빠른 정보 전달에 적합

**Explanatory (설명적):**
- 상세한 설명 포함
- 예시와 비유 활용
- 학습 목적에 적합

**Creative (창의적):**
- 자유로운 표현
- 은유와 비유 활용
- 창작 작업에 적합

### 5-3 커스텀 스타일 생성

기본 스타일 외에 자신만의 스타일을 만들 수 있다. 특정 톤, 형식, 길이 등을 지정하면 Claude가 해당 스타일을 학습하고 적용한다.

**커스텀 스타일 예시:**
- "기술 블로그 작성자처럼 설명해줘"
- "10살 아이도 이해할 수 있게 말해줘"
- "마케팅 카피라이터 스타일로 작성해줘"

한 번 설정한 스타일은 해당 대화 또는 프로젝트 전체에 적용되어, 매번 지정할 필요가 없다.

### 5-4 활용 팁

스타일을 효과적으로 사용하는 방법:

**상황별 스타일 선택:**
- 보고서 작성 → Formal
- 빠른 답변 필요 → Concise
- 개념 학습 → Explanatory
- 아이디어 발상 → Creative

**스타일 조합:**
- "Formal하면서 Concise하게" 같은 조합도 가능
- 상황에 맞게 여러 특성을 섞어 사용

**일관성 유지:**
- 프로젝트 단위로 스타일 설정
- 팀원과 동일한 스타일 공유

## 6. 정리

대화/검색 도구는 Claude와의 대화를 더 풍부하고 효과적으로 만든다.

| 도구 | 용도 | 플랜 제한 |
|------|------|----------|
| 웹 검색 | 실시간 정보 검색 | Pro 이상 |
| Research | 심층 조사 및 보고서 | Pro 이상 |
| 심층 사고 | 복잡한 문제 추론 | 없음 |
| 스타일 | 응답 톤/형식 설정 | 없음 |

다음 편에서는 개발과 작업을 돕는 내부 도구들을 살펴본다. Commands, Hooks, Subagents, Skills 등 Claude Code 사용자에게 특히 유용한 도구들이다.
