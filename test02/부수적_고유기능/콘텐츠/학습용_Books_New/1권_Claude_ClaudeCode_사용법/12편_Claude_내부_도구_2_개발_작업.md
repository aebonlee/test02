# 12편 | Claude 내부 도구 (2) - 개발/작업

---

이전 편에서 대화와 검색을 돕는 내부 도구를 살펴보았다. 이번 편에서는 개발과 작업을 돕는 도구들을 다룬다. 특히 Claude Code 사용자에게 유용한 기능들이 많다.

## 1. 개발/작업 도구 개요

### 1-1 대화/검색 도구와의 차이

대화/검색 도구(웹 검색, Research, 심층 사고, 스타일)는 정보를 얻고 응답 품질을 높이는 데 초점을 맞춘다. 개발/작업 도구는 다르다. 실제 작업을 자동화하고, 효율을 높이고, 협업을 돕는 데 초점을 맞춘다.

### 1-2 6가지 개발/작업 도구

| 도구 | 핵심 기능 |
|------|----------|
| Slash Commands | 자주 쓰는 작업을 단축 명령어로 실행 |
| Subagents | 전문화된 하위 에이전트로 작업 분담 |
| Anthropic Skills | 문서, 스프레드시트 등 파일 생성 |
| Checkpoints | 작업 상태 자동 저장 및 복원 |
| Desktop Extensions | 로컬 파일, 스크린샷 등 접근 |
| Hooks | 특정 시점에 자동 실행되는 동작 설정 |

## 2. Slash Commands

### 2-1 Slash Commands란

Slash Commands는 슬래시(/)로 시작하는 단축 명령어다. 자주 반복하는 작업을 한 줄 명령어로 실행할 수 있다. 매번 같은 지시를 길게 입력하는 대신, `/명령어` 한 번으로 처리한다.

예를 들어 코드 리뷰를 요청할 때마다 "이 코드를 검토해줘. 버그, 성능, 보안 측면에서 분석하고..."라고 입력하는 대신, `/review`만 입력하면 된다.

### 2-2 기본 제공 명령어

Claude Code에서 기본으로 제공하는 명령어들:

| 명령어 | 기능 |
|--------|------|
| `/help` | 도움말 표시 |
| `/clear` | 대화 기록 초기화 |
| `/compact` | 컨텍스트 압축 |
| `/config` | 설정 확인/변경 |
| `/cost` | 토큰 사용량 확인 |

### 2-3 커스텀 명령어 만들기

기본 명령어 외에 자신만의 명령어를 만들 수 있다. `.claude/commands/` 폴더에 마크다운 파일을 생성하면 된다.

**커스텀 명령어 예시:**

`.claude/commands/review-pr.md` 파일:
```markdown
코드 변경사항을 검토해주세요:
- 버그 가능성
- 성능 영향
- 보안 취약점
- 코드 스타일
```

이제 `/review-pr`을 입력하면 위 지시사항이 자동으로 적용된다.

### 2-4 활용 예시

**개발 워크플로우:**
- `/test` - 테스트 실행
- `/deploy` - 배포 시작
- `/commit` - 변경사항 커밋

**문서 작업:**
- `/docs` - 문서 생성
- `/translate` - 번역 시작
- `/summary` - 요약 생성

※ Slash Commands는 Claude Code에서 사용할 수 있다.

## 3. Subagents

### 3-1 Subagents란

Subagents는 전문화된 하위 에이전트다. 복잡한 작업을 여러 전문가에게 분담하듯이, Claude도 특정 역할에 최적화된 하위 에이전트를 활용할 수 있다.

메인 에이전트가 전체를 조율하고, 각 Subagent가 자신의 전문 분야를 담당한다.

### 3-2 병렬 처리 방식

Subagents의 강력한 점은 병렬 처리다. 여러 작업을 동시에 진행할 수 있어 전체 소요 시간이 크게 줄어든다.

**순차 처리 (Subagents 없이):**
```
작업 A (10분) → 작업 B (10분) → 작업 C (10분) = 총 30분
```

**병렬 처리 (Subagents 활용):**
```
작업 A (10분) ─┐
작업 B (10분) ─┼→ 총 10분
작업 C (10분) ─┘
```

### 3-3 전문화된 역할 분담

각 Subagent는 특정 역할에 최적화되어 있다:

| Subagent 유형 | 전문 분야 |
|---------------|----------|
| frontend-developer | 프론트엔드 개발 |
| backend-developer | 백엔드 개발 |
| code-reviewer | 코드 리뷰 |
| test-engineer | 테스트 작성 |
| devops-troubleshooter | 인프라/배포 |

### 3-4 활용 예시

**복잡한 기능 개발:**
- frontend-developer: UI 컴포넌트 개발
- backend-developer: API 개발
- test-engineer: 테스트 케이스 작성

세 작업이 동시에 진행되어 개발 속도가 빨라진다.

※ Subagents는 Claude Code에서 사용할 수 있다.

## 4. Anthropic Skills

### 4-1 Skills란

Skills는 재사용 가능한 지시사항, 스크립트, 리소스의 패키지다. 특정 유형의 작업을 수행하는 방법을 캡슐화하여, 누구나 쉽게 해당 작업을 수행할 수 있게 한다.

### 4-2 기본 제공 Skills 목록

Anthropic이 기본으로 제공하는 Skills:

| Skill | 기능 |
|-------|------|
| Excel Skill | 스프레드시트 생성 및 편집 |
| Word Skill | 문서 생성 및 편집 |
| PowerPoint Skill | 프레젠테이션 생성 |
| PDF Skill | PDF 문서 생성 |

### 4-3 Skills 사용법

Skills는 필요할 때 자동으로 활성화된다. "엑셀 파일 만들어줘"라고 요청하면 Excel Skill이 작동한다.

**사용 예시:**
- "이 데이터로 엑셀 표 만들어줘" → Excel Skill 활성화
- "보고서 PDF로 만들어줘" → PDF Skill 활성화
- "프레젠테이션 초안 만들어줘" → PowerPoint Skill 활성화

### 4-4 외부 Skills와의 차이

Anthropic이 제공하는 기본 Skills 외에, 파트너사나 사용자가 만든 외부 Skills도 있다. 외부 Skills는 확장 생태계에 속하며, 13편에서 다룬다.

| 구분 | Anthropic Skills | 외부 Skills |
|------|------------------|-------------|
| 제공자 | Anthropic | 파트너/사용자 |
| 분류 | 내부 도구 | 확장 생태계 |
| 설치 | 기본 제공 | 별도 설치 필요 |

## 5. Checkpoints

### 5-1 Checkpoints란

Checkpoints는 작업 상태를 자동으로 저장하는 기능이다. 작업 중 문제가 발생하면 이전 상태로 쉽게 되돌릴 수 있다. 게임의 세이브 포인트와 비슷하다.

### 5-2 자동 저장 방식

Checkpoints는 백그라운드에서 자동으로 작동한다. 사용자가 직접 저장 버튼을 누를 필요가 없다.

**저장되는 시점:**
- 중요한 파일 변경 후
- 명령 실행 전후
- 일정 시간 간격

### 5-3 복원 방법

문제가 발생하면 이전 Checkpoint로 되돌릴 수 있다:

1. Checkpoint 목록 확인
2. 원하는 시점 선택
3. 해당 상태로 복원

복원해도 Checkpoint 기록은 유지되어, 다시 앞으로 되돌리는 것도 가능하다.

### 5-4 활용 시나리오

**코드 실험:**
- 새로운 접근법 시도
- 문제 발생 시 이전 상태로 복원
- 안전하게 실험 가능

**리팩토링:**
- 대규모 코드 변경 전 Checkpoint 확인
- 변경이 잘못되면 즉시 복원
- 위험 부담 없이 개선 시도

※ Checkpoints는 Claude Code에서 사용할 수 있다.

## 6. Desktop Extensions

### 6-1 Desktop Extensions란

Desktop Extensions는 Claude 데스크톱 앱이 로컬 환경에 접근할 수 있게 하는 기능이다. 웹 버전에서는 불가능한 로컬 파일 시스템, 스크린샷, 클립보드 등에 접근할 수 있다.

### 6-2 로컬 파일 시스템 접근

데스크톱 앱에서는 컴퓨터의 파일을 직접 읽고 분석할 수 있다.

**가능한 작업:**
- 로컬 폴더의 파일 목록 확인
- 파일 내용 읽기 및 분석
- 이미지, 문서 파일 처리

### 6-3 스크린샷/클립보드

화면 캡처와 클립보드 내용도 활용할 수 있다.

**스크린샷 활용:**
- 현재 화면 캡처
- 오류 화면 분석
- UI 디자인 피드백

**클립보드 활용:**
- 복사한 텍스트 바로 분석
- 이미지 붙여넣기
- 데이터 빠른 전달

### 6-4 활용 예시

**개발 작업:**
- 에러 화면 스크린샷 → 문제 분석
- 로컬 로그 파일 → 디버깅

**문서 작업:**
- 로컬 PDF 파일 → 내용 요약
- 여러 파일 → 종합 분석

※ Desktop Extensions는 데스크톱 앱에서 사용할 수 있다.

## 7. Hooks

### 7-1 Hooks란

Hooks는 특정 시점에 자동으로 실행되는 동작을 설정하는 기능이다. "이런 일이 발생하면 저런 동작을 해라"는 규칙을 미리 정의해두는 것이다.

### 7-2 Hook 유형

다양한 시점에 Hook을 설정할 수 있다:

| Hook 유형 | 실행 시점 |
|-----------|----------|
| PreToolUse | 도구 사용 전 |
| PostToolUse | 도구 사용 후 |
| Notification | 알림 발생 시 |
| Stop | 작업 중단 시 |

### 7-3 설정 방법

Hooks는 설정 파일에서 정의한다. JSON 형식으로 조건과 동작을 지정한다.

**Hook 설정 예시:**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write",
        "command": "npm run lint"
      }
    ]
  }
}
```

위 설정은 "파일 작성 후 자동으로 린트 실행"을 의미한다.

### 7-4 활용 시나리오

**품질 관리 자동화:**
- 코드 작성 후 → 자동 린트
- 커밋 전 → 자동 테스트
- 배포 전 → 자동 빌드 검증

**워크플로우 자동화:**
- 파일 생성 후 → 자동 포맷팅
- API 호출 후 → 로그 기록
- 에러 발생 시 → 알림 전송

Hooks는 초기 설정에 시간이 들지만, 한 번 설정하면 반복 작업을 크게 줄일 수 있다.

※ Hooks는 Claude Code에서 사용할 수 있다.
