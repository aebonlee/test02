## 개요

### 책의 목적
PROJECT SSAL Grid를 활용한 체계적 프로젝트 관리 방법 완전 가이드. AI 협업 시대의 프로젝트 관리 방법론 습득.

### 대상 독자
- Claude Code로 실제 프로젝트를 진행하는 사람
- 체계적인 프로젝트 관리 방법을 배우고 싶은 사람
- AI와 협업하여 개발할 때 품질을 보장하고 싶은 사람

### 핵심 원칙
1. **SSAL Grid 전체 사용법**: 개념부터 실전 활용까지 완전 가이드
2. **논리적 학습 순서**: 개념 → 구조 → 생성 → 검증 → 도구 → 운영
3. **실전 기반**: SSALWorks 실제 개발 경험에서 검증된 방법론

### 주요 소스
```
S0_Project-SSAL-Grid_생성/
├── manual/
│   ├── PROJECT_SSAL_GRID_MANUAL.md (27개 섹션)

```

## 논리적 흐름

```
[1편] SSAL Grid가 뭐냐? → 22개 속성으로 Task를 관리한다
                ↓
[2편] 22개 속성 정의 → 각 속성이 뭘 의미하는지
                ↓
[3편] 5x11 Matrix → 속성 정의를 위해 Stage/Area가 필요
                ↓
[4편] Task 생성 → Matrix 기반으로 Task 생성
                ↓
[5편] Task Instruction → 생성된 Task에 지시사항 작성
                ↓
[6편] Verification Instruction → 검증 기준 작성
                ↓
[7편] 검증 시스템 가동 → 작성된 Instruction으로 검증 수행
                ↓
[8편] Supabase + Viewer → 위 과정을 DB에 저장하고 Viewer로 확인
                ↓
[9편] 매뉴얼 활용 → 이 모든 것을 관리하는 매뉴얼
                ↓
[10편] Order Sheet → 작업 지시 체계
                ↓
[11편] AI 작업 6대 규칙 → 6대 규칙 + 5군데 반영
                ↓
[12편] Orders/Reports JSON → Human-AI 통신 시스템
```

### 3편: 5x11 Matrix (Stage × Area)

**목적**: Stage와 Area 체계 이해

**주요 내용**:
- 5개 Stage 정의 (S1~S5)

- 11개 Area 정의

- 5x11 Matrix 구조
- Task ID 체계 (Stage + Area + Number)

### 11편: AI 작업 6대 규칙 및 규칙 반영 방법

**목적**: Grid 작업 규칙을 프로젝트에 반영하고, AI가 따라야 할 핵심 원칙 이해

**주요 내용**:

**Part A: 작업 규칙 (6대 규칙)** - `.claude/rules/` 폴더
1. File Naming - 파일 명명 규칙 (kebab-case, Task ID 주석)
2. Save Location - 저장 위치 규칙 (이중 저장)
3. Area/Stage - 11개 Area, 5개 Stage 매핑
4. Grid Writing - Grid 22개 속성 작성 규칙
5. Execution Process - 6단계 실행 프로세스
6. Verification - Task/Stage Gate/PO 검증 기준

**Part B: 규칙 반영 대상 5군데**
1. CLAUDE.md - 작업 규칙 참조
2. Project Directory Structure - 저장 위치 규칙
3. Task Instruction - Task별 상세 지시
4. Supabase - DB 스키마 규칙
5. Order Sheet - 작업 지시 형식

> **참고**: 준수사항(AI 행동 원칙)은 1권 Part 4 10편 "프로젝트 관리"에서 CLAUDE.md와 함께 설명
