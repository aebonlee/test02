## 1. 관리자 대시보드 접속

### 1.1 접속 URL

| 환경 | URL |
|------|-----|
| **프로덕션** | https://ssalworks.ai.kr/admin-dashboard.html |
| **로컬** | http://localhost:3000/admin-dashboard.html |

### 1.2 접속 권한

관리자 대시보드는 **관리자 권한이 있는 계정만** 접속할 수 있습니다.

**권한 부여 방법:**
```sql
-- Supabase SQL Editor에서 실행
UPDATE users
SET role = 'admin'
WHERE email = '관리자이메일@example.com';
```

### 1.3 로그인 방법

1. 관리자 URL 접속
2. Google 로그인 또는 이메일 로그인
3. 로그인 후 자동으로 Admin Dashboard로 이동

## 2. 대시보드 개요

### 2.1 메인 화면

관리자 대시보드 메인 화면에서는 주요 통계를 한눈에 확인할 수 있습니다.

**상단 지표 카드:**
- 총 가입자 수
- 활성 사용자
- 오늘 매출
- 이번 달 매출

**긴급 작업:**
- 미확인 개설비 입금 건
- 크레딧 소진 임박 사용자
- 미답변 문의

### 2.2 메뉴 구조

```
Admin Dashboard
├── 콘텐츠 관리
│   └── 공지사항 관리
│
├── 회원/결제
│   ├── 회원 관리
│   ├── 빌더 계정 개설비 관리
│   ├── 프로젝트 관리
│   ├── 크레딧 관리
│   └── 문의 관리
│
├── 통계
│   ├── 통계 대시보드
│   └── API 사용량
│
└── 시스템
    └── 시스템 설정
```

## 3. 콘텐츠 관리

### 3.1 공지사항 관리

**접속 경로:** 콘텐츠 관리 → 공지사항 관리

**가능 작업:**
- 공지사항 추가/수정/삭제
- 중요 공지 상단 고정
- 공개/비공개 설정

**공지사항 추가:**
1. "+ 새 공지사항" 버튼 클릭
2. 제목, 내용 입력
3. 중요 여부 선택
4. "저장" 클릭

## 4. 회원 관리

### 4.1 회원 목록

**접속 경로:** 회원/결제 → 회원 관리

**표시 항목:**
- 회원 ID
- 이메일
- 닉네임
- 플랜
- 상태
- 크레딧 잔액
- 가입일

### 4.2 회원 검색

상단 검색창에서 이메일, 닉네임, 회원 ID로 검색 가능합니다.

### 4.3 회원 상세 보기

1. 회원 목록에서 "상세" 링크 클릭
2. 상세 정보 모달 표시

**표시 내용:**
- 회원 ID, 이메일
- 닉네임, 실명
- 가입일, 플랜
- 빌더 계정 개설비 납부 여부
- 크레딧 잔액

### 4.4 회원 상태 변경

모달에서 상태 선택 후 "상태 저장" 버튼 클릭:
- 활성
- 비활성
- 정지
- 대기

### 4.5 회원 삭제

1. 회원 상세 모달에서 "회원 삭제" 버튼 클릭
2. 확인 메시지에서 "확인" 클릭
3. 관련 데이터(크레딧, 결제내역 등)도 함께 삭제됨

**주의:** 삭제는 되돌릴 수 없습니다.

## 5. 빌더 계정 개설비 관리

### 5.1 개요

SSAL Works 프로젝트 생성을 위해 빌더 계정 개설비(₩3,000,000)를 납부해야 합니다.

### 5.2 입금 대기 목록

**접속 경로:** 회원/결제 → 빌더 계정 개설비 관리

**표시 항목:**
- 신청일
- 사용자 이름/이메일
- 금액
- 상태 (입금 대기/입금 완료/입금 거부)
- 확인일
- 작업 버튼

### 5.3 입금 확인 절차

1. "입금 대기" 상태의 건 확인
2. 은행 계좌에서 실제 입금 여부 확인
3. "확인" 버튼 클릭
4. 상세 정보 확인 후 "확인" 버튼 클릭

**입금 확인 시 자동 처리:**
- 상태 → "입금 완료"
- 사용자 service_status → "active"
- 웰컴 크레딧 ₩50,000 자동 지급

### 5.4 입금 거부 절차

1. "거부" 버튼 클릭
2. 거부 사유 입력 (필수)
3. "거부" 클릭

## 6. 크레딧 관리

### 6.1 비즈니스 모델 및 크레딧 시스템

SSAL Works 비즈니스 모델:

| 항목 | 금액 | 결제 방식 | 비고 |
|------|------|----------|------|
| 빌더 계정 개설비 | ₩3,000,000 | 무통장 입금 | 1회 |
| 월 이용료 | ₩50,000/월 | 토스페이먼츠 | 4개월차부터 |
| 초기 무료 크레딧 | ₩50,000 | 자동 지급 | 개설비 입금 확인 시 |
| 무료 이용 기간 | 3개월 | - | 개설비 납부 후 1~3개월차 |
| 크레딧 이익률 | 20% | - | 크레딧 판매 시 마진 |

**크레딧 시스템:**
- 개설비 납부 시 ₩50,000 웰컴 크레딧 자동 지급
- 추가 크레딧은 충전을 통해 구매 (20% 이익률 적용)

### 6.2 크레딧 관리 화면

**접속 경로:** 회원/결제 → 크레딧 관리

**표시 항목:**
- 전체 크레딧 현황
- 사용자별 잔액
- 거래 내역

### 6.3 수동 크레딧 충전/차감

관리자가 필요 시 수동으로 크레딧을 조정할 수 있습니다.

1. 해당 사용자의 "충전/차감" 버튼 클릭
2. 금액 입력 (양수=충전, 음수=차감)
3. 사유 입력
4. "적용" 클릭

**사유 예시:**
- 이벤트 보너스
- 오류 보상
- 환불 처리

## 7. 문의 관리

**접속 경로:** 회원/결제 → 문의 관리

**필터 옵션:**
- 전체
- 대기 중

## 9. 트러블슈팅

### 9.1 관리자 대시보드 접속 불가

**문제:** 관리자 URL 접속 시 권한 오류

**확인:**
1. Supabase users 테이블에서 role 확인
2. `role` 값이 `'admin'`인지 확인

**해결:**
```sql
UPDATE users
SET role = 'admin'
WHERE email = '해당이메일@example.com';
```

### 9.2 데이터 로드 실패

**문제:** 목록이 로드되지 않음

**확인:**
1. 브라우저 콘솔 (F12) 에서 오류 확인
2. 네트워크 탭에서 API 응답 확인

**해결:**
- 새로고침 (Ctrl+F5)
- Supabase 대시보드에서 RLS 정책 확인
- API 키 유효성 확인

### 9.3 입금 확인 오류

**문제:** 입금 확인 버튼이 작동하지 않음

**확인:**
1. installation_fees 테이블 권한 확인
2. credit_transactions 테이블 권한 확인

**해결:**
- Supabase RLS 정책 확인
- 관리자 권한 재확인
