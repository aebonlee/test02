## 📊 5 Stages × 11 Areas = 55개 표준 영역

> **중요 변경사항 (v3.0)**:
> - 기존 6×11 → **5×11 매트릭스**로 변경
> - P1-P3 (Preliminary)는 **GRID 범위 밖**에서 별도 관리
> - PROJECT SAL GRID는 **S1-S5 실행단계만** 관리

### **전체 프로세스 (GRID 범위 포함)**
```
╔═══════════════════════════════════════════════════════════════════════╗
║              PRELIMINARY (예비단계) - GRID 범위 밖                     ║
╠═══════════════════════════════════════════════════════════════════════╣
║                                                                       ║
║   [P1 사업계획] ──→ [P2 프로젝트 기획] ──→ [P3 프로토타입 제작]        ║
║                                                                       ║
║   ※ Preliminary 단계는 GRID 생성 전에 완료해야 함                     ║
║   ※ 프로토타입(P3) 완료 후 GRID 생성                                  ║
║                                                                       ║
╚═══════════════════════════════════════════════════════════════════════╝
                              │
                              ▼
                    ┌────────────────┐
                    │   Gate 검증    │
                    │  P3 프로토타입 │
                    │  승인 여부     │
                    └────────────────┘
                              │
                              ▼
╔═══════════════════════════════════════════════════════════════════════╗
║                PROJECT SAL GRID 관리 범위 (S1-S5)                     ║
╠═══════════════════════════════════════════════════════════════════════╣
║                                                                       ║
║   Stage 1: 개발 준비 (Development Setup)        ← 일부 Areas          ║
║   Stage 2: 개발 1차 (Core Development)          ← 11 Areas            ║
║   Stage 3: 개발 2차 (Advanced Features)         ← 11 Areas            ║
║   Stage 4: 개발 3차 (QA & Optimization)         ← 일부 Areas          ║
║   Stage 5: 운영 (Operations)                    ← 9 Areas             ║
║                                                                       ║
╚═══════════════════════════════════════════════════════════════════════╝
```

### **기호 설명**
- **P (Preliminary)**: 예비단계 - GRID 범위 밖 (P1, P2, P3)
- **S (Stage)**: 실행단계 - GRID 관리 범위 (S1, S2, S3, S4, S5)

### **5 Stages (GRID 관리 범위)**
```
Stage 1: Development Setup (개발 준비)
Stage 2: Core Development (개발 1차)
Stage 3: Advanced Features (개발 2차)
Stage 4: QA & Optimization (개발 3차)
Stage 5: Operations (운영)
```

### **11 Areas (영역)**
```
M  - Documentation (문서화)
U  - Design (UI/UX 디자인)
F  - Frontend (프론트엔드)
BI - Backend Infrastructure (백엔드 기반)
BA - Backend APIs (백엔드 API)
D  - Database (데이터베이스)
S  - Security (보안/인증/인가)
T  - Testing (테스트)
O  - DevOps (운영/배포)
E  - External (외부 연동)
C  - Content System (콘텐츠 시스템)
```

## 🎯 매트릭스 활용 방법

### **1. 프로젝트 초기 설정**
```
1. P1, P2, P3 완료 (GRID 범위 밖)
2. P3 → S1 Gate 통과
3. PROJECT SAL GRID 생성
4. 5개 Stage 확인
5. 각 Stage에서 필요한 Area 선택
6. 55개 조합 중 실제 사용할 영역만 활성화
```

### **2. Task 생성**
```
예시: "회원가입 API 구현"
→ Stage 2 (개발 1차) + Area BA (Backend APIs)
→ Task ID: S2BA1
```

### **3. 진행 상황 추적**
```
5×11 그리드에서:
- 완료된 영역: ✅
- 진행중인 영역: 🔄
- 계획된 영역: 📋
- 불필요한 영역: -
```

## 🎯 원칙

### **모든 작업은 5×11 매트릭스 안에 배치**
- 새로운 작업이 생기면 → 기존 55개 조합 중 하나에 매핑
- 어디에도 맞지 않으면 → 가장 가까운 영역에 포함
- 절대 새로운 Stage나 Area 추가하지 않음

### **프로젝트 특성에 따라 선택적 사용**
- 간단한 프로젝트: 15-30개 영역 사용
- 복잡한 프로젝트: 40-55개 영역 사용

### **P1-P3는 GRID 밖에서 관리**
- Preliminary 단계는 별도 폴더 및 관리 체계
- P3 완료 후 Gate 검증 → GRID 생성 → S1 시작
