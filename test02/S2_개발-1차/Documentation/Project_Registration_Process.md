## 1. 개요

### 프로젝트 등록이란?

SSAL Works에서 **새로운 웹사이트 프로젝트**를 시작하기 위해 빌더가 수행하는 초기 설정 프로세스입니다.

### 대상 사용자

- 빌더 계정을 보유한 사용자
- 새로운 웹사이트 프로젝트를 시작하려는 사용자

### ⭐ 첫 번째 프로젝트 vs 두 번째 이후 프로젝트

| 구분 | 첫 번째 프로젝트 | 두 번째 이후 프로젝트 |
|------|-----------------|---------------------|
| **Dev Package 다운로드** | 필요 ✅ | 불필요 ❌ |
| **개발 도구 설치** | 필요 ✅ | 불필요 ❌ |
| **Claude Code 설치** | 필요 ✅ | 불필요 ❌ |
| **서약서 동의** | 필요 ✅ | 불필요 ❌ (이미 동의) |
| **프로젝트 등록 모달** | 필요 ✅ | 필요 ✅ |
| **패키지 설치 안내 모달** | 표시 ✅ | 표시 안 함 ❌ |

## 2. 등록 전 조건

| 조건 | 설명 | 확인 방법 |
|------|------|----------|
| **빌더 계정 개설** | 빌더 계정이 개설되어 있어야 함 | `users.installation_fee_paid = true` |
| **진행 중인 프로젝트 없음** | 한 번에 하나의 프로젝트만 진행 가능 | `projects` 테이블 확인 |
| **기존 프로젝트 완료** | 진행 중인 프로젝트가 있으면 완료 후 등록 | 모든 프로세스 100% |

## 3. 첫 번째 프로젝트 등록 프로세스

> **적용 대상**: 빌더 계정 개설 후 처음으로 프로젝트를 등록하는 경우

```
┌─────────────────────────────────────────────────────────────────┐
│              첫 번째 프로젝트 등록 프로세스 (Full)               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [1] 프로젝트 등록 모달에서 등록                                 │
│      ↓                                                          │
│  [2] 패키지 설치 안내 모달 표시 ← 첫 번째만!                     │
│      ↓                                                          │
│  [3] Dev Package 다운로드                                       │
│      ↓                                                          │
│  [4] 개발 도구 설치 (Git, Node.js, Claude Code)                 │
│      ↓                                                          │
│  [5] Claude Code로 개발 환경 설정                               │
│      ↓                                                          │
│  [6] 플랫폼 사용법 안내 전화 (선택)                              │
│      ↓                                                          │
│  [7] 서약서 동의 (예시 프로젝트 열람 시)                         │
│      ↓                                                          │
│  [8] 개발 진행                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Step 1: 프로젝트 등록

대시보드 좌측 사이드바에서 **[➕ 새 프로젝트]** 클릭 → 프로젝트 정보 입력 → 등록

### Step 2: 패키지 설치 안내 확인

프로젝트 등록 완료 후 **패키지 설치 안내 모달**이 자동으로 표시됩니다.

### Step 3: Dev Package 다운로드

**Dev Package**는 웹사이트 개발에 필요한 모든 폴더와 파일이 미리 구성된 템플릿입니다.

**패키지 내용:**

| 폴더/파일 | 용도 |
|-----------|------|
| `P0_작업_디렉토리_구조_생성/` | 프로젝트 상태 관리 |
| `P1_사업계획/` | 사업 기획 |
| `P2_프로젝트_기획/` | 프로젝트 설계 |
| `P3_프로토타입_제작/` | 프로토타입 개발 |
| `S0~S5_개발/` | 본개발 단계 |
| `.claude/` | AI 작업 규칙 |
| `Briefings_OrderSheets/` | 안내문/작업지시서 |
| `Human_ClaudeCode_Bridge/` | AI 통신 |

**다운로드 후 작업:**
1. ZIP 파일 압축 해제
2. 원하는 위치에 폴더 이동 (권장: `C:\Projects\내프로젝트명`)
3. 폴더 이름을 프로젝트에 맞게 변경

### Step 4: 개발 도구 설치

| 도구 | 역할 | 설치 방법 |
|------|------|----------|
| **Git** | 버전 관리 | https://git-scm.com |
| **Node.js** | 실행 환경 | https://nodejs.org (LTS 버전) |
| **npm** | 패키지 관리 | Node.js에 포함 |
| **Claude Code** | AI 어시스턴트 | `npm install -g @anthropic-ai/claude-code` |

### Step 5: Claude Code로 개발 시작

1. 패키지 폴더에서 터미널 열기
2. `claude` 명령어 실행
3. "프로젝트 개발 환경 설정을 위한 필수 도구 다 설치해 줘" 요청

### Step 6: 플랫폼 사용법 안내 (권장)

- **소요 시간**: 10분 이내
- **내용**: 플랫폼 사용법, 개발 진행 방법 안내
- **전화번호**: 010-5067-8306

## 4. 두 번째 이후 프로젝트 등록 프로세스

> **적용 대상**: 이미 한 개 이상의 프로젝트를 완료한 빌더

```
┌─────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [1] 기존 프로젝트 완료 확인 (모든 프로세스 100%)               │
│      ↓                                                          │
│  [2] 프로젝트 등록 모달에서 등록                                 │
│      ↓                                                          │
│  [3] 개발 진행 (기존 환경 그대로 사용)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 왜 간단한가?

- **Dev Package**: 이미 설치되어 있음 (새 프로젝트 폴더만 생성)
- **개발 도구**: Git, Node.js, Claude Code 이미 설치됨
- **서약서**: 이미 동의 완료
- **플랫폼 사용법**: 이미 숙지

### Step 1: 기존 프로젝트 완료 확인

- 현재 진행 중인 프로젝트의 모든 프로세스가 100%여야 함
- 완료되지 않은 상태에서 새 프로젝트 등록 시 경고 메시지 표시

### Step 2: 프로젝트 등록

대시보드 좌측 사이드바에서 **[➕ 새 프로젝트]** 클릭 → 프로젝트 정보 입력 → 등록

- 패키지 설치 안내 모달 **표시되지 않음**
- 바로 개발 시작 가능

### Step 3: 개발 진행

기존 환경을 그대로 사용하여 새 프로젝트 개발 시작

## 5. 서약서 동의 절차

### 서약서 표시 시점

- 예시 프로젝트 열람 시 최초 1회 표시
- 동의 후 재표시 없음

### 서약서 내용 요약

1. **참고자료 활용 및 비공유**: 타인에게 제공 금지
2. **유사 프로젝트 개발 제한**: 유사 프로젝트 개발 시 사전 협의 필요
3. **사업화 목적 활용**: 세상에 도움이 되는 서비스로 사업화

### 동의 후 처리 로직

```javascript
// 서약 완료 시
async function submitAgreement() {
    // 1. users 테이블 업데이트
    await supabase.from('users').update({
        pledge_agreed: true,
        pledge_agreed_at: new Date().toISOString()
    }).eq('id', user.id);

    // 2. 예시 프로젝트 열람 권한 부여
    // 3. 성공 모달 표시
}
```

### DB 저장 필드

| 테이블 | 필드 | 타입 | 설명 |
|--------|------|------|------|
| `users` | `pledge_agreed` | boolean | 서약 동의 여부 |
| `users` | `pledge_agreed_at` | timestamp | 서약 동의 시간 |

## 9. 5단계 개발 프로세스
