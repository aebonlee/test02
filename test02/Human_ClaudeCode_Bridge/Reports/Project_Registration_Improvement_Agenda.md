## 문서별 개편 계획

### 1. Default.md (인간용 - 첫 방문)

**목적**: "SSAL Works가 뭔지, 왜 필요한지, 관심 유발"

**목표 분량**: 100-120줄 (현재 159줄)

**포함할 내용**:
- [ ] SSAL Works 소개 (짧게)
- [ ] 누구를 위한 플랫폼?
- [ ] 둘러보기 안내 (어디 뭐가 있는지)
- [ ] 빌더 계정 혜택/가격
- [ ] 비전

**제외할 내용** (다른 문서로 이동):
- [ ] Dev Package 구조 → Project_Registration으로
- [ ] 설치 방법 → Project_Registration으로
- [ ] Claude Code 규칙 → README로

**작업 내용**:
1. 마케팅/온보딩 내용만 남기고 나머지 삭제
2. "자세한 설치 방법은 프로젝트 등록 시 안내됩니다" 문구 추가
3. 구조 정리하여 읽기 쉽게

### 2. Project_Registration.md (인간+AI 공용 - 등록 시)

**목적**: "등록하고, 설치하고, 첫 실행까지"

**목표 분량**: 150-180줄 (현재 406줄 → **대폭 축소**)

**포함할 내용**:

#### 인간용 파트
- [ ] STEP 1: Dev Package 다운로드 (간략)
- [ ] STEP 2: 개발 도구 설치 (Git, Node, Claude Code)
  - Windows/Mac 모두 지원
  - Claude Code 비용 안내 포함
- [ ] STEP 3: 첫 실행 방법
  - 터미널 사용법 (초보자용)
  - .ssal-project.json 초기 설정

#### Claude Code용 파트
- [ ] 자동 초기화 시 수행할 작업 체크리스트
- [ ] 설치 확인 체크리스트

**폴더 구조 처리 방침**:
- 전체 트리: **전부 보여줌** (인간이 뭐가 생기는지 알아야 함)
- 설명: **간략하게** (각 폴더 한 줄 설명)
- 상세 설명: README.md 참조로 연결

**제외할 내용** (README로 이동):
- [ ] 각 폴더의 상세 역할/사용법 설명
- [ ] 7대 규칙 상세 설명 → README
- [ ] JSON CRUD 방법 → README

**작업 내용**:
1. 406줄 → 180-200줄로 축소
2. 폴더 트리는 유지, 설명만 간략화
3. 설치에만 집중하는 Quick Start 형태로 재구성
4. 기존 7개 아젠다 중 관련 내용 통합:
   - Claude Code 비용 안내 (아젠다 #2)
   - .ssal-project.json 초기 설정 (아젠다 #3)
   - 터미널 설명 (아젠다 #4)
   - Mac 지원 (아젠다 #5)
   - 초기화 명령어 구체화 (아젠다 #6)
