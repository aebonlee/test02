## 개요

Admin Dashboard는 SSALWorks Dashboard의 공지사항과 학습 콘텐츠를 관리하는 관리자 전용 페이지입니다.

**주요 기능:**
- 공지사항 작성/수정/삭제
- 학습 콘텐츠 3단계 계층 구조 관리
- 실시간 데이터 동기화
- 통계 확인

**파일 위치:**
```
C:\!SSAL_Works_Private\P3_프로토타입_제작\Frontend\Prototype\admin-dashboard_prototype.html
```

## 공지사항 관리

### 위치
왼쪽 사이드바 > "공지사항 관리" 클릭

---

### 공지사항 추가하기

**단계:**

1. **"공지사항 추가" 버튼 클릭**
   - 화면 우측 상단의 녹색 버튼

2. **정보 입력**
   ```
   제목: (필수) 공지사항 제목 입력
   내용: (필수) 공지사항 내용 입력
   중요 표시: 체크박스 선택/해제
   ```

   **중요 표시란?**
   - 체크 시: 빨간 아이콘(🔴)과 함께 최우선 표시
   - Frontend에서 상단에 표시됨

3. **"추가" 버튼 클릭**
   - 성공 시: "공지사항이 추가되었습니다" 메시지
   - 목록에 즉시 반영

**예시:**
```
제목: 시스템 점검 안내
내용:
2025년 12월 10일 오전 2시-4시
시스템 정기 점검이 있습니다.
해당 시간에는 서비스 이용이 불가합니다.

중요 표시: ☑ (체크)
```

---

### 공지사항 수정하기

**단계:**

1. **수정할 공지사항 찾기**
   - 목록에서 확인

2. **[수정] 버튼 클릭**
   - 각 공지사항 옆의 파란색 버튼

3. **정보 수정**
   - 기존 내용이 자동으로 입력됨
   - 원하는 부분 수정

4. **"수정" 버튼 클릭**
   - 성공 시: "공지사항이 수정되었습니다" 메시지

---

### 공지사항 삭제하기

**단계:**

1. **삭제할 공지사항 찾기**

2. **[삭제] 버튼 클릭**
   - 각 공지사항 옆의 빨간색 버튼

3. **확인 모달**
   ```
   "공지사항명"을(를) 삭제하시겠습니까?
   [취소] [삭제]
   ```

4. **"삭제" 버튼 클릭**
   - 성공 시: "공지사항이 삭제되었습니다" 메시지
   - 목록에서 즉시 제거

**⚠️ 주의:** 삭제된 공지사항은 복구할 수 없습니다!

---

### 공지사항 통계

화면 상단에 표시:
```
📊 통계
총 공지사항: 15개
중요 공지: 3개
```

## 학습 콘텐츠 관리

### 위치
왼쪽 사이드바 > "학습 콘텐츠 관리" 클릭

---

### 계층 구조 이해하기

학습 콘텐츠는 **3단계 계층 구조**로 구성됩니다:

```
Depth1 (대분류)
└── Depth2 (중분류)
    └── Depth3 (소분류) ← 실제 학습 자료 (URL)
```

**예시:**
```
웹개발 기초 (Depth1)
├── HTML/CSS (Depth2)
│   ├── HTML 기본 구조 (Depth3) → https://...
│   ├── CSS 선택자 (Depth3) → https://...
│   └── Flexbox 레이아웃 (Depth3) → https://...
├── JavaScript (Depth2)
│   ├── JavaScript 기초 (Depth3) → https://...
│   └── DOM 조작 (Depth3) → https://...
└── ...
```

---

### Depth1 (대분류) 추가하기

**단계:**

1. **"대분류 추가" 버튼 클릭**
   - 화면 우측 상단

2. **이름 입력**
   ```
   대분류 이름: 예) "데이터 분석"
   ```

3. **"추가" 버튼 클릭**

**결과:**
```
데이터 분석 (새 대분류 생성됨)
```

---

### Depth2 (중분류) 추가하기

**단계:**

1. **대분류 옆 "+중분류" 버튼 클릭**
   ```
   웹개발 기초 [수정] [+중분류] [삭제]
                      ↑ 여기
   ```

2. **이름 입력**
   ```
   중분류 이름: 예) "TypeScript"
   ```

3. **"추가" 버튼 클릭**

**결과:**
```
웹개발 기초
└── TypeScript (새 중분류 추가됨)
```

---

### Depth3 (소분류) 추가하기

**가장 중요! 실제 학습 자료 등록**

**단계:**

1. **중분류 옆 "+소분류" 버튼 클릭**
   ```
   HTML/CSS [수정] [+소분류] [삭제]
                   ↑ 여기
   ```

2. **정보 입력**
   ```
   소분류 이름: (필수) 예) "TypeScript 기초"
   URL: (필수) https://www.youtube.com/watch?v=...
   설명: (선택) TypeScript 기본 문법 강의
   ```

3. **"추가" 버튼 클릭**

**결과:**
```
웹개발 기초
└── TypeScript
    └── TypeScript 기초 → https://...
```

**Frontend에서 보이는 모습:**
```
[웹개발 기초] ▶
  [TypeScript] ▶
    • TypeScript 기초 (클릭 시 URL로 이동)
```

---

### 학습 콘텐츠 수정하기

**Depth1/2 수정:**
1. 해당 항목 옆 [수정] 버튼 클릭
2. 새 이름 입력
3. "수정" 클릭

**⚠️ 중요:** Depth1/2를 수정하면 **모든 하위 항목**에도 자동 반영됩니다!

**Depth3 수정:**
1. 소분류 옆 [수정] 버튼 클릭
2. 이름, URL, 설명 수정
3. "수정" 클릭

---

### 학습 콘텐츠 삭제하기

**⚠️ Cascade 삭제 주의!**

#### Depth1 (대분류) 삭제

1. 대분류 옆 [삭제] 버튼 클릭

2. **경고 모달 표시:**
   ```
   "웹개발 기초"을(를) 삭제하시겠습니까?

   ⚠️ 경고: 모든 하위 중분류 및 소분류도 함께 삭제됩니다!

   [취소] [삭제]
   ```

3. "삭제" 클릭 시:
   - 대분류 삭제
   - 해당 대분류의 **모든 중분류 삭제**
   - 해당 대분류의 **모든 소분류 삭제**

**예시:**
```
웹개발 기초 (삭제)
├── HTML/CSS (함께 삭제됨)
│   ├── HTML 기본 구조 (함께 삭제됨)
│   └── CSS 선택자 (함께 삭제됨)
└── JavaScript (함께 삭제됨)
    └── ... (함께 삭제됨)
```

#### Depth2 (중분류) 삭제

1. 중분류 옆 [삭제] 버튼 클릭

2. **경고 모달:**
   ```
   "HTML/CSS"를 삭제하시겠습니까?

   ⚠️ 경고: 모든 하위 소분류도 함께 삭제됩니다!
   ```

3. "삭제" 클릭 시:
   - 중분류 삭제
   - 해당 중분류의 **모든 소분류 삭제**

#### Depth3 (소분류) 삭제

1. 소분류 옆 [삭제] 버튼 클릭

2. **간단한 확인:**
   ```
   "HTML 기본 구조"를 삭제하시겠습니까?
   ```

3. "삭제" 클릭 시:
   - 해당 소분류만 삭제

## 주의사항

### ⚠️ 데이터 안전

1. **삭제 전 반드시 확인**
   - 삭제된 데이터는 복구 불가능
   - Cascade 삭제 시 하위 항목 모두 삭제됨

2. **중요 공지사항 표시**
   - 중요 표시는 신중하게 사용
   - Frontend에서 최우선 표시됨

3. **URL 형식 확인**
   - 학습 자료 URL은 반드시 `https://`로 시작
   - 올바른 YouTube, 문서 링크 확인

### ⚠️ 네이밍 규칙

1. **명확한 이름 사용**
   ```
   ✅ 좋은 예:
   - HTML 기본 구조
   - CSS Flexbox 레이아웃
   - JavaScript 배열 메서드

   ❌ 나쁜 예:
   - 강의1
   - 테스트
   - 임시
   ```

2. **일관된 명명**
   - 같은 레벨에서는 유사한 형식 사용
   - 예: "HTML 기초", "CSS 기초", "JavaScript 기초"

### ⚠️ 계층 구조 유지

1. **3단계 구조 준수**
   - 반드시 Depth1 → Depth2 → Depth3 순서로 생성
   - Depth2 없이 Depth3 생성 불가

2. **적절한 분류**
   - 대분류: 넓은 개념 (예: 웹개발, 앱개발)
   - 중분류: 세부 기술 (예: React, Vue)
   - 소분류: 구체적 주제 (예: useState Hook)

## 문제 해결

### 문제 1: 데이터가 표시되지 않음

**원인:**
- Supabase 연결 실패
- 네트워크 오류

**해결 방법:**
1. F12 → Console 탭 확인
2. 오류 메시지 확인
3. 페이지 새로고침 (Ctrl + F5)

---

### 문제 2: 추가/수정/삭제 실패

**원인:**
- RLS 정책 (권한 없음)
- 필수 필드 누락

**해결 방법:**

1. **RLS 정책 확인**
   ```sql
   -- Supabase Dashboard에서 확인
   -- authenticated 역할 필요
   ```

2. **필수 필드 확인**
   - 공지사항: 제목, 내용
   - 학습 콘텐츠: 모든 depth, URL (Depth3)

---

### 문제 3: Cascade 삭제 실수

**원인:**
- 경고 확인 없이 대분류/중분류 삭제

**해결 방법:**
- **삭제 전 반드시 백업**
- 실수로 삭제한 경우:
  1. Supabase Dashboard 접속
  2. Database → learning_contents
  3. 수동으로 재입력

**예방:**
- 삭제 전 확인 모달 꼼꼼히 읽기
- 중요 데이터는 별도 백업

---

### 문제 4: Frontend에 반영 안 됨

**원인:**
- 브라우저 캐시

**해결 방법:**
1. Frontend 페이지 새로고침 (Ctrl + F5)
2. 캐시 삭제 (Ctrl + Shift + Delete)
