## 프로젝트 초기 요청 프롬프트

### 사용자 원본 요청
```
프로젝트 구조 설계 
& OOP 기초
학생 출결·성적 관리 시스템 (1.5시간 × 5회)
• 요구사항 분석 및 시스템 구조 설계• Student/Attendance/Grade 클래스 구조 이해• 실습: Student 클래스 구현 및 객체 리스트 생성
출결 관리 기능 구현
• datetime 기반 출결 처리 구조 이해• 실습: Attendance 클래스 구현, 출결 기록 저장• JSON 데이터 저장 기능 기초 구현
성적 관리 기능 구현
• 성적 구조(시험/과제) 설계• 실습: Grade 클래스 구현 및 CRUD 기능 작성• 예외 처리(입력 오류·범위 검증) 적용
GUI 화면 구성(Tkinter)
• 기본 위젯(Button, Entry, Treeview) 이해• 실습: 학생/출결/성적 관리 GUI 화면 구성
기능 통합 및 테스트
• 파일 저장 및 전체 기능 연동• 실습: 모든 GUI 화면 통합 및 오류 수정• 최종 테스트 및 시스템 v1.0 완성

이 내용으로 파이썬 프로젝트를 하려고 해. 설계해서 구현해줘
```

## 효과적인 프롬프트 패턴

### 1. 프로젝트 설계 요청 시
```
[프로젝트명] 프로젝트를 다음 요구사항으로 설계해주세요:
- 주요 기능: [기능 목록]
- 기술 스택: [언어/프레임워크]
- 아키텍처: [MVC, OOP 등]
- 추가 요구사항: [성능, 확장성 등]
```

### 2. 클래스 구현 요청 시
```
[클래스명] 클래스를 다음 사양으로 구현해주세요:
- 속성: [필드 목록]
- 메서드: [기능 목록]
- 상속/인터페이스: [관계 정의]
- 예외 처리: [검증 로직]
```

### 3. GUI 구현 요청 시
```
[화면명] GUI를 Tkinter로 구현해주세요:
- 레이아웃: [배치 구조]
- 위젯: [사용할 컴포넌트]
- 이벤트: [사용자 상호작용]
- 스타일: [디자인 가이드]
```

### 4. 통합/테스트 요청 시
```
전체 시스템을 통합하고 다음 사항을 확인해주세요:
- 모듈 간 연동 테스트
- 데이터 흐름 검증
- 오류 처리 확인
- 사용자 시나리오 테스트
```

## 고급 프롬프트 기법

### 1. 단계별 개발 요청
```
다음 순서로 개발해주세요:
1. [1단계 작업]
2. [2단계 작업]
3. [3단계 작업]

각 단계별로 완료 후 다음 단계 진행 여부를 확인해주세요.
```

### 2. 조건부 구현 요청
```
다음 조건을 만족하는 [기능]을 구현해주세요:
- 필수 조건: [반드시 포함해야 할 요소]
- 선택 조건: [있으면 좋은 요소]
- 제약 조건: [피해야 할 요소]
```

### 3. 성능 최적화 요청
```
[기능]의 성능을 최적화해주세요:
- 시간 복잡도: [목표 복잡도]
- 메모리 사용량: [제한 사항]
- 병목 지점: [예상 문제점]
```

## 프롬프트 작성 팁

### DO's
- 구체적이고 명확한 요구사항 명시
- 예상되는 결과물 설명
- 기술적 제약사항 포함
- 우선순위 명시

### DON'Ts
- 모호한 표현 사용
- 너무 많은 요구사항을 한 번에 요청
- 기술적 배경 정보 누락
- 예외 상황 고려 누락
