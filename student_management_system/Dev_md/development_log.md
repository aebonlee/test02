## 기술적 결정 사항

### 1. 아키텍처 선택
**결정**: MVC (Model-View-Controller) 패턴 채택  
**이유**: 
- 비즈니스 로직과 UI 분리
- 유지보수성 향상
- 코드 재사용성 증대

### 2. 데이터 저장 방식
**결정**: JSON 파일 기반 저장  
**이유**:
- 설정이 간단하고 가벼움
- 사람이 읽기 쉬운 형식
- 백업/복원이 용이
- 향후 데이터베이스 마이그레이션 가능

### 3. GUI 프레임워크
**결정**: Tkinter 선택  
**이유**:
- Python 기본 라이브러리
- 별도 설치 불필요
- 크로스 플랫폼 지원
- 학습 곡선이 낮음

### 4. 클래스 설계 원칙
**결정**: 객체지향 설계 원칙 적용  
**세부 사항**:
- 캡슐화: private 멤버 사용 (`_variable`)
- 프로퍼티: getter/setter를 통한 데이터 접근 제어
- 검증: 입력 데이터 유효성 검사
- 직렬화: `to_dict()`, `from_dict()` 메서드 제공
